## Sprint 2 — DB 基线统一（3-5 天）

| 迁移 | 内容 | 风险/幂等 |
|---|---|---|
| 022_unify_collation | `ALTER TABLE ... CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci`，遍历 21+1 张非 0900 表；用 `INFORMATION_SCHEMA.COLUMNS.COLLATION_NAME` 守卫 | 中：CONVERT 重建索引，大表锁表，低峰执行 |
| 023_unify_soft_delete | 30+ 表 `ALTER TABLE ... MODIFY deleted TINYINT NOT NULL DEFAULT 0`；用 `IS_NULLABLE` 守卫；移除 `TINYINT(1)` 显示宽度 | 低 |

## 关键风险与缓解

1. **主键类型变更（028）break 代码**：分表提交，每表后跑 `ts-check`+全量单元测试+手动冒烟；保留旧列 7 天
2. **Collation CONVERT 锁表（022）**：低峰执行；大表用 `pt-online-schema-change`
3. **补 FK 遇孤儿数据（029+）**：每个 FK 前跑孤儿体检查询，先清理或软删孤儿再加约束
4. **JWT Cookie 迁移 break 现有客户端**：双通道过渡期，`extractToken` 同时读 Cookie 与 header
5. **StrictMode 暴露副作用 bug**：1.4 可能引发一批 dev 报错，预留修复时间
