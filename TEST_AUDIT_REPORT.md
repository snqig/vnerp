#### BUG-002: 测试用例质量低下
**严重程度**: 🔴 Critical  
**文件**: tests/inbound.spec.ts

**问题描述**:
```typescript
// 问题1: 条件判断过多，测试不严谨
if (await createButton.isVisible().catch(() => false)) {
  // 如果按钮不存在，测试仍然通过
```

- 添加数据验证和状态检查

### 2. ⚠️ 高危缺陷 (High)

#### BUG-003: SQL 注入风险
**严重程度**: 🟠 High  
**位置**: 多个 API 路由文件

**问题描述**:
虽然大部分代码使用了参数化查询，但存在潜在风险：

```typescript
// src/app/api/quality/lab-test/route.ts
// 动态构建 SQL，虽然有参数化，但字段名未验证
if (product_id !== undefined) { 
  fields.push('product_id = ?'); 
  values.push(product_id); 
}
```

**风险点**:
- 字段名来自代码而非用户输入（相对安全）
- 但缺少输入验证层
- 未使用 ORM 或查询构建器

**建议**:
- 引入 Zod 或 Joi 进行请求验证
- 使用 Drizzle ORM（项目已安装但未充分利用）
- 添加 SQL 注入扫描工具

---

#### BUG-004: 敏感信息泄露风险
**严重程度**: 🟠 High  
**文件**: 历史记录中包含 GitHub Token

**问题描述**:
- 之前提交中包含 Personal Access Token
- 虽然已使用 `git filter-branch` 清理
- 但 token 可能已被 GitHub 扫描记录

**建议**:
- ✅ 已撤销泄露的 token
- 添加 `.env` 到 `.gitignore`
- 使用 Git Hooks 防止敏感信息提交
- 启用 GitHub Secret Scanning

---

#### BUG-005: 硬编码凭据
**严重程度**: 🟠 High  

**问题描述**:
```typescript
// tests/auth.spec.ts - 硬编码测试账号
await page.fill('input#username', 'admin');
await page.fill('input#password', 'admin');
```

**风险**:
- 生产环境可能使用相同凭据
- 测试代码泄露账号信息

**建议**:
- 使用环境变量存储测试账号
- 创建专用测试账号
- 添加 `.env.test` 文件

### 3. 🔶 中危缺陷 (Medium)

#### BUG-006: 错误处理不一致
**严重程度**: 🟡 Medium  

**问题分布**: 全局

**问题描述**:
```typescript
// 有的地方使用 try-catch
try {
  const result = await query(sql, values);
} catch (error) {
  console.error(error); // 仅打印，未返回用户友好错误
}

// 有的地方使用 withErrorHandler 包装器

  // ...
});
```

**影响**: 
- 用户体验不一致
- 错误信息可能泄露系统细节
- 难以统一监控和日志

**建议**:
- 统一使用 `withErrorHandler` 包装器
- 实现全局错误边界
- 标准化错误响应格式

---

#### BUG-007: 缺少输入验证
**严重程度**: 🟡 Medium  

**问题描述**:
```typescript
// src/app/api/equipment/route.ts
const page = parseInt(searchParams.get('page') || '1');
const pageSize = parseInt(searchParams.get('pageSize') || '20');
// 未验证范围，可能导致性能问题
```

**风险**:
- `pageSize=999999` 可能导致服务器崩溃
- 负数页码可能引发错误
- 缺少类型验证

**建议**:
```typescript
// 使用 Zod 验证
const querySchema = z.object({
  page: z.coerce.number().min(1).max(1000).default(1),
  pageSize: z.coerce.number().min(1).max(100).default(20),
});
```

---

#### BUG-008: 性能问题 - N+1 查询
**严重程度**: 🟡 Medium  

**问题描述**:
在列表查询中可能存在 N+1 问题：
- 先查询主表
- 循环中查询关联表

**影响**: 
- 响应时间慢
- 数据库负载高

**建议**:
- 使用 JOIN 查询
- 实现数据加载器
- 添加查询性能监控
