# 模块一：系统全局配置中心（优先级 P1 - 基石）

## 1.1 模块概述

**目标**：建立全系统唯一的参数配置入口，实现"**配置驱动、不写死代码、管理员可改、修改需审批**"的核心架构。

### 核心价值

```
传统模式：业务规则硬编码 → 改规则需重新开发部署 → 响应慢、风险高
    ↓
VNERP模式：业务规则配置化 → 管理员在线修改 → 即时生效、零代码变更
```

## 1.4 技术实现要点

### 1.4.1 配置缓存策略

```typescript
// 缓存设计：TTL = 5分钟
const CACHE_TTL = 5 * 60 * 1000; // 5分钟
let configCache: Map<string, any> = new Map();
let cacheTimestamp: number = 0;

export function getConfig(key: string): any {
  const now = Date.now();
  
  // 缓存未过期，直接返回
  if (now - cacheTimestamp < CACHE_TTL && configCache.has(key)) {
    return configCache.get(key);
  }
  
  // 缓存过期或不存在，从数据库加载
  const value = loadFromDatabase(key);
  configCache.set(key, value);
  cacheTimestamp = now;
  
  return value;
}
```

### 1.4.2 默认值兜底机制

```typescript
// 即使数据库无记录，也返回默认值，确保系统稳定运行
const DEFAULT_CONFIGS: Record<string, any> = {
  mould_life_days: 90,
  screen_life_days: 60,
  fifo_enabled: true,
  // ... 45+个默认值
};

function loadFromDatabase(key: string): any {
  const dbValue = queryDB(key);
  return dbValue ?? DEFAULT_CONFIGS[key];
}
```

# 模块二：仓库管理7步闭环（优先级 P4 - 核心）

## 2.1 模块概述

**目标**：将20年实战仓库管理经验固化到系统中，实现**强制闭环、不可绕过**的标准化管理。

### 核心理念

```
传统仓库问题：
❌ 整料直接发料 → 浪费严重
❌ 随意选择批次 → FIFO无法执行
❌ 无单也能领料 → 库存混乱
❌ 盘点靠月底 → 问题发现太晚

VNERP解决方案：
✅ 强制拆小料 → 精细化管控
✅ 系统推荐批次 → FIFO严格执行
✅ 无单绝对不发 → 流程规范化
✅ 循环自动盘点 → 问题早发现早处理
```

## 2.2 七步闭环详解

### 第一步：验收入库（IQC检验前置）

**业务场景**：采购物料到货后，必须经过IQC检验才能入库

**系统强制规则**：

```sql
-- 入库流程控制
CREATE PROCEDURE sp_inbound_check(
  IN p_inbound_id INT,
  IN p_iqc_result VARCHAR(20)
)
BEGIN
  DECLARE v_status INT;
  
  -- 获取当前入库单状态
  SELECT status INTO v_status FROM inv_inbound_order WHERE id = p_inbound_id;
  
  -- 只有待检状态才能操作
  IF v_status != 1 THEN
    SIGNAL SQLSTATE '45000' 
    SET MESSAGE_TEXT = '当前状态不允许此操作';
  END IF;
  
  -- IQC合格才能入库
  IF p_iqc_result = 'qualified' THEN
    UPDATE inv_inbound_order SET status = 4 WHERE id = p_inbound_id;
    -- 自动分配到待检区库位
    CALL sp_assign_location(p_inbound_id, 'WAIT');
  ELSE
    -- 不合格转入不良品区
    UPDATE inv_inbound_order SET status = 5 WHERE id = p_inbound_id;
    CALL sp_assign_location(p_inbound_id, 'BAD');
  END IF;
END;
```

**关键控制点**：
- ✅ **强制IQC检验**：未检验物料禁止入库
- ✅ **自动分区**：合格→待检区，不合格→不良品区
- ✅ **检验留痕**：记录检验人、检验时间、检验结果

### 第二步：大料拆小料（自动化拆分引擎）

**业务场景**：整卷物料（如100米PET薄膜）必须拆分成标准小料（10米/卷）后才能发料

**拆分规则表**（从全局配置读取）：

| 原料类型 | 整料规格 | 拆分单位 | 小料规格 | 配置键 |
|---------|---------|---------|---------|--------|
| PET薄膜 | 100米/卷 | 10米/卷 | 10米/卷 | `film_split_length` |
| PVC薄膜 | 100米/卷 | 10米/卷 | 10米/卷 | `pvc_split_length` |
| 油墨 | 5kg/桶 | 1kg/桶 | 1kg/桶 | `ink_split_weight` |
| 溶剂 | 20L/桶 | 5L/桶 | 5L/桶 | `solvent_split_volume` |
| 网布 | 50米/卷 | 10米/卷 | 10米/卷 | `mesh_split_length` |

**系统自动拆分逻辑**：

```typescript
async function autoSplitMaterial(inboundItemId: number) {
  const item = await getInboundItem(inboundItemId);
  const material = await getMaterial(item.material_id);
  
  if (!material.split_enable) {
    return; // 不需要拆分的物料跳过
  }
  
  const splitUnit = getConfig(`${material.type}_split_length`);
  const childCount = Math.floor(item.qty / splitUnit);
  const remainder = item.qty % splitUnit;
  
  // 生成子二维码（父子关联）
  for (let i = 0; i < childCount; i++) {
    const childQR = generateChildQR(item.qr_code, i);
    
    await createMaterialSplit({
      parent_qr_code: item.qr_code,
      child_qr_code: childQR,
      material_id: item.material_id,
      parent_qty: item.qty,
      child_qty: splitUnit,
      status: 'available'
    });
    
    // 创建小料批次库存
    await createInventoryBatch({
      material_id: item.material_id,
      batch_no: item.batch_no,
      qr_code: childQR,
      qty: splitUnit,
      location_id: getSmallMaterialLocation(), // 小料区
      inbound_time: new Date()
    });
  }
  
  // 余料处理（不足标准单位的自动标记为余料）
  if (remainder > 0) {
    await createRemainderMaterial(item, remainder);
  }
}
```

**关键控制点**：
- ✅ **整料禁发**：整料库存锁定，只能拆分后的小料出库
- ✅ **自动生成子QR**：每个小料有唯一二维码，支持追溯
- ✅ **余料自动识别**：不足标准单位的自动标记为余料
- ✅ **拆分记录完整**：保留父子关系，支持全链路查询

### 第四步：账卡一致（实时同步机制）

**双轨记账原则**：

```
每一笔库存变动同时更新：
├── 总账（inv_inventory）：按物料汇总的数量和金额
├── 批次账（inv_inventory_batch）：按批次+二维码明细
└── 货位卡（warehouse_location_card）：按库位的实时台账
```

**数据一致性保证**：

```typescript
// 使用数据库事务确保三账一致
async function updateInventory(transaction, operation) {
  try {
    // 1. 更新总账
    await transaction.execute(`
      UPDATE inv_inventory 
      SET qty = qty + ?, 
          last_update_time = NOW()
      WHERE material_id = ?
    `, [operation.qty, operation.material_id]);
    
    // 2. 更新批次账
    await transaction.execute(`
      INSERT INTO inv_inventory_batch_log 
      (batch_id, operation_type, qty, operator, operate_time)
      VALUES (?, ?, ?, ?, NOW())
    `, [operation.batch_id, operation.type, operation.qty, operation.operator]);
    
    // 3. 更新货位卡
    await transaction.execute(`
      INSERT INTO warehouse_location_card 
      (location_id, material_id, batch_no, operation_type, qty, balance, time)
      VALUES (?, ?, ?, ?, ?, 
        (SELECT balance FROM warehouse_location_card 
         WHERE location_id = ? AND material_id = ? 
         ORDER BY id DESC LIMIT 1) + ?,
        NOW())
    `, [operation.location_id, operation.material_id, operation.batch_no, 
        operation.type, operation.qty, operation.location_id, 
        operation.material_id, operation.qty]);
    
    await transaction.commit();
  } catch (error) {
    await transaction.rollback();
    throw error;
  }
}
```

### 第五步：先进先出FIFO（系统强制推荐）

**FIFO算法实现**：

```typescript
async function getFIFOBatches(
  materialId: number, 
  requiredQty: number
): Promise<FIFOBatch[]> {
  
  if (!getConfig('fifo_enabled')) {
    // 如果关闭FIFO，返回所有可用批次（按用户选择）
    return getAllAvailableBatches(materialId);
  }
  
  // FIFO开启时，严格按入库时间排序
  const batches = await query(`
    SELECT id, batch_no, qr_code, qty, inbound_time, location_id
    FROM inv_inventory_batch
    WHERE material_id = ?
      AND qty > 0
      AND status = 'available'
    ORDER BY inbound_time ASC, id ASC
  `, [materialId]);
  
  let remainingQty = requiredQty;
  const selectedBatches: FIFOBatch[] = [];
  
  for (const batch of batches) {
    if (remainingQty <= 0) break;
    
    const allocQty = Math.min(batch.qty, remainingQty);
    selectedBatches.push({
      ...batch,
      allocated_qty: allocQty
    });
    
    remainingQty -= allocQty;
  }
  
  if (remainingQty > 0) {
    throw new Error(`库存不足！还需 ${remainingQty} 个单位`);
  }
  
  return selectedBatches;
}
```

```typescript
class LifecycleManager {
  
  async checkAndWarn() {
    // 检查所有在用耗材
    const activeItems = await this.getActiveConsumables();
    
    for (const item of activeItems) {
      const rules = this.getLifecycleRules(item.type);
      const usage = await this.getCurrentUsage(item.id);
      
      // 计算剩余寿命
      const daysRemaining = this.calcDaysRemaining(item.activate_date, rules.validDays);
      const timesRemaining = rules.maxTimes - usage.timesUsed;
      
      // 判断是否需要预警
      if (daysRemaining <= rules.warnDays || timesRemaining <= rules.warnTimes) {
        await this.sendWarning(item, daysRemaining, timesRemaining);
      }
      
      // 判断是否需要锁定
      if (daysRemaining <= 0 || timesRemaining <= 0) {
        await this.lockItem(item, daysRemaining, timesRemaining);
      }
    }
  }
  
  async lockItem(item: ConsumableItem, daysLeft: number, timesLeft: number) {
    // 更新状态为已锁定
    await updateStatus(item.id, 'locked');
    
    // 记录锁定原因
    await createLockRecord({
      consumable_id: item.id,
      lock_reason: daysLeft <= 0 ? 'expired' : 'usage_exceeded',
      detail: daysLeft <= 0 
        ? `已过期 ${Math.abs(daysLeft)} 天` 
        : `使用次数超标，超出 ${Math.abs(timesLeft)} 次`,
      locked_by: 'system',
      locked_at: new Date()
    });
    
    // 发送通知给相关人员
    await notifyLocked(item);
    
    // 如果有关联的在制工单，标记风险
    const relatedWorkOrders = await getRelatedWorkOrders(item.id);
    for (const wo of relatedWorkOrders) {
      await markWorkOrderAtRisk(wo.id, `${item.type}已锁定，可能影响生产`);
    }
  }
}
```

```typescript
async function getTraceabilityChain(qrCode: string): Promise<TraceChain[]> {
  const chain: TraceChain[] = [];
  let currentQR = qrCode;

    const record = await getQRRecord(currentQR);
```

## 6.2 成本核算模型

### 6.2.1 工单成本构成

```
工单总成本 = 材料成本 + 人工成本 + 制造费用

其中：
├── 材料成本 = Σ(领料数量 × FIFO单价)
│   └── 通过FIFO算法获取每批物料的实际成本
│
├── 人工成本 = Σ(各工序报工工时 × 小时工资率)
│   └── 工时来自process_reports表
│
└── 制造费用 = 总人工工时 × 制造费用分配率
    └── 可配置的分配率（默认：50元/工时）
```

### 6.2.2 成本自动核算触发点

| 业务动作 | 触发的财务操作 | 自动生成的单据 |
|---------|--------------|---------------|
| 销售订单确认 | 预计应收登记 | accounts_receivable (status=pending) |
| 发货出库 | 应收确认 | accounts_receivable (status=unpaid) |
| 采购入库 | 预计应付确认 | accounts_payable (status=unpaid) |
| 领料出库 | 材料成本归集 | material_batch_costs |
| 工序报工 | 人工成本计算 | process_reports (含工时统计) |
| 成品入库 | 工单成本汇总 | work_order_costs |
| 收款到账 | 应收核销 | accounts_receivable (status=paid) |
| 付款完成 | 应付核销 | accounts_payable (status=paid) |

```typescript
async function calculateWorkOrderCost(workOrderId: number) {
  const workOrder = await getWorkOrder(workOrderId);
  
  // 1. 计算材料成本（FIFO加权平均）
  const materialCost = await calculateMaterialCost(workOrderId);
  
  // 2. 计算人工成本
  const laborCost = await calculateLaborCost(workOrderId);
  
  // 3. 计算制造费用

  await upsertWorkOrderCost({
    wo_id: workOrderId,
    material_cost: materialCost,
    labor_cost: laborCost,
    manufacture_cost: manufactureCost,
    total_cost: totalCost,
    unit_cost: unitCost,
    status: 'calculated',
    calculated_at: new Date()
  });
  
  return { totalCost, unitCost };
}
```

## 7.2 权限控制粒度

| 层级 | 控制对象 | 实现方式 |
|------|---------|---------|
| **一级** | 菜单可见性 | 角色关联菜单，控制页面访问 |
| **二级** | 按钮权限 | 页面内按钮显隐（新增/编辑/删除/审批） |
| **三级** | 数据权限 | 数据范围控制（本人/部门/全部） |
| **四级** | 字段权限 | 特定字段的查看/编辑权限 |
