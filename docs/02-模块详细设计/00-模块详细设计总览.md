# 模块一：系统全局配置中心（优先级 P1 - 基石）

## 1.1 模块概述

**目标**：建立全系统唯一的参数配置入口，实现"**配置驱动、不写死代码、管理员可改、修改需审批**"的核心架构。

### 核心价值

```
传统模式：业务规则硬编码 → 改规则需重新开发部署 → 响应慢、风险高
    ↓
VNERP模式：业务规则配置化 → 管理员在线修改 → 即时生效、零代码变更
```

## 1.4 技术实现要点

### 1.4.1 配置缓存策略

```typescript
// 缓存设计：TTL = 5分钟
const CACHE_TTL = 5 * 60 * 1000; // 5分钟
let configCache: Map<string, any> = new Map();
let cacheTimestamp: number = 0;

export function getConfig(key: string): any {
  const now = Date.now();
  
  // 缓存未过期，直接返回
  if (now - cacheTimestamp < CACHE_TTL && configCache.has(key)) {
    return configCache.get(key);
  }
  
  // 缓存过期或不存在，从数据库加载
  const value = loadFromDatabase(key);
  configCache.set(key, value);
  cacheTimestamp = now;
  
  return value;
}
```

### 1.4.2 默认值兜底机制

```typescript
// 即使数据库无记录，也返回默认值，确保系统稳定运行
const DEFAULT_CONFIGS: Record<string, any> = {
  mould_life_days: 90,
  screen_life_days: 60,
  fifo_enabled: true,
  // ... 45+个默认值
};

function loadFromDatabase(key: string): any {
  const dbValue = queryDB(key);
  return dbValue ?? DEFAULT_CONFIGS[key];
}
```

# 模块二：仓库管理7步闭环（优先级 P4 - 核心）

> 📖 详细设计请参阅：[仓库管理核心原则专项说明](./26-仓库管理核心原则专项说明.md)

## 2.1 模块概述

**目标**：将20年实战仓库管理经验固化到系统中，实现**强制闭环、不可绕过**的标准化管理。

### 核心理念

```
传统仓库问题：
❌ 整料直接发料 → 浪费严重
❌ 随意选择批次 → FIFO无法执行
❌ 无单也能领料 → 库存混乱
❌ 盘点靠月底 → 问题发现太晚

VNERP解决方案：
✅ 强制拆小料 → 精细化管控
✅ 系统推荐批次 → FIFO严格执行
✅ 无单绝对不发 → 流程规范化
✅ 循环自动盘点 → 问题早发现早处理
```

## 2.2 七步闭环总览

```
┌─────────────────────────────────────────────────────────────┐
│                   仓库管理7步闭环                             │
├──────────┬──────────┬──────────┬───────────────────────────┤
│  第一步   │  第二步   │  第三步   │  第四步                    │
│          │          │          │                            │
│  验收入库  │  大料拆小料│  定置存放  │  账卡一致                  │
│  IQC检验  │  自动拆分  │  库位分区  │  三账同步                  │
├──────────┼──────────┼──────────┼───────────────────────────┤
│  第五步   │  第六步   │  第七步   │                            │
│          │          │          │                            │
│  先进先出  │  按单发料  │  循环盘点  │                            │
│  FIFO推荐  │  工单驱动  │  自动任务  │                            │
└──────────┴──────────┴──────────┴───────────────────────────┘
```

### 各步骤关键规则

| 步骤 | 名称 | 强制规则 | 违规后果 |
|------|------|---------|---------|
| 1 | 验收入库 | 必须IQC检验才能入库 | 无法创建入库单 |
| 2 | 大料拆小料 | 整料必须拆分后才能领用 | 整料库存锁定 |
| 3 | 定置存放 | 系统自动分配库位 | 人工无法指定错误库位 |
| 4 | 账卡一致 | 总账=批次账=货位卡 | 数据不一致报警 |
| 5 | 先进先出 | 系统按时间推荐批次 | 选择非FIFO批次需审批 |
| 6 | 按单发料 | 必须关联生产工单 | 无工单不能创建领料单 |
| 7 | 循环盘点 | A类每日/B类每周/C类每月 | 过期未盘自动提醒 |

# 模块三：生产流程标准化（优先级 P5）

> 📖 详细设计请参阅各子模块文档

## 3.1 模块概述

**目标**：建立以**工单驱动**的生产管理体系，实现打样→转产→工序控制→检验→入库的全流程标准化。

### 核心流程

```
销售订单(SO)
    ↓
创建打样工单(SAMPLE)
    ↓
完成打样 + FQC检验
    ↓
【合格】→ 转为正式生产工单(WO)
    ↓
关联BOM和标准卡(SCC/SCP/SCQ)
    ↓
按工序顺序报工（禁止跳工序）
    ↓
每道工序完成后可选IPQC检验
    ↓
最后一道工序完成 → 强制FQC检验
    ↓
【合格】→ 成品入库(FPR)
    ↓
关联发货单(SH) → 出库
```

### 关键控制点

| 控制点 | 规则 | 可配置项 |
|--------|------|---------|
| 打样转正式 | FQC合格才能转产 | - |
| 工序顺序 | 禁止跳工序报工 | `allow_skip_process` |
| IPQC触发 | 特定工序完成后自动创建 | 标准卡配置 |
| FQC强制 | 入库前必须检验 | `quality_check_mandatory` |
| 成本归集 | 报工时自动计算人工成本 | 工时费率表 |

# 模块四：设备寿命管理（优先级 P6）

## 4.1 模块概述

**目标**：实现刀模、网版、油墨的全生命周期管理，包括**自动预警、自动锁定、禁止超期使用**。

### 寿命参数一览

| 设备类型 | 有效期限 | 使用次数上限 | 预警提前期 | 报废判定 |
|---------|---------|-------------|-----------|---------|
| **刀模** | 90天 | 5000次 | 15天 | both(任一达标即报废) |
| **网版** | 60天 | 3000次 | 10天 | both |
| **油墨(未开盖)** | 168小时(7天) | - | 24小时 | expire(到期即报废) |
| **油墨(已开盖)** | 96小时(4天) | - | 12小时 | expire |
| **混合油墨** | 24小时 | - | 6小时 | expire |

### 自动化机制

```
设备启用 → 开始计时/计数
    ↓
[定时任务每天检查]
    ↓
距到期 ≤ 预警天数？
    ├── 是 → 发送预警通知（邮件/短信/站内信）
    │       └── 每日持续提醒直到替换或到期
    ↓
到期或次数超标？
    ├── 是 → 自动锁定(status=locked)
    │       └── 禁止分配给新工单
    │       └── 正在使用的工单可继续完成
    ↓
手动报废处理 → 创建报废单(BF)
    └── 记录报废原因、照片证据
```

# 模块五：质量管理（优先级 P5）

## 5.1 四级检验体系

| 检验类型 | 触发时机 | 检验对象 | 关联节点 |
|---------|---------|---------|---------|
| **IQC** | 原材料入库前 | 采购物料 | 仓库入库 |
| **IPQC** | 生产过程中 | 在制品 | 特定工序完成后 |
| **FQC** | 生产完工后 | 成品 | 成品入库前 |
| **OQC** | 出货前 | 待发货成品 | 发货出库前 |

### 不合格处理流程

```
检验结果为"不合格"
    ↓
生成不合格处理单(UNQ)
    ↓
选择处理方式：
├── 返工(REWORK) → 创建返工工单(REWO) → 重新走生产流程
├── 报废(SCRAP) → 扣减产量 → 记录损失成本
├── 特采(WAIVER) → 特殊放行 → 需高层审批
└── 退货(RETURN) → 退回供应商 → 生成退货单
```

## 6.1 模块概述

**目标**：实现业务财务一体化，**发货自动生成应收，领料自动归集成本，无需手工录入凭证**。

                    ↓
                应收账款台账
                    ↓
                账龄分析报表（30/60/90天）

生产领料(MR) ──→ 材料成本归集
工序报工 ──→ 人工成本归集
制造费用分摊 ──→ 制造费用归集
                    ↓
                工单成本核算(WO Cost)
                    ↓
                成本分析报表
