# 物料领用设计

## 1. 概述

物料领用是生产环节向仓库获取原材料的业务流程，承接生产工单（`prd_work_order`）与标准卡 BOM（`prd_process_card.materials`），按 FIFO 推荐批次并扣减库存。领用完成后归集到工单成本。本文档基于 `src/lib/material-requisition.ts` 现状编写。

## 3. 领用类型

| 类型 | 含义 | 触发场景 |
|---|---|---|
| `normal` | 正常领用 | 工单创建后按 BOM 标准数量自动生成 |
| `over` | 超额领用 | 实际消耗超出标准，需审批 |
| `supplementary` | 补充领用 | 生产中发现物料不足，临时补充 |

## 4. 核心服务 `MaterialRequisitionService`

文件：`src/lib/material-requisition.ts`

### 4.1 主要方法

| 方法 | 职责 |
|---|---|
| `autoGenerateRequisition(workOrderId)` | 根据工单 BOM 自动生成正常领用单 |
| `submitOverRequisition(workOrderId, items, reason)` | 提交超额领用申请 |
| `submitSupplementaryRequisition(workOrderId, items, reason)` | 提交补充领用申请 |
| `approveRequisition(id, approverId)` | 审批领用单 |
| `issueMaterial(id, allocations)` | 发料：按 FIFO 批次扣减库存 |
| `createReturn(id, items, reason)` | 创建退料单 |
| `confirmReturn(id, returnedItems)` | 确认退料入库 |

### 4.2 FIFO 批次推荐

发料时调用 `warehouse-core.ts` 的 `allocateFIFO` 算法：

```
推荐顺序：
1. split_flag = 2（余料）优先消耗
2. 按 expire_date 升序（近效期优先）
3. 按 inbound_date 升序（先入先出）
4. 跳过已锁定（stocktaking_flag=1）的批次
5. 跳过 quarantine_status = 1（隔离）的批次
```

返回推荐批次列表，仓管员可确认或调整（调整需记录 FIFO 覆盖日志）。

### 4.3 整料检查

当物料为卷材/片材时，需校验整料领用：

```
若需求量 = 整料库存量 → 直接领用整料
若需求量 < 整料库存量 → 提示需小料拆分
若需求量 > 整料库存量 → 多批次组合
```

小料拆分调用 `warehouse-core.ts` 的 `splitMaterial`，生成新的 `qrcode_record` 与 `material_splits` 记录。

## 5. 业务流程

```
工单创建
   ↓ autoGenerateRequisition
正常领用单 (type=normal, status=draft)
   ↓ submit (draft→pending)
审批 (pending→approved)
   ↓ issueMaterial 发料
   ↓ FIFO 推荐批次，扣减 inv_inventory_batch.available_qty
   ↓ 写入 inv_inventory_transaction 流水（type=outbound）
   ↓ 累加 reserved_qty（领用前预占）
   ↓ status=issued
生产过程中发现不足
   ↓ submitSupplementaryRequisition
   ↓ 审批 → 发料
生产超耗
   ↓ submitOverRequisition
   ↓ 审批（需说明超耗原因）→ 发料
生产剩余
   ↓ createReturn 退料
   ↓ confirmReturn 入库（inv_inventory.quantity 增加）
   ↓ 写入 inv_inventory_transaction（type=inbound）
```

## 6. 成本归集

领用发料后，成本自动归集到工单：

```sql
INSERT INTO work_order_costs (
  work_order_id, cost_type, amount, ref_no, ref_id, create_time
) VALUES (?, 'material', ?, ?, ?, NOW());
```

- `cost_type = material`：物料成本
- `amount`：按 FIFO 批次的 `unit_price * quantity` 计算
- `ref_no`：领用单号
- 工单成本汇总通过 `calculateWorkOrderCost(workOrderId)` 查询

## 9. 工程约定

1. **FIFO 强约束**：默认按 FIFO 推荐批次，覆盖需调用 `logFIFOOverride` 记录原因
2. **预占库存**：审批通过后预占 `reserved_qty`，发料时扣减 `available_qty`
3. **精度计算**：数量与金额使用 `Decimal.js`，避免浮点误差
4. **整料校验**：卷材类物料需校验整料规则，避免不合理拆分
5. **退料入库**：退料必须回到原仓库原批次（除非批次已过期）
6. **超额审批**：`over` 类型必须经过审批，记录超耗原因
