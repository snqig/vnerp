# 财务管理设计

## 1. 概述

财务管理模块采用"业务驱动财务"模式，所有应收/应付/成本单据均由业务事件自动生成，不支持手工录入。模块负责核算应收应付、记录收付款、归集工单成本。本文档基于 `src/lib/finance-core.ts`、`src/app/api/finance/cost/route.ts` 现状编写。

## 3. 核心服务 `FinanceCore`

文件：`src/lib/finance-core.ts`

### 3.1 应收管理

| 方法 | 职责 |
|---|---|
| `generateReceivable(shipmentId)` | 发货完成自动生成应收单 |
| `recordReceipt(receivableId, amount, method)` | 记录收款，更新已收金额与状态 |

应收状态流转：
- `pending`（待收款）→ `partial`（部分收款）→ `received`（已收款）
- `pending`/`partial` → `disputed`（争议）

### 3.2 应付管理

| 方法 | 职责 |
|---|---|
| `generatePayable(purchaseOrderId)` | 采购入库自动生成应付单 |
| `recordPayment(payableId, amount, method)` | 记录付款，更新已付金额与状态 |

应付状态流转：
- `pending` → `partial` → `paid`

### 3.3 工单成本核算

| 方法 | 职责 |
|---|---|
| `calculateWorkOrderCost(workOrderId)` | 汇总工单所有成本（material/labor/overhead/outsource） |
| `recordMaterialCost(workOrderId, requisitionId, amount)` | 物料领用归集成本 |
| `recordLaborCost(workOrderId, reportId, amount)` | 人工成本归集（来自生产报工） |
| `recordOverheadCost(workOrderId, amount, allocation)` | 制造费用分摊 |
| `recordOutsourceCost(workOrderId, outsourceId, amount)` | 外包成本归集 |

## 4. 业务驱动财务

### 4.1 应收生成

```
发货完成 (shipments.status = shipped)
   ↓ generateReceivable(shipmentId)
   ↓ 读取发货明细与产品单价
   ↓ 计算 amount = SUM(item.quantity * product.price)
   ↓ 写入 fin_receivable (status=pending)
```

### 4.2 应付生成

```
采购入库完成 (inv_inbound.status = completed)
   ↓ generatePayable(purchaseOrderId)
   ↓ 读取采购明细与采购单价
   ↓ 计算 amount = SUM(item.quantity * unit_price)
   ↓ 写入 fin_payable (status=pending)
   ↓ 同时写入 material_batch_costs（批次成本）
```

### 4.3 成本归集

```
物料领用发料 (requisition.status = issued)
   ↓ recordMaterialCost
   ↓ 按 FIFO 批次 unit_price * quantity
   ↓ 写入 work_order_costs (cost_type=material)

生产报工 (production_report)
   ↓ recordLaborCost
   ↓ 按 standard_time * hourly_rate
   ↓ 写入 work_order_costs (cost_type=labor)
```

## 6. 成本类型说明

| cost_type | 含义 | 来源 |
|---|---|---|
| `material` | 物料成本 | 物料领用（FIFO 批次成本） |
| `labor` | 人工成本 | 生产报工（工时 × 时薪） |
| `overhead` | 制造费用 | 分摊（电费/折旧等） |
| `outsource` | 外包成本 | 外协加工单 |

## 7. 工程约定

1. **业务驱动**：所有财务单据由业务事件触发，禁止手工录入应收/应付
2. **金额精度**：使用 `Decimal.js` 计算，存储保留两位小数
3. **状态一致性**：收款/付款后自动更新应收/应付状态（pending→partial→received/paid）
4. **批次成本**：采购入库时记录 `material_batch_costs`，发料时按 FIFO 取批次单价
5. **成本归集时机**：领用发料（material）、生产报工（labor）、外协确认（outsource）
6. **软删除**：`fin_cost_record` 使用 `deleted` 字段，支持撤销
