# 一、全局编码规则（所有单据统一格式）

## 1.1 编码设计原则

### 核心原则

```
✅ 唯一性：全系统范围内唯一，不重复
✅ 可读性：包含业务含义，便于人工识别
✅ 可扩展性：预留足够位数的流水号
✅ 可配置性：前缀和格式可通过系统配置修改
✅ 时间有序：按时间自然排序，便于查询
```

## 1.2 统一编码公式

```
基础数据编码：前缀 + 固定位数流水号
示例：CUST000001

业务单据编码：前缀 + 日期(YYYYMMDD) + 固定位数流水号
示例：WO202605110001
```

**关键参数**（可配置）：

| 参数 | 配置键 | 默认值 | 说明 |
|------|-------|-------|------|
| 日期格式 | `doc_date_format` | YYYYMMDD | 单据中的日期部分 |
| 流水号长度 | `serial_number_length` | 4 | 末尾流水号位数 |

## 1.11 二维码编码规范（全局唯一）

### 1.11.1 编码结构

```
总长度：19位字符
格式：VNR + 类型(1) + 时间戳(8) + 序号(7) + 拆分标识(1)

位置分解：
┌────┬────┬─────────┬──────────┬────┐
│ VNR│ T │YYYYMMDD │NNNNNNN   │ S  │
│ 3位│1位│  8位    │  7位     │1位 │
└────┴────┴─────────┴──────────┴────┘

字段说明：
├── VNR : 固定前缀（VNERP缩写）
├── T   : 物料类型标识（预留扩展）
├── YYYYMMDD : 生成日期
├── NNNNNNN : 当日序号（7位，每日从0000001开始）
└── S   : 拆分标识（0=整料, 1=小料, 2=余料）
```

### 1.11.2 拆分标识详解

| 标识值 | 含义 | 生成场景 | 使用限制 |
|--------|------|---------|---------|
| **0** | 整料 | 采购入库时生成 | 禁止直接领用，必须先拆分 |
| **1** | 小料 | 从整料拆分后生成 | 正常领用的最小单位 |
| **2** | 余料 | 拆分后的剩余部分 | 优先于小料领用 |

### 1.11.3 示例

```
整料二维码：VNR2026051100000010
├── 生成日期：2026-05-11
├── 当日序号：0000001
└── 类型：整料

拆分后的小料：VNR2026051100000011
├── 父级QR：VNR2026051100000010
├── 拆分时间：2026-05-11 08:30:00
└── 类型：小料（可正常领用）

剩余的余料：VNR2026051100000012
├── 父级QR：VNR2026051100000010
├── 数量：不足标准单位的部分
└── 类型：余料（优先消耗）
```

## 2.2 刀模寿命与更换规则（系统强制）

### 2.2.1 参数定义

| 参数名称 | 配置键 | 数据类型 | 默认值 | 单位 | 取值范围 |
|---------|-------|---------|-------|------|---------|
| 刀模有效天数 | `mould_life_days` | 数字 | 90 | 天 | 30-365 |
| 刀模最大使用次数 | `mould_max_times` | 数字 | 5000 | 次 | 100-50000 |
| 刀模预警天数 | `mould_warn_days` | 数字 | 15 | 天 | 1-30 |
| 刀模报废判定规则 | `mould_scrap_rule` | 下拉选择 | both | - | both/expire/times |

### 2.2.2 报废判定逻辑

```typescript
function shouldScrapMould(mould: MouldRecord): boolean {
  const rules = {
    validDays: getConfig('mould_life_days'),      // 90天
    maxTimes: getConfig('mould_max_times'),        // 5000次
    warnDays: getConfig('mould_warn_days'),        // 15天
    scrapRule: getConfig('mould_scrap_rule')       // 'both'
  };
  
  const daysUsed = daysBetween(mould.activate_date, new Date());
  const timesUsed = mould.usage_count;
  
  switch(rules.scrapRule) {
    case 'both':      // 到期 或 超次 都报废
      return daysUsed >= rules.validDays || timesUsed >= rules.maxTimes;
      
    case 'expire':   // 仅到期报废
      return daysUsed >= rules.validDays;
      
    case 'times':    // 仅超次报废
      return timesUsed >= rules.maxTimes;
      
    default:
      return false;
  }
}
```

### 2.2.3 强制更换规则

```
✅ 到期自动锁定：系统在到期日自动将刀模状态设为"locked"
✅ 次数到自动锁定：使用次数达到上限时立即锁定
✅ 磨损/变形强制报废：检验发现物理损伤时手动标记报废
✅ 必须重新开模：报废后禁止继续使用，必须开新模
✅ 必须重新绑定工单：新刀模需重新关联到生产工单
```

### 2.3.2 报废标准清单

```
以下情况之一出现即报废：
❌ 拉丝：印刷时出现线条拉丝现象
❌ 破损：网布有破洞或裂缝
❌ 变形：网框变形导致套印不准
❌ 感光胶脱落：感光胶层脱落导致图案不完整
❌ 精度超标：套印误差超过允许公差（默认±0.05mm）

报废后系统操作：
✅ 自动锁定：禁止继续分配给任何工单
✅ 记录原因：记录具体的报废原因和照片证据
✅ 关联影响：检查是否有在制工单受影响并预警
```

| 参数名称 | 配置键 | 数据类型 | 默认值 | 单位 | 说明 |

### 2.4.2 不同类型油墨的换算

```
基础保质期（小时）：
├── 溶剂型油墨(Solvent)：未开盖 168h / 已开盖 96h
├── UV油墨(UV)：未开盖 672h (168×4) / 已开盖 384h (96×4)
└── 水性油墨(Water)：未开盖 96h / 已开盖 48h

混合油墨规则：
⚠️ 调配后的混合油墨必须在24小时内用完
⚠️ 超过24小时自动冻结，禁止继续使用
⚠️ 冻结的油墨只能报废处理，不可解冻
```

### 2.5.2 保质期管理流程

```
采购入库 → 记录生产日期/批次号 → 计算到期日
    ↓
[距离到期≤30天?] ──→ 是 → 系统标黄预警
    ↓ 否

    ↓

                ↓
           禁止发料、禁止领用
           只能报废或退供应商
```

### 2.6.2 余料自动生成规则

```
当整料数量不是拆分单位的整数倍时：
├── 标准部分：按单位拆分为小料（status=available）
└── 剩余部分：自动生成为余料（status=remainder）

示例：105米的PET薄膜
├── 标准小料：10卷 × 10米 = 100米（可用）
└── 余料：1卷 × 5米 = 5米（优先消耗）

余料特点：
✅ 优先于小料领用（减少浪费）
✅ 不可再拆分
✅ 数量不足标准单位时不生成余料（直接报废或特殊处理）
```

## 2.7 仓库管理强制规则

### 2.7.1 FIFO控制

| 规则名称 | 配置键 | 默认值 | 说明 |
|---------|-------|-------|------|
| 是否强制先进先出 | `fifo_enabled` | true | 开启后系统严格按入库时间推荐批次 |

**FIFO执行机制**：
```
开启FIFO时：
✅ 系统自动按入库时间升序排列可用批次
✅ 用户无法自选批次（除非提交例外审批）
✅ 批次选择异常会触发审批流程

关闭FIFO时：
⚠️ 用户可自由选择批次（不推荐）
⚠️ 可能导致旧料积压、新料先用的问题
```

### 2.7.2 发料控制

| 规则名称 | 配置键 | 默认值 | 说明 |
|---------|-------|-------|------|
| 允许直接发整料 | `allow_whole_material_issue` | false | 整料是否可不拆分直接领用 |

**整料禁发机制**：
```
当 allow_whole_material_issue = false 时：
❌ 整料库存被锁定（status=locked_for_split）
❌ 领料时只显示已拆分的小料库存
❌ 必须先执行拆分操作才能领料

当 allow_whole_material_issue = true 时（特殊情况）：
⚠️ 可以直接领用整料（需要管理员审批）
⚠️ 仅适用于紧急补料等特殊场景
```

### 2.7.3 领料优先级

```
系统推荐的领料顺序（强制）：
1️⃣ 余料（remainder）- 最优先，减少浪费
2️⃣ 小料（small）- 其次，正常领用
3️⃣ 整料（whole）- 最后，需审批

实现方式：
SELECT * FROM inv_inventory_batch 
WHERE material_id = ? AND qty > 0
ORDER BY 
  CASE status
    WHEN 'remainder' THEN 1
    WHEN 'available' THEN 2
    ELSE 3
  END,
  inbound_time ASC;  -- 同级别按FIFO
```

### 2.7.4 审批控制

| 场景 | 配置键 | 默认值 | 审批人 |
|------|-------|-------|--------|
| 超额领料 | `allow_over_requisition` | false | 生产主管 |
| 补料申请 | （硬编码） | - | 品质+生产双审 |
| FIFO例外 | `fifo_enabled` | true时生效 | 仓库主管 |

## 2.8 循环盘点周期（系统自动生成任务）

### 2.8.1 ABC分类法

| 分类 | 定义 | 物料示例 | 盘点频率 | 配置建议 |
|------|------|---------|---------|---------|
| **A类** | 高价值、快周转 | 薄膜、油墨 | 每日 | critical |
| **B类** | 中等价值 | 溶剂、网布 | 每周 | normal |
| **C类** | 低价值、慢周转 | 辅料、耗材 | 每月 | low |

### 2.8.2 自动任务生成逻辑

```sql
-- 每日凌晨自动执行的存储过程
CREATE EVENT ev_generate_stocktaking_tasks
ON SCHEDULE EVERY 1 DAY
STARTS (TIMESTAMP(CURRENT_DATE) + INTERVAL 1 DAY)
DO
BEGIN
  -- A类物料：如果超过1天未盘点则生成任务
  INSERT INTO stocktaking_task (material_id, task_type, priority)
  SELECT m.id, 'cycle_count', 'high'
  FROM inv_material m
  WHERE m.abc_category = 'A'
    AND NOT EXISTS (
      SELECT 1 FROM stocktaking_task t 
      WHERE t.material_id = m.id 
        AND t.created_date > DATE_SUB(NOW(), INTERVAL 1 DAY)
        AND t.status IN ('pending', 'in_progress')
    );
    
  -- B类物料：如果超过7天未盘点则生成任务
  INSERT INTO stocktaking_task (material_id, task_type, priority)
  SELECT m.id, 'cycle_count', 'normal'
  FROM inv_material m
  WHERE m.abc_category = 'B'
    AND NOT EXISTS (
      SELECT 1 FROM stocktaking_task t 
      WHERE t.material_id = m.id 
        AND t.created_date > DATE_SUB(NOW(), INTERVAL 7 DAY)
        AND t.status IN ('pending', 'in_progress')
    );
    
  -- C类物料：如果超过30天未盘点则生成任务
  INSERT INTO stocktaking_task (material_id, task_type, priority)
  SELECT m.id, 'cycle_count', 'low'
  FROM inv_material m
  WHERE m.abc_category = 'C'
    AND NOT EXISTS (
      SELECT 1 FROM stocktaking_task t 
      WHERE t.material_id = m.id 
        AND t.created_date > DATE_SUB(NOW(), INTERVAL 30 DAY)
        AND t.status IN ('pending', 'in_progress')
    );
END;
```

## 5.2 错误处理规范

### 5.2.1 统一错误处理

```typescript
// 使用 withErrorHandler 包装API处理器
export const GET = withErrorHandler(async (request: NextRequest) => {
  try {
    // 业务逻辑
    const data = await fetchData();
    return successResponse(data);
  } catch (error) {
    // 已知业务错误
    if (error instanceof BusinessError) {
      return errorResponse(error.message, error.code);
    }
    // 未知错误（会被withErrorHandler捕获并记录日志）
    throw error;
  }
});
```
