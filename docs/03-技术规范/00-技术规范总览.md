## 阅读约定

- 所有规范文档对应代码以 `src/` 目录为准；冲突时以代码现状为准。
- 涉及命令的，统一使用 `pnpm <script>` 形式（`package.json` `preinstall` 经 `only-allow pnpm` 强制 pnpm@9.0.0）。
- 字段名/表名示例直接来自 `src/lib/db/schema.ts` 与各 API Route 中的 SQL；表结构以 `database/vnerpdacahng_schema.sql` 为权威 DDL。
- 命令脚本的权威来源为 `package.json#scripts`：
  - 全量初始化：`pnpm setup:db`（执行 `scripts/setup-db.mjs`，建库 + 导入 schema + 可选 seed）
  - 增量迁移：`pnpm migrate` / `migrate:down` / `migrate:status` / `migrate:create`（执行 `scripts/migrate.ts`，迁移文件位于 `database/migrations/`，编号 001–060）
  - 备份：`pnpm backup` / `backup:list` / `backup:restore`
  - `db:generate` / `db:push` / `db:migrate` 已**废弃**（运行仅打印警告），`db:studio` 仍可用
- 中间件入口为 `src/proxy.ts`（Next.js 16 自动探测，替代已删除的 `src/middleware.ts`），勿再引用 `middleware.ts`。
- 文档维护原则：代码变更后同步修订，避免规范与实现脱节；最新更新时间统一为 `2026-07-10`。
