现有项目分步改造落地方案
整体原则：最小侵入、存量兼容、分步落地、开关可控，不重构现有库存核心逻辑，以「扩字段 + 加表 + 扩展钩子 + 新增页面」的方式迭代，每一步都可独立测试、可回退，不影响现有业务正常使用。
一、改造前置：整体实施顺序
按「底层数据 → 独立新业务 → 存量流程扩展 → 报表前端 → 打印集成」的顺序开发，风险从低到高，前一步验收通过再进入下一步：
数据层改造（扩字段 + 建新表）→ 无业务影响
新增物料分切模块（纯新增功能）→ 不碰老流程
入库流程扩展（二维码生成）→ 后置钩子，可开关
出库流程扩展（FIFO 管控）→ 可开关，默认沿用原逻辑
库存日报表改造（视图切换 + 自定义表头）→ 老样式保留
标签打印集成 → 最后对接硬件
二、分阶段详细改造内容
阶段 1：数据层改造（基础打底，零业务影响）
改造范围
现有核心表扩字段 + 新增业务表，所有新增字段均设默认值，历史数据自动兼容。
1. 现有表扩展字段

2. 新增 4 张业务表
全部为独立新表，不与现有表强耦合：
分切单主表：分切单号、分切日期、母料批次 ID、出库总数量、总损耗、审核状态、操作人
分切单明细表：关联分切单号、子料批次 ID、子料宽幅、单份数量、分切份数、子料金额
标签模板配置表：模板类型（大料 / 小料）、尺寸、显示字段列表、二维码大小、样式参数
用户报表配置表：用户 ID、报表标识、勾选字段列表、勾选分类、日期范围，用于记忆自定义表头
阶段 2：新增「物料分切单」独立模块（纯新增，零侵入）
改造范围
新增独立菜单、独立页面、独立业务逻辑，完全不修改现有出入库页面，仅复用底层库存增减接口。
具体修改点
菜单入口：库存管理 → 物料分切单（新增）
核心页面功能：
新建分切单：扫码 / 输入母料批次号 → 自动带出母料规格、宽幅、可用库存
明细录入：填写分切份数、单份长度、损耗数量；宽幅字段锁定不可编辑，自动等于母料宽幅
提交前自动校验：母料出库量 = 子料总量 + 损耗量，不平衡拦截
审核按钮：审核后自动执行 3 件事
① 生成小料批次（batch_type=1，继承母料基础属性，规格 = 宽幅值）
② 调用现有库存接口，生成 3 条流水：
大料出库：出入库类型 = 模切，扣减母料库存
小料入库：出入库类型 = 其他，增加子料库存
损耗出库：出入库类型 = 品管，扣减损耗数量
③ 生成小料批次二维码 ID
追溯页面：批次详情页新增「分切追溯」标签页，母料查所有子料、子料查母料源头
兼容处理：现有库存流水接口、库存计算逻辑完全复用，仅新增 3 种业务类型的枚举值，不影响原有类型统计。
阶段 3：大料入库流程扩展（后置钩子，可开关）
改造范围
在现有采购入库 / 生产入库单的审核通过后追加逻辑，不修改原有审核流程。
具体修改点
后台加全局开关：「入库自动生成批次二维码」，默认关闭，开启后生效
入库单审核后置逻辑：
审核通过、生成批次后，自动生成雪花算法唯一 ID 写入qrcode_uid
自动从物料档案带入宽幅、长度到批次表
大料物料自动校验单位必须为「支 / 卷」，可配置是否强制拦截
前端页面扩展：
入库单详情页、批次列表页新增「打印标签」按钮（批量 + 单条）
批次列表新增「宽幅」「批次类型」筛选条件
兼容处理：开关关闭时，入库流程与改造前完全一致，无任何变化。
阶段 4：出库流程 FIFO 改造（扩展逻辑，可开关）
改造范围
现有出库单（领料 / 销售出库）的批次选择环节扩展，保留原有人工选择模式，新增自动匹配模式。
具体修改点
后台加配置项：
FIFO 管控等级：关闭 / 提示模式 / 强制模式
FIFO 匹配维度：默认按「物料编码 + 宽幅」匹配
前端出库单改造：
选择物料后，新增「自动匹配 FIFO 批次」按钮
点击后系统按「入库时间从早到晚」自动填充批次与数量，支持手动调整
提示模式：选非最早批次时弹窗提醒，可确认继续
强制模式：非最早批次置灰不可选，管理员可输入权限密码绕过
扫码出库改造：PDA 扫码时校验批次是否为推荐 FIFO 批次，不符合则提示拦截
兼容处理：配置为「关闭」时，出库单与原有逻辑完全一致，无任何新增按钮和校验。
阶段 5：库存日报表改造（老样式保留，新增能力）
改造范围
基于现有横版日报，新增视图切换 + 自定义表头能力，原有报表样式、数据完全保留。
具体修改点
视图切换：报表顶部新增切换标签：全部批次 / 仅大料 / 仅小料
大料视图：规格显示 宽幅*长度，单位显示支 / 卷
小料视图：规格仅显示宽幅，单位显示业务单位（m/㎡等）
全部视图：按批次类型各自显示对应格式
自定义表头面板：
新增「字段设置」按钮，弹出勾选面板：基础字段、出入库分类、日期范围
支持按天 / 区间选择，不必强制显示全月 31 天
自动保存用户上次勾选配置，下次进入默认加载
Excel 导出改造：
导出列与页面勾选完全一致，动态生成
保留多级表头合并（日期→分类→数量 / 金额）、冻结首行与首列
大小料规格格式与页面显示一致
追溯穿透：点击日报单元格数量，可弹窗查看对应批次明细、分切单、出入库单号
兼容处理：用户不做任何设置时，默认显示原有全量列、全月日期，与改造前完全一致。
阶段 6：二维码标签打印集成
改造范围
在已有入口接入打印能力，复用标签模板配置。
具体修改点
统一打印入口：入库单、分切单、批次列表、日报列表均接入同一打印组件
双模板独立配置：
大料标签：显示完整规格（宽幅 * 长度）、单位支 / 卷、批次号、二维码、供应商
小料标签：显示宽幅规格、单位、母料批次、二维码、分切日期
打印机适配：兼容热敏标签机与普通激光打印机，支持自定义纸张尺寸
三、历史数据兼容方案
历史批次：默认batch_type=0（大料），宽幅长度可从现有规格字段正则拆分批量填充，拆分失败则为 0，不影响显示
历史物料：单位保持原值，新增物料可强制选择支 / 卷；老物料可后台批量更新单位类型
历史报表：数据统计逻辑不变，仅新增格式渲染规则，历史数据查询不受任何影响
回退方案：所有新功能均可通过后台开关关闭，关闭后系统恢复至改造前状态
四、验收优先级建议
先验收分切单：独立模块，验证数量平衡、库存自动增减、母子批次关联
再验收入库二维码：验证自动生成、标签打印正常
再验收 FIFO 出库：验证自动匹配、提示 / 强制模式生效
最后验收报表：验证视图切换、自定义导出、数据穿透追溯
