> 文档编号：VNERP-BIZ-001 | 版本：V2.1 | 基于：代码现状 2026-07-10
> 关联代码：`src/app/api/production/{work-orders,work-order,work-report,material-issue,schedule,trace,process,product-label}/*`、`src/domain/production/`、`src/domain/sample/`、`src/domain/dcprint/`、`src/lib/bom-expansion.ts`、`src/lib/finance-core.ts`、`src/lib/mrp-engine*`、`src/lib/cost-engine.ts`、`src/lib/multi-color-printing.ts`

## 1. 工单状态机

工单状态为字符串枚举（`src/app/api/production/work-orders/route.ts`）：

```
pending → confirmed → producing → completed
              ↓           ↓
          cancelled    cancelled
```

| 状态 | 值 | 允许操作 |
|------|-----|---------|
| 待排产 | `pending` | 修改信息、删除、转已排产 |
| 已排产 | `confirmed` | 派工、转生产中、取消 |
| 生产中 | `producing` | 报工、转完工；**不可删除** |
| 已完工 | `completed` | 仅查询；不可改状态 |
| 已取消 | `cancelled` | 不可恢复 |

约束：
- 完工/取消状态工单不可再改 `status`。
- 删除生产中工单抛错 `生产中的工单不能删除，请先取消`（软删除 `deleted=1`）。

## 2. 工单创建（`POST /api/production/work-orders`）

事务内流程：
1. `SELECT ... FOR UPDATE` 锁定销售订单（`sal_order`），校验未取消（`status != '5'` 且 `status != 'cancelled'`）。
2. 校验同一销售订单不存在未取消工单（`COUNT(*) WHERE status != 'cancelled'`）。
3. `generateDocumentNo('work_order')` 生成工单号；`totalQuantity = Σ items.quantity`。
4. `INSERT INTO prod_work_order`：默认 `status='pending'`、`priority='normal'`。
5. 逐条 `INSERT INTO prod_work_order_item`（`line_no` 自增）。
6. 若传入 `bom_id`，从 `bom_line` 展开物料需求：
   ```
   required_qty = bom_line.quantity × totalQuantity × (1 + scrap_rate/100)
   ```
   写入 `prod_work_order_material_req`。
7. 更新销售订单 `status=3`（生产中）。
8. 生成工单二维码：`qr_code = 'WO-' + UUID前16位`，`INSERT INTO qrcode_record`（`qr_type='workorder'`）。
9. `logOperation` 审计（`oper_type='production'`）。

## 3. 工单更新与状态联动（`PUT`）

- 更新 `status`/`priority`/`plan_start_date`/`plan_end_date`。
- 工单完工（`status='completed'`）时，检查同销售订单是否还有未完工/未取消工单；若无，销售订单置 `status=4`（已完工）。
- 工单取消（`status='cancelled'`）时，若无其他活跃工单，销售订单回退 `status=2`（已确认）。
- 工单删除（`DELETE`）同理联动销售订单回退。

## 4. 销售订单状态映射

`sal_order.status` 为数字枚举：

| 数字 | 状态 |
|------|------|
| 1 | 已确认（默认创建） |
| 2 | 已确认（工单取消后回退） |
| 3 | 生产中（工单创建后） |
| 4 | 已完工（全部工单完工） |
| 5 | 已取消 |

## 5. 多色印刷工单

入口：`POST /api/production/work-order/create-multi-color`、`/color-seq`。

`src/lib/multi-color-printing.ts` 处理多色印刷工单：

- 色序管理：定义颜色顺序（如 CMYK + 专色），生成色序工序。
- 每色生成独立报工步骤，按色序累积良品数。
- 油墨消耗按色序与油墨配方计算（联动 `dcprint_ink_formula_version`）。

## 6. BOM 展开（`src/lib/bom-expansion.ts`）

支持多层 BOM 递归展开，核心能力：

- **递归展开**：半成品自动展开至原材料，`level` 字段标识层级。
- **损耗率累加**：`accumulatedLossRate = Σ 父级 lossRate`，`actualQuantity = baseQuantity × (1 + accumulatedLossRate/100)`。
- **循环引用检测**：发现自引用时记录 `circularReferenceWarnings`，不中断流程。
- **递归深度限制**：`maxDepth` 选项（默认无限制），超限截断。
- **物料合并**：相同 `materialId` 自动合并 `baseQuantity`。
- **缓存**：`enableCache` 选项，命中缓存时零查询；`clearBomExpansionCache()` 失效。
- **批量展开**：`expandBomBatch([{productId, quantity}])` 并行处理多产品。

## 7. MRP 物料需求计划

实现位于 `src/lib/mrp-engine.ts` 与 `src/lib/mrp-engine-v2.ts`：

- 输入：销售订单 / 工单 + BOM + 库存 + 在途采购 + 安全库存。
- 输出：物料净需求（净需求 = 毛需求 − 可用库存 − 在途数量 + 安全库存）。
- v2 支持多层 BOM 展开、在制工单扣减、损耗率。
- 路由：`/api/production/mrp`、`/api/production/schedule/auto`、`/api/production/schedule/capacity`。

## 8. 报工与标准卡

报工表 `prd_work_report`，关联 `work_order_id`，记录 `good_qty`（良品）、`defect_qty`（不良）、`actual_hours`、`hourly_rate`，并通过 053 迁移新增工具列（`tool_id`/`tool_usage_count` 等）。

| 报工类型 | 关联资源 |
|---------|---------|
| 印刷报工 | 标准卡油墨/网版明细 |
| 模切报工 | 刀具模板（`die_template_id`） |
| 装配报工 | - |

标准卡（`prd_standard_card` + 四张明细表 `prd_standard_card_ink`/`_screen`/`_die`/`_process`）记录丝网印刷工艺参数。每个产品可关联多版本标准卡，变更需新建版本，旧版标记为历史。`/api/standard-card` 提供 CRUD 与扫描校验（`scan`、`check-deviation`、`by-material`、`by-work-order`、`approve`）。

## 9. 生产排程

入口：`/api/production/schedule`（含 `auto`/`stats`/`capacity`）。

- `src/lib/production-scheduling.ts` 与 `production-scheduling-enhanced.ts` 提供排程算法。
- 输入：工单 + 工序路线 + 设备/班组产能。
- 输出：每日任务清单、产能负荷、瓶颈预警。
- 055 迁移为 `prd_schedule`/`prd_schedule_detail` 添加索引以加速排程查询。

## 10. 成本核算（`src/lib/finance-core.ts` `calculateWorkOrderCost` / `src/lib/cost-engine.ts`）

```
工单总成本 = 直接材料成本 + 直接人工成本 + 制造费用
单位成本 = 工单总成本 / 完工数量
```

| 成本项 | 数据来源 | 计算 |
|--------|---------|------|
| 直接材料 | `material_requisition_items.total_cost`（领料明细） | `SUM(total_cost) WHERE status=2` |
| 直接人工 | `prd_work_report` | `SUM(actual_hours × hourly_rate)` |
| 制造费用 | 简化计算 | `laborCost × 0.5`（**待优化**，见《已知问题》INV-002） |
| 工装摊销 | `dcprint_tool` | `unit_cost × 使用次数`（`Tool.recordUsage`） |
| 油墨成本 | `dcprint_ink_formula_version` | 配方版本激活后按配方明细归集 |

结果写入 `work_order_costs` 表（存在则 UPDATE，否则 INSERT），`status=1` 表示已核算。

`/api/finance/cost-variance` 提供成本差异分析，`/api/production/trace` 提供工单全链路追溯。

## 11. 印前工艺卡（dcprint 模块）

dcprint 模块（印前管理）覆盖工艺卡、工装、油墨配方三大核心：

### 11.1 工艺卡（`SampleProcessCard`）

`src/domain/dcprint/aggregates/SampleProcessCard.ts`：

- 状态机：`1(草稿) → 2(已提交) → 3(已确认) → 4(已取消)`。
- 包含物料明细（`SampleProcessItem`）与工序明细（`SampleProcessStep`）。
- 路由：`/api/dcprint/sample-card`（含 `submit`/`confirm`/`cancel`/`duplicate`/`save-as-template`/`upload-diagram`/`cost-variance`/`generate-quote`/`convert-work-order`）。
- 模板：`/api/dcprint/sample-card/template`、`/api/dcprint/process-cards`。
- 关联表：`sample_process_template`、`sample_process_template_item`、`sample_process_template_step`。

### 11.2 工装（`Tool`）

`src/domain/dcprint/aggregates/Tool.ts` + `src/domain/dcprint/value-objects/ToolStatus.ts`：

- 三类工装：刀模（体系B）、网版（体系C）、其他工装。
- 状态机：`1(待用) → 2(在用) → 3(维修中) / 4(预警) → 5(已报废)`。
  ```
  standby(1) → active(2) → maintenance(3) / warning(4) → scrapped(5)
  ```
- 寿命管理：`total_life`、`used_count`、`remain_life`、`warning_threshold`。
- 成本摊销：`unit_cost = original_cost / total_life`，每次使用 `recordUsage(useCount)` 摊销 `amortizedCost = unit_cost × useCount`。
- 维护：`startMaintenance` / `completeMaintenance(maintenanceCost, lifeAfter)` 重算 `net_value` 与 `unit_cost`。
- 报废：`scrap(reason, operatorId)` 不可逆。
- 关联表：`dcprint_tool`、`dcprint_tool_usage`、`dcprint_tool_maintenance`。
- 路由：`/api/dcprint/tool`（含 `[id]`/`usage`/`scrap`/`activate`/`maintenance/dashboard`）。

### 11.3 油墨配方（`InkFormulaVersion`）

`src/domain/dcprint/aggregates/InkFormulaVersion.ts` + `FormulaStatus` + `FormulaItemVO`：

- 配方版本管理：`draft → active → archived`，同一颜色仅一个 `active` 版本。
- 配方明细：`dcprint_ink_formula_item` 记录每种油墨色墨的占比、用量、单价。
- 成本计算：`FormulaCostService` 计算配方总成本，`FormulaCompareService` 比较版本差异。
- 路由：`/api/dcprint/formula/version`（含 `[id]`/`activate`/`recalculate-cost`/`preview-cost`/`compare`）、`/api/dcprint/formula/color`、`/api/dcprint/ink-formula`。
- 油墨生命周期：`ink-opening`（开罐）→ `ink-mixing`（调配）→ `ink-dispatch`（发料）→ `ink-surplus`（余料）→ `ink-usage`（消耗）→ `ink-query`。
- 关联表：`dcprint_ink_color`、`dcprint_ink_formula_version`、`dcprint_ink_formula_item`。

## 12. 打样订单（sample 模块）

`src/domain/sample/aggregates/SampleOrder.ts` + `SampleOrderStatus`：

- 状态机：
  ```
  draft → pending → in_progress → completed → confirmed → converted
                                                                ↓
                                                            cancelled
  ```
  - `draft`：草稿
  - `pending`：待打样（提交后）
  - `in_progress`：打样中
  - `completed`：已完成
  - `confirmed`：已确认（客户确认合格）
  - `converted`：已转大货（终态）
  - `cancelled`：已作废（终态）
- 关联字段（053 迁移）：`processCardId`、`workOrderId`、`salesOrderId`、`sampleFee`、`feeCharged`、`feeDeductible`、`feeDeducted`、`sampleVersion`、`parentVersionId`、`convertedAt`、`convertedBy`。
- 打样费：可全额收取、可抵扣大货订单金额（`feeDeducted` 累计 ≤ `feeDeductible`）。
- 事件：`SampleOrderCreatedEvent`、`SampleOrderSubmittedEvent`、`SampleOrderStartedEvent`、`SampleOrderCompletedEvent`、`SampleOrderConfirmedEvent`、`SampleOrderConvertedEvent`、`SampleOrderCancelledEvent`。
- 处理器：`SampleOrderConversionHandler`（转大货时联动销售订单）、`SampleOrderInventoryHandler`（打样领料出库）。
- 路由：`/api/sample/orders`（含 `linkage`/`status`）、`/api/sample/feedback`、`/api/sample/inventory`。
- 单号生成：`SP{YYYYMMDD}{4位序号}`。
