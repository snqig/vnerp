## 2. 来料检验（IQC）

### 2.1 单号规则

`IQC{YYYYMMDD}{3位序号}`，序号取当日同前缀最大值 +1（`MAX(inspection_no) LIKE 'IQC{date}%'`）。

### 2.2 必填字段

`inspectionDate`、`supplierName`、`materialCode`、`materialName`、`specification`、`batchNo`、`quantity`、`unit`、`inspectionType`、`inspectionResult`、`inspectorName`。

### 2.3 检验结果与库存联动（关键）

`inspectionResult` 支持中英双语值：

| 检验结果 | 库存批次处理 | 入库单处理 |
|---------|------------|-----------|
| `qualified` / `合格` | `inv_inventory_batch.status='normal'`，`freeze_reason=NULL` | `inv_inbound_order.inspection_status=1` |
| `unqualified` / `不合格` | `inv_inventory_batch.status='frozen'`，`freeze_reason='进料检验不合格'` | `inv_inbound_order.inspection_status=2` |

联动通过 `UPDATE ... WHERE batch_no=?`（按批次号）+ 子查询关联 `qrcode_record.ref_no` 更新入库单。写入失败 `.catch(() => {})` 静默吞错（**待优化**，应记录告警）。

### 2.4 明细批量插入

`INSERT INTO qc_incoming_inspection_item ... VALUES ?` 使用 mysql2 批量占位符，字段：`item_name`、`standard`、`actual_value`、`result`、`remark`。

### 2.5 软删除

事务内同时更新主表与明细表 `deleted=1`。

## 3. SPC 统计过程控制（`/api/quality/spc`）

入口：`GET ?action=xbar-r|pareto|p-chart`，实现位于 `src/lib/spc-analysis.ts`。

### 3.1 X-bar R 控制图（计量值）

- 参数：`material_id`、`inspection_type`（默认 `PQC`）、`start_date`（默认 30 天前）、`end_date`、`subgroup_size`（默认 5）。
- `getSPCDataFromDB(conn, materialId, inspectionType, startDate, endDate, subgroupSize)` 分组取数。
- `calculateXbarRChart(spcData)` 计算均值、极差、控制限。

### 3.2 帕累托图（缺陷分布）

- 数据源：`qc_unqualified` 按 `defect_type` 聚合 `COUNT(*)`，降序排列。
- `calculatePareto(defects)` 计算累计占比与 80% 截断线。

### 3.3 P 控制图（不良率）

- 数据源：`qc_inspection` 按 `DATE(inspection_date)` 聚合，`inspection_result=2` 计为不良。
- `calculatePChart(data)` 计算平均不良率与控制限。

### 3.4 控制限公式

```
UCL = X̄ + 3σ    CL = X̄    LCL = X̄ - 3σ
```

超出控制限自动触发异常预警（业务侧需订阅）。

## 4. 不合格品处理

表 `qc_unqualified`，记录 `defect_type`（缺陷类型）、关联物料批次、采购订单、供应商、不良现象与原因。

| 处理方式 | 审批级别 |
|---------|---------|
| 退货 | 品质主管 |
| 特采（让步接收） | 生产经理 |
| 返工 | 品质主管 |
| 报废 | 总经理 |

## 5. SGS 第三方检测（`/api/quality/sgs`）

- 独立路由 + `[id]` 子路由 + `/expiry-warning` 临近有效期预警。
- 记录第三方检测报告编号、检测项目、有效期，过期自动告警。

## 6. 单据作废

检验单暂未纳入 `soft-delete.ts` 的 `CANCELLABLE_DOCUMENTS`，仅支持软删除（`deleted=1`）。如需正式作废流程，需补充配置。
