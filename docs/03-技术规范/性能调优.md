# 性能调优

> 基于 `next.config.ts`、`src/lib/db/index.ts`、`src/lib/api-cache.ts`、`src/lib/performance/` 描述性能优化配置与监控指标。

## 1. Next.js 性能配置

`next.config.ts` 关键项：

| 配置 | 值 | 说明 |
|------|-----|------|
| `output` | `standalone` | 生产构建为独立部署包，包含最小化 node_modules |
| `reactStrictMode` | `false` | 当前关闭，避免双渲染副作用（开发期可临时开启） |
| `devIndicators` | `false` | 关闭开发期路由指示器，减少干扰 |
| `poweredByHeader` | `false` | 移除 `X-Powered-By` 头，安全 + 减少指纹 |
| `compress` | `true` | 启用 gzip 压缩 |

### 1.1 图片优化

```ts
images: {
  remotePatterns: [{ protocol: 'https', hostname: 'lf-coze-web-cdn.coze.cn', pathname: '/**' }],
  formats: ['image/avif', 'image/webp'],
  minimumCacheTTL: 3600,
}
```

- 优先使用 `next/image` 自动转 WebP/AVIF 与按需缩放。
- 远程图床需在 `remotePatterns` 显式白名单。

### 1.2 包体积优化

```ts
experimental: {
  optimizePackageImports: ['lucide-react', '@radix-ui/react-icons', 'date-fns', 'lodash'],
}
```

按需引入大型库，避免 barrel file 导致首屏 JS 膨胀。

### 1.3 静态资源缓存

`headers()` 中：

```
/:path*.(js|css|woff2|woff|ttf|ico|png|jpg|svg) →
  Cache-Control: public, max-age=31536000, immutable
```

带 hash 的静态资源可一年强缓存。

## 2. next-intl 国际化

`src/i18n/request.ts` 通过 `createNextIntlPlugin('./src/i18n/request.ts')` 注入。locale 配置位于 `src/i18n/locales.ts`：

- `localePrefix: 'as-needed'`：默认 locale 不带前缀，减少 URL 长度。
- 翻译资源按 namespace 拆分，按需加载。

## 3. 数据库连接池

`src/lib/db/index.ts`：

```ts
{
  connectionLimit: 20,        // 上限按服务器 CPU/内存调整
  maxIdle: 10,
  idleTimeout: 30000,
  queueLimit: 100,
  enableKeepAlive: true,
  keepAliveInitialDelay: 10000,
  connectTimeout: 8000,
}
```

- 使用 `globalThis` 单例池，避免 Next.js 热重载创建多个连接。
- `query` 在 `ER_CON_COUNT_ERROR` / `PROTOCOL_CONNECTION_LOST` 时自动重试 2 次。
- `transactionWithRetry` 处理乐观锁冲突（指数退避 100ms → 1s）。
- 大批量写入使用 batch insert；查询只 SELECT 必要字段，避免 `SELECT *`。

## 4. API 缓存

`src/lib/api-cache.ts` 提供：

- `cachedApiRoute`：包装 Route Handler，按 URL + 参数生成 cache key。
- `invalidateCache`：写操作后主动失效相关缓存。
- 多实例部署需替换为 Redis 后端（当前为进程内）。

## 5. 应用层优化

- 列表查询默认分页 20 条；大表必须 `LIMIT + OFFSET`。
- 复杂统计查询考虑物化视图或定时预聚合。
- 前端列表大数据量使用虚拟滚动；`useMemo` / `useCallback` / `React.memo` 避免不必要重渲染。
- 大型组件使用 `dynamic(() => import(...), { ssr: false })` 按需加载。
- `Suspense` + `loading.tsx` 实现流式渲染。

## 6. 监控指标

| 指标 | 阈值 | 监控方式 |
|------|------|----------|
| API 响应时间 | < 500ms (P95) | `/api/health` 上报 / APM |
| 数据库查询时间 | < 200ms (P95) | 慢查询日志 |
| 堆内存使用率 | < 85% | `/api/health` `checks.memory` |
| 数据库连接数 | < connectionLimit | MySQL `SHOW STATUS LIKE 'Threads_connected'` |
| 进程运行时间 | 持续 | `/api/health` `checks.uptime` |

`/api/health` 返回结构：

```json
{
  "status": "healthy" | "degraded",
  "timestamp": "...",
  "version": "...",
  "environment": "...",
  "latency": 12,
  "checks": { "database": {...}, "memory": {...}, "uptime": {...} }
}
```

整体健康返回 200，降级返回 503，供负载均衡探针使用。

## 7. 慢查询排查

```sql
-- 查看慢查询
SELECT * FROM mysql.slow_log ORDER BY start_time DESC LIMIT 20;

-- 查看执行计划
EXPLAIN SELECT * FROM inv_inventory_batch WHERE material_id = ? AND status = 'normal';

-- 添加缺失索引
ALTER TABLE inv_inventory_batch ADD INDEX idx_material_warehouse (material_id, warehouse_id, status);
```

索引原则：

- WHERE / ORDER BY / JOIN 高频字段建索引。
- 联合索引遵循最左前缀。
- 区分度低的字段（如 `deleted`）不单独建索引。
- 单表索引数量不超过 5 个。

## 8. 性能优化检查清单

- [ ] 列表 API 强制分页，单页 ≤ 100 条。
- [ ] 数据库查询显式指定字段，避免 `SELECT *`。
- [ ] 高频查询字段已建索引。
- [ ] 大型库使用 `optimizePackageImports`。
- [ ] 图片使用 `next/image`。
- [ ] 写操作使用事务，但事务粒度尽量小。
- [ ] 缓存对热数据生效，写操作正确失效缓存。
