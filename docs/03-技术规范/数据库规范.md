> 基于 `drizzle.config.ts`、`src/lib/db/schema.ts`、`src/lib/db/index.ts` 描述 ORM 用法、命名约定与迁移流程。

## 2. 连接与查询

`src/lib/db/index.ts` 基于 `mysql2` 连接池，使用 `globalThis` 单例防止热重载时重复创建。

连接池关键参数：

```ts
{
  connectionLimit: 20,
  maxIdle: 10,
  idleTimeout: 30000,
  queueLimit: 100,
  enableKeepAlive: true,
  keepAliveInitialDelay: 10000,
  connectTimeout: 8000,
}
```

提供的查询函数：

| 函数 | 用途 |
|------|------|
| `query<T>(sql, values)` | SELECT，返回 `T[]`；自动重试连接错误 2 次 |
| `queryOne<T>(sql, values)` | 返回首行或 `null` |
| `execute(sql, values)` | INSERT/UPDATE/DELETE，返回 `ResultSetHeader` |
| `transaction(cb)` | 事务，回滚 + 连接自动释放 |
| `transactionWithRetry(cb, maxRetries=3)` | 乐观锁冲突自动重试（指数退避） |
| `queryPaginated(...)` | 分页查询（兼容新旧两种签名） |
| `drizzleDb` | Drizzle ORM 实例（`drizzle({ client, schema, mode })`） |

**新代码优先使用 Drizzle 查询构建器（`drizzleDb`），旧代码继续使用 `query`/`execute` 保持兼容**。所有 SQL 必须参数化占位符 `?`，禁止字符串拼接。

## 3. Schema 定义

`src/lib/db/schema.ts` 使用 `mysqlTable` 定义表，按模块分组：

```ts
export const customers = mysqlTable('customers', {
  id: serial('id').primaryKey(),
  code: varchar('code', { length: 50 }).notNull().unique(),
  name: varchar('name', { length: 200 }).notNull(),
  creditLimit: decimal('credit_limit', { precision: 15, scale: 2 }).default('0'),
  status: varchar('status', { length: 20 }).default('active'),
  createdAt: datetime('created_at').default('CURRENT_TIMESTAMP'),
  updatedAt: datetime('updated_at').default('CURRENT_TIMESTAMP'),
});
```

类型映射要点：

- 主键：`serial('id').primaryKey()`（自增 BIGINT）。
- 金额：`decimal('xxx_amount', { precision: 15, scale: 2 })`；高精度用量用 `scale: 4/6`。
- 时间：`datetime('xxx_time').default('CURRENT_TIMESTAMP')`；更新时间需 `onUpdate` 在 SQL 层定义。
- 外键：`int('customer_id').references(() => customers.id)`。
- 枚举字符串：`varchar('status', { length: 20 })` + 业务层枚举常量。
- JSON 字段：`json('inspection_items')`。

## 4. 命名约定

### 4.1 表名

```
{模块前缀}_{业务实体}
```

| 前缀 | 模块 | 示例 |
|------|------|------|
| `sys_` | 系统 | `sys_user`、`sys_role`、`sys_menu`、`sys_operation_log` |
| `crm_` | 客户 | `crm_customer` |
| `inv_` | 仓库 / 库存 | `inv_warehouse`、`inv_inventory_batch`、`inv_inventory_transaction` |
| `prd_` | 生产 | `prd_work_order`、`prd_work_report`、`prd_standard_card` |
| `pur_` | 采购 | `pur_purchase_order`、`pur_request` |
| `sal_` | 销售 | `sal_order`、`sal_delivery` |
| `fin_` | 财务 | `fin_receivable`、`fin_payable`、`fin_cost_record`、`fin_voucher` |
| `qms_` | 质量 | `qms_sgs_cert`、`qms_inspection` |
| `hr_` | 人力 | `hr_employee`、`hr_attendance` |
| `qrcode_` | 二维码 | `qrcode_record` |

> ⚠️ 当前 `src/lib/db/schema.ts` 中部分表（如 `customers`、`products`、`inventory_batches`、`work_orders`、`purchase_orders`）未携带模块前缀，与数据库实际表名不一致；API Route 中 SQL 使用带前缀的真实表名。详见 [已知问题](./已知问题.md)。

### 4.2 字段命名

- 使用 snake_case：`material_id`、`create_time`。
- 主键：`id`；外键：`{entity}_id`；编号：`{entity}_no`。
- 时间字段：`xxx_time`（DATETIME）；日期字段：`xxx_date`（DATE）。
- 金额：`xxx_amount`；数量：`xxx_qty` / `xxx_quantity`；单价：`unit_price`。
- 布尔 / 标志：`is_xxx`；状态：`status`；类型：`xxx_type`。
- 软删除：`deleted TINYINT DEFAULT 0`；乐观锁：`version INT`。

### 4.3 标准字段

每张业务表必须包含：

```sql
id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
deleted     TINYINT DEFAULT 0
```

涉及操作人的额外加 `create_by` / `update_by`。

### 4.4 索引命名

```
PRIMARY KEY (id)
uk_{表}_{字段}        -- 唯一索引
idx_{表}_{字段}       -- 普通索引
idx_{表}_{f1}_{f2}    -- 联合索引
```

## 5. 迁移流程

```bash
pnpm db:generate   # 对比 schema.ts 与数据库，生成 SQL 迁移到 ./drizzle
pnpm db:push       # 直接将 schema 推送到数据库（开发环境快速同步）
pnpm db:migrate    # 应用 ./drizzle 下的迁移（生产推荐）
pnpm db:studio     # 启动 Drizzle Studio 可视化
pnpm setup:db      # 一键建表 + 种子（开发环境）
```

迁移文件位于 `drizzle/`，meta 记录在 `drizzle/meta/_journal.json`。当前已有：

- `0000_chilly_retro_girl.sql`
- `0001_ordinary_valkyrie.sql`

迁移规范：

- 每个迁移必须可独立执行，含 `IF NOT EXISTS` / `IF EXISTS`。
- 破坏性操作（DROP TABLE / DROP COLUMN）必须注释，确认后手动执行。
- 迁移一旦提交不可修改，只能新增回滚迁移。
- 业务表外的辅助 SQL（种子、菜单、权限）放在 `database/seeds/` 与各业务 SQL 文件。

## 6. 事务使用规范

必须使用事务的场景：

- 多表写入（如入库同时写 `inv_inventory_batch` + `inv_inventory_transaction` + `inv_inventory_log`）。
- 金额 / 库存数量变更。
- 状态流转 + 关联流水生成。
- 二维码记录创建 / 更新。

模板：

```ts
await transaction(async (conn) => {
  await conn.execute('INSERT INTO ... VALUES (?, ?)', [...]);
  await conn.execute('UPDATE ... SET available_qty = ? WHERE id = ? AND version = ?', [...]);
  // 任何抛错会自动 ROLLBACK
});
```

事务中禁止 RPC 调用或耗时操作；连接必须在 `finally` 释放（`transaction` 已自动处理）。

## 7. 禁止事项

- 禁止 `SELECT *`，必须显式列出字段。
- 禁止物理删除业务单据，统一 `UPDATE ... SET deleted = 1`。
- 禁止在数据库存储大对象（图片、文件），只存路径。
- 禁止使用外键约束（在应用层保证一致性）。
- 禁止使用存储过程 / 触发器，业务逻辑在应用层。
- 禁止在字段名使用数据库保留字。
- 禁止字符串拼接 SQL（必须参数化）。
