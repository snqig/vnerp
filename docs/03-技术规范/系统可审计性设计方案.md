# 一、什么是系统可审计性？

## 1.1 审计定义

审计 = **谁**、在**什么时间**、用**什么 IP**、做了**什么操作**、操作**前后数据是什么**、是否**成功**。

## 1.2 ERP 必须满足的五大审计要求

| 要求 | 说明 | 实现手段 |
|------|------|---------|
| **操作可追溯** | 每个操作都能追溯到具体人员 | 全流程操作日志 |
| **数据可对账** | 数据变更前后可对比 | 关键数据变更日志 |
| **责任可认定** | 明确谁做了什么 | 审核流程 + 操作人记录 |
| **篡改可发现** | 数据被修改能被发现 | 不可修改的流水记录 |
| **错误可回查** | 错误操作可追溯 | 完整的操作链路 |

## 1.3 审计核心原则

> **所有操作留痕，所有修改留底，所有单据不可删，所有变动有据可查。**

# 二、保证可审计性的 8 大核心手段

## 2.1 全流程操作日志（必须）

### 记录范围

所有 **增、删、改、审核、反审、导入、导出、打印** 都必须记录。

### 日志必须包含的字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键 |
| `module` | VARCHAR(64) | 操作模块（采购/销售/库存/财务） |
| `type` | VARCHAR(32) | 操作类型（新增/修改/删除/审核） |
| `username` | VARCHAR(64) | 操作人 |
| `content` | VARCHAR(1000) | 操作内容摘要 |
| `before_data` | TEXT | 修改前数据（JSON） |
| `after_data` | TEXT | 修改后数据（JSON） |
| `ip` | VARCHAR(50) | 客户端 IP |
| `user_agent` | VARCHAR(500) | 浏览器/设备信息 |
| `status` | TINYINT | 1=成功 0=失败 |
| `error_msg` | VARCHAR(500) | 失败原因 |
| `duration_ms` | INT | 执行耗时（毫秒） |
| `create_time` | DATETIME | 操作时间 |

### 实现方式

```typescript
// 方式1: AOP 自动拦截（推荐）
// 方式2: 手动调用（特殊场景）
await AuditLogger.log({
  module: 'production',
  type: 'update',
  content: '修改工单状态: pending → producing',
  beforeData: { status: 'pending' },
  afterData: { status: 'producing' },
});
```

## 2.2 关键数据变更日志（核心！）

### 必须记录的变更类型

| 数据类型 | 变更示例 | 审计要求 |
|---------|---------|---------|
| **金额** | 采购单金额从 1000 → 1200 | 必须记录前后值 |
| **数量** | 库存从 50 → 出库变为 30 | 必须记录前后值 |
| **价格** | 单价从 10.5 → 11.0 | 必须记录前后值 |
| **状态** | 订单状态从草稿 → 已审核 | 必须记录状态流转 |
| **库存** | 任意库存变动 | 必须记录变动前后 |

### 变更日志示例

## 2.3 单据不可删除、只能作废（企业铁律）

### ❌ 绝对禁止

- 删除已保存单据
- 删除已审核单据
- 删除出入库单据
- 删除财务应收应付

### ✅ 正确做法

| 单据状态 | 允许操作 | 实现方式 |
|---------|---------|---------|
| 草稿 | 可删除 | 物理删除或逻辑删除 |
| 已审核 | 只能作废 | 更新 status = 'cancelled' |
| 已出入库 | 只能作废 | 更新 status = 'cancelled' + 反向操作 |
| 已生成凭证 | 只能红冲 | 生成反向凭证 |

### 作废必须留记录

```sql
-- 作废记录表
CREATE TABLE cancel_record (

  source_type VARCHAR(64) COMMENT '来源类型',
  source_id BIGINT COMMENT '来源单据ID',

  cancel_reason VARCHAR(500) COMMENT '作废原因',
  cancel_by VARCHAR(64) COMMENT '作废人',
  cancel_time DATETIME COMMENT '作废时间',
  original_data TEXT COMMENT '原单据数据（JSON）'
);
```

## 2.4 所有业务必须走审核流程（无审核不生效）

### 审核流程设计

```
草稿 → 提交审核 → 审核中 → 审核通过/驳回 → 生效
  ↑___________________________________________|
```

### 必须审核的业务

| 业务模块 | 审核节点 | 不审核的后果 |
|---------|---------|-------------|
| 采购订单 | 采购审核 | 无法入库 |
| 销售订单 | 销售审核 | 无法出库 |
| 出入库单 | 仓库审核 | 不扣减/增加库存 |
| 生产工单 | 生产审核 | 无法领料 |
| 应收应付 | 财务审核 | 不生成凭证 |

### 审核 = 确认责任 = 审计节点

```typescript
interface ApprovalRecord {

  source_type: string;
  source_id: number;
  approver_id: number;
  approver_name: string;
  approval_action: 'approve' | 'reject' | 'return';
  approval_comment: string;
  approval_time: string;
  next_approver_id?: number;
}
```

## 2.5 库存流水不可修改、不可删除（灵魂）

### 每一次库存变动必须生成流水

- 采购入库
- 销售出库
- 生产领料
- 生产入库
- 盘点调整
- 调拨

### 流水特点

| 特性 | 说明 | 实现 |
|------|------|------|
| **不可修改** | 一旦生成，永久不可修改 | 无 UPDATE 接口 |
| **不可删除** | 一旦生成，永久不可删除 | 无 DELETE 接口 |
| **来源可追溯** | 知道是哪张单据产生的 | source_type + source_id |
| **库存前后值可查** | 变动前后的库存数量 | stock_before + stock_after |

### 库存可审计 = ERP 可审计


## 2.8 财务数据自动留痕（应收/应付/收支）

### 财务操作要求

| 要求 | 说明 | 实现 |
|------|------|------|
| **自动生成单据** | 所有财务操作必须基于业务单据 | 出库自动应收、入库自动应付 |
| **自动留痕** | 财务变动自动记录 | fin_voucher + fin_voucher_line |
| **禁止直接改库** | 不允许手动直接修改数据库 | 只能通过API操作 |
| **禁止无单据调整** | 不允许无单据的财务调整 | 必须有 source_type + source_id |

# 四、后端代码实现方案

## 4.1 AOP 自动日志中间件（Next.js 实现）

```typescript
// src/lib/audit-logger.ts
import { query } from '@/lib/db';
import { secureLog } from '@/lib/logger';

export interface AuditLogEntry {
  module: string;
  type: string;
  title?: string;
  content?: string;
  beforeData?: Record<string, any>;
  afterData?: Record<string, any>;
  requestUrl?: string;
  requestMethod?: string;
  requestParam?: any;
  responseResult?: any;
  status?: number;
  errorMsg?: string;
  durationMs?: number;
}

export class AuditLogger {
  static async log(entry: AuditLogEntry): Promise<void> {
    try {
      const user = await this.getCurrentUser();
      const ip = await this.getClientIp();
      const userAgent = await this.getUserAgent();

      await query(
        `INSERT INTO sys_operate_log (
          module, type, title, username, user_id, content,
          before_data, after_data, ip, user_agent,
          request_url, request_method, request_param, response_result,
          status, error_msg, duration_ms
        ) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)`,
        [
          entry.module,
          entry.type,
          entry.title || '',
          user?.username || 'system',
          user?.id || 0,
          entry.content || '',
          entry.beforeData ? JSON.stringify(entry.beforeData) : null,
```

## 4.2 数据变更前后对比工具

```typescript
// src/lib/data-diff.ts

export interface FieldDiff {
  field: string;
  oldValue: any;
  newValue: any;
  changed: boolean;
}

export function diffData<T extends Record<string, any>>(
  oldData: T,
  newData: T
): FieldDiff[] {
  const diffs: FieldDiff[] = [];
  const allKeys = new Set([...Object.keys(oldData), ...Object.keys(newData)]);

  for (const key of allKeys) {
    const oldValue = oldData[key];
    const newValue = newData[key];

    if (JSON.stringify(oldValue) !== JSON.stringify(newValue)) {
      diffs.push({
        field: key,
        oldValue,
        newValue,
        changed: true,
      });
    }
  }

  return diffs;
}

export function formatDiffForAudit(diffs: FieldDiff[]): string {
  return diffs
    .map((d) => `${d.field}: ${JSON.stringify(d.oldValue)} → ${JSON.stringify(d.newValue)}`)
    .join('; ');
}

// 使用示例
export async function updateWithAudit<T extends Record<string, any>>(
  table: string,
  id: number,
  oldData: T,
  newData: T,
  options: {
    module: string;
    title: string;
  }
): Promise<void> {
  const diffs = diffData(oldData, newData);

  if (diffs.length === 0) return;

  // 1. 执行更新
  // await db.update(table, id, newData);

  // 2. 记录审计日志
  await AuditLogger.log({
    module: options.module,
    type: 'update',
    title: options.title,
    content: `修改${table}记录(ID:${id}): ${formatDiffForAudit(diffs)}`,
    beforeData: oldData,
    afterData: newData,
  });
}
```

```typescript
export async function softDelete(
  table: string,
  id: number,
  options: {
    module: string;
    reason?: string;
    operator?: string;
  }
): Promise<void> {
  // 1. 查询原数据
  const [oldData]: any = await query(`SELECT * FROM ${table} WHERE id = ?`, [id]);

    throw new Error('记录不存在');
  }

  // 2. 检查状态，已审核的单据不允许删除
  if (oldData[0].status >= 20) {
    throw new Error('已审核/已生效的单据不允许删除，请使用作废功能');
  }

  // 3. 执行逻辑删除
  await query(
    `UPDATE ${table} SET deleted = 1, update_time = NOW() WHERE id = ?`,
    [id]
  );

  // 4. 记录审计日志
  await AuditLogger.log({
    module: options.module,
    type: 'delete',
    title: `删除${table}记录`,
    content: `逻辑删除${table}记录(ID:${id}), 原因: ${options.reason || '未说明'}`,
    beforeData: oldData[0],
    afterData: { ...oldData[0], deleted: 1 },
  });
}
```

```typescript
// src/lib/db-transaction.ts
import { transaction } from '@/lib/db';
import { AuditLogger } from './audit-logger';

export async function executeWithAudit<T>(
  operation: () => Promise<T>,
  options: {
    module: string;
    title: string;
    onSuccess?: (result: T) => void;
    onError?: (error: Error) => void;
  }
): Promise<T> {
  const startTime = Date.now();

  try {
    const result = await transaction(async (conn) => {
      return await operation();
    });

    const duration = Date.now() - startTime;

    await AuditLogger.log({
      module: options.module,
      type: 'transaction',
      title: options.title,
      status: 1,
      durationMs: duration,
    });

    options.onSuccess?.(result);
    return result;
  } catch (error: any) {
    const duration = Date.now() - startTime;

    await AuditLogger.log({
      module: options.module,
      type: 'transaction',
      title: `${options.title} - 失败`,
      status: 0,
      errorMsg: error.message,
      durationMs: duration,
    });

    options.onError?.(error);
    throw error;
  }
}
```

```sql
  module,
  type,

  SUM(CASE WHEN status = 1 THEN 1 ELSE 0 END) as success_count,
```
