# 编码规范

> 基于 `eslint.config.mjs`、`.prettierrc`、`tsconfig.json` 描述代码风格与静态检查规则。

## 1. TypeScript 严格模式

`tsconfig.json` 关键配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `strict` | `true` | 启用所有严格类型检查 |
| `noEmit` | `true` | 仅做类型检查，不输出 JS |
| `isolatedModules` | `true` | 每个文件独立编译，禁止跨文件常量枚举等 |
| `moduleResolution` | `bundler` | 使用打包器解析策略 |
| `jsx` | `react-jsx` | React 19 自动 runtime |
| `paths` | `@/*` → `./src/*` | 路径别名，统一使用 `@/` 前缀 |

`exclude` 中显式排除 `scripts`、`vnerp`、`node_modules`，避免脚本污染构建类型检查。

## 2. ESLint 配置

`eslint.config.mjs` 采用扁平配置（Flat Config），组合 `eslint-config-next` 的 `core-web-vitals` 与 `typescript` 两套预设，并注册自定义插件。

### 2.1 自定义规则 `i18n/no-chinese-hardcode`

来自 `eslint-rules/no-chinese-hardcode.js`，用于禁止在源码中硬编码中文字符串：

```js
'i18n/no-chinese-hardcode': ['warn', {
  allowInFunctions: ['tc', 't', 'useTranslations', 'formatMessage'],
  allowInFiles: ['i18n/', 'locales/', 'messages/'],
  ignoreComments: true,
  generateTranslationKey: true,
  ignorePatterns: ['node_modules/', '.next/', 'dist/', 'build/'],
}]
```

- 命中中文硬编码时仅告警（`warn`），需用 `useTranslations` / `tc` 替换。
- 翻译函数 `tc` / `t` 内部允许包含中文（已处理国际化）。
- `messages/`、`i18n/`、`locales/` 目录豁免（翻译资源所在）。

### 2.2 忽略规则

`globalIgnores` 在 Next.js 默认忽略基础上额外豁免：

- `messages/**`、`eslint-rules/**`：翻译资源与规则定义。
- `**/*.test.{ts,tsx,js,jsx}`、`**/*.spec.*`、`**/__tests__/**`：测试文件不强制中文化。

### 2.3 执行命令

```bash
pnpm lint         # eslint src/
pnpm lint:fix     # 自动修复
pnpm ts-check     # tsc --noEmit
```

## 3. Prettier 配置

`.prettierrc`：

```json
{
  "semi": true,
  "trailingComma": "es5",
  "singleQuote": true,
  "printWidth": 100,
  "tabWidth": 2
}
```

- 语句末尾保留分号；字符串使用单引号。
- 尾逗号采用 `es5`（兼容旧浏览器与部分工具）。
- 行宽 100；缩进 2 空格（与 TS 配置一致）。

执行命令：

```bash
pnpm format         # prettier --write "src/**/*.{ts,tsx,js,jsx,json}"
pnpm format:check   # 仅检查不写入
```

## 4. 命名约定速查

| 类型 | 规则 | 示例 |
|------|------|------|
| 页面文件 | kebab-case | `production-report/page.tsx` |
| 组件文件 | PascalCase | `WorkOrderCard.tsx` |
| 工具/库文件 | kebab-case | `fifo-allocation.ts` |
| API Route | kebab-case | `process-reports/route.ts` |
| 变量/函数 | camelCase | `allocateFIFO` |
| 常量 | UPPER_SNAKE_CASE | `MAX_LOGIN_ATTEMPTS` |
| 类型/接口 | PascalCase | `UserInfo`、`FIFOAllocationResult` |
| React 组件 | PascalCase | `<WorkOrderList />` |
| 自定义 Hook | `use` 前缀 | `useAuth`、`usePermission` |
| 数据库表 | snake_case + 模块前缀 | `inv_inventory_batch`、`prd_work_order` |
| 数据库字段 | snake_case | `material_id`、`create_time` |

## 5. TypeScript 编码要点

- 优先 `interface` 描述对象类型，`type` 用于联合 / 工具类型。
- 禁止 `any`，必要时代替为 `unknown` 并收窄。
- 禁止 `@ts-ignore`，应修复类型错误。
- 异步统一使用 `async/await`，避免 `.then()` 链。
- 涉及数据库的接口必须使用参数化查询（见 [数据库规范](./数据库规范.md)）。

## 6. React / Next.js 编码要点

- 页面组件使用 `export default function PageName()`。
- 共享组件使用命名导出 `export function ComponentName()`。
- Props 必须定义 `interface`，事件处理函数以 `handle` 前缀命名。
- 服务端组件优先；仅需要交互的子树标记 `'use client'`。
- 样式优先使用 Tailwind 工具类与 shadcn/ui 组件，颜色使用 CSS 变量（见 [颜色主题修复方案](./颜色主题修复方案.md)）。

## 7. i18n 编码要点

- 业务文案统一使用 `useTranslations('namespace')` + `tc/t('key')`。
- 翻译资源位于 `messages/{en,zh-CN,zh-TW,vi}.json`。
- 新增页面后必须同步 4 个语言文件，否则 `pnpm lint` 会告警。
