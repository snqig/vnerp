## 1. 核心问题回顾

当前系统跨模块操作存在以下一致性问题：

| 问题 | 影响 | 典型场景 |
|------|------|----------|
| 事务边界不完整 | 部分操作未在事务内执行 | 领料成功但库存未扣减 |
| 补偿机制弱 | 事件处理失败无回滚能力 | 财务凭证生成失败，无法回滚库存 |
| 并发风险 | 高并发下数据不一致 | 多用户同时领料导致负库存 |
| 库存超扣 | 无预留机制 | 工单已释放但库存被其他操作占用 |
| HR 数据不同步 | 报工数据与薪资计算脱节 | 报工变更影响已结算工资 |
| 缺少状态追踪 | 无法追踪跨模块操作状态 | 无法定位哪个环节失败 |

### 1.1 当前链路分析

```
生产工单领料 → 库存批次扣减 → 财务成本归集 → HR 计件薪资计算
       ↓             ↓             ↓                ↓
    事件触发      事件触发      事件触发         事件触发
       ↓             ↓             ↓                ↓
  [Outbox]      [Outbox]      [Outbox]         [Outbox]
       ↓             ↓             ↓                ↓
  Poller        Poller        Poller           Poller
       ↓             ↓             ↓                ↓
  Redis Stream → Redis Stream → Redis Stream → Redis Stream
       ↓             ↓             ↓                ↓
  Handler       Handler       Handler          Handler
```

**问题**：各步骤独立执行，无全局协调，某步失败无法自动补偿。

```
┌─────────────────────────────────────────────────────────────────────────┐

│  └─────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 核心设计原则

1. **保持 DDD 纯净性**：领域实体不依赖基础设施
2. **事务内写入 Outbox**：业务数据与事件同事务持久化
3. **Saga 状态追踪**：每个跨模块操作有完整状态记录
4. **补偿优先于重试**：失败时触发补偿事件而非无限重试
5. **幂等性保障**：所有事件处理支持重复执行

**Saga 状态流转**：

```
Pending → Executing → Success
            │            │
            │            ▼
            │       Completed
            │
            ▼
         Failed → Compensating → Compensated
                       │
                       ▼
                   Failed
```

**CrossModuleSagaHandler 核心逻辑**：

```typescript
// 伪代码示例（待实现）
export class CrossModuleSagaHandler {
  async handleWorkOrderCompletion(sagaId: string, workOrderId: number): Promise<void> {
    // 1. 创建 Saga 记录
    await this.sagaLogRepository.create(sagaId, 'workorder_completion', { workOrderId });
    
    try {
      // 2. 步骤 1：更新工单状态
      await this.markStepPending(sagaId, 'update_workorder');
      await this.workOrderService.complete(workOrderId);
      await this.markStepSuccess(sagaId, 'update_workorder');
      
      // 3. 步骤 2：成品入库
      await this.markStepPending(sagaId, 'inventory_inbound');
      await this.eventBus.publish('workorder.completed', { workOrderId });
      // 等待异步处理完成...
      await this.markStepSuccess(sagaId, 'inventory_inbound');
      
      // 4. 步骤 3：财务成本归集
      await this.markStepPending(sagaId, 'finance_cost');
      await this.eventBus.publish('production.finance.cost_collect', { workOrderId });
      await this.markStepSuccess(sagaId, 'finance_cost');
      
      // 5. 步骤 4：HR 计件薪资计算
      await this.markStepPending(sagaId, 'hr_salary');
      await this.eventBus.publish('hr.salary.piece_calculate', { workOrderId });
      await this.markStepSuccess(sagaId, 'hr_salary');
      
      // 6. 完成 Saga
      await this.completeSaga(sagaId);
      
    } catch (error) {
      // 7. 触发补偿
      await this.triggerCompensation(sagaId);
      throw error;
    }
  }
  
  async triggerCompensation(sagaId: string): Promise<void> {
    const saga = await this.sagaLogRepository.get(sagaId);
    // 反向回滚已成功的步骤
    const steps = saga.steps.filter(s => s.status === 'success').reverse();
    
    for (const step of steps) {
      await this.markStepCompensating(sagaId, step.name);
      try {
        await this.eventBus.publish(`saga.compensate.${step.name}`, saga.payload);
        await this.markStepCompensated(sagaId, step.name);
      } catch (compError) {
        await this.markStepCompensateFailed(sagaId, step.name, String(compError));
      }
    }
  }
}
```

```typescript
      }
```
```typescript
    }
```
```typescript
  });
```
