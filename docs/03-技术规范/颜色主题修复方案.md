# 颜色主题修复方案

> 基于 `src/app/globals.css` 与 `next-themes` 实际实现描述。项目使用 Tailwind v4 + shadcn/ui + `next-themes` 主题系统。

## 1. 现状

| 状态 | 说明 |
|------|------|
| ✅ 已有 | `next-themes` 提供的深 / 浅 / 自定义主题切换 |
| ✅ 已有 | `globals.css` 中完整的 CSS 变量（亮色 `:root` + 暗色 `.dark`） |
| ✅ 已有 | `@theme inline` 将 CSS 变量映射为 Tailwind 工具色（`bg-background`、`text-foreground`…） |
| ❌ 问题 | 后期新增模块仍可能硬编码 `#fff` / `rgb(...)` / `white` 等颜色，导致深色模式显示异常 |

## 3. 修复原则

- **最小改动**：仅替换硬编码颜色，不动现有正常工作的主题系统。
- **变量复用**：直接使用已有变量，不新增冗余变量。
- **快速验证**：每修一个模块立即切深色 / 浅色模式验证。
- **预防未来**：通过 ESLint / 评审防止再引入硬编码颜色。

## 4. 替换对照表

| 硬编码值 | 替换为 | 适用 |
|---------|--------|------|
| `#ffffff` / `white` | `var(--card)` / `var(--primary-foreground)` | 白色背景 / 主色文字 |
| `#000000` / `black` | `var(--foreground)` | 主要文字 |
| `#f8fafc` / `#f1f5f9` | `var(--background)` / `var(--secondary)` / `var(--muted)` | 次要背景 |
| `#e2e8f0` / `#ddd` | `var(--border)` | 边框 / 分割线 |
| `#64748b` / `#94a3b8` | `var(--muted-foreground)` | 次要文字 |
| `#3b82f6` / `#007bff` | `var(--primary)` | 主色调 / 按钮 / 链接 |
| `#10b981` | `var(--success)` | 成功 |
| `#f59e0b` | `var(--warning)` | 警告 |
| `#ef4444` | `var(--destructive)` / `var(--danger)` | 危险 |

在 JSX 中优先使用 Tailwind 工具类（`bg-card`、`text-foreground`、`border-border`、`bg-primary`、`text-primary-foreground`），暗色模式自动跟随。

## 5. 排查与修复

### 5.1 全局搜索正则

```regex
#([0-9a-fA-F]{3}|[0-9a-fA-F]{6})\b|rgb\(\s*\d+\s*,\s*\d+\s*,\s*\d+\s*\)|rgba\(\s*\d+\s*,\s*\d+\s*,\s*\d+\s*,\s*[\d.]+\s*\)|\b(white|black|red|green|blue|yellow|purple|orange|gray|grey)\b
```

重点目录：`src/app/`、`src/components/`，排除 `src/app/globals.css` 与 `src/components/theme-settings.tsx`。

### 5.2 典型修复

```tsx
// ❌ 错误
<div className="bg-white text-black">生产工单</div>
<div style={{ background: '#fff', color: '#333', border: '1px solid #eee' }} />

// ✅ 正确
<div className="bg-card text-foreground">生产工单</div>
<div className="bg-card text-foreground border border-border" />
```

```ts
// ❌ 错误
const statusColors = { pending: '#ffc107', running: '#28a745', failed: '#dc3545' };

// ✅ 正确
const statusColors = {
  pending: 'var(--warning)',
  running: 'var(--success)',
  failed: 'var(--destructive)',
};
```

## 6. 防护机制

- ESLint 自定义规则（参考 `eslint-rules/` 模式）可扩展校验硬编码颜色字面量。
- shadcn/ui 组件已封装主题色，业务代码应直接使用组件而非自写颜色。
- Code Review 重点检查新增样式是否使用 CSS 变量或 Tailwind 工具类。
- 深色模式回归：每次涉及 UI 的 PR 必须切换深 / 浅模式截图验证。
