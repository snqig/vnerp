# 04-运维指南总览

## 关键源码位置

- 部署编排：`Dockerfile`、`docker-compose.yml`、`docker-compose.prod.yml`、`docker-compose.monitoring.yml`
- CI/CD 流水线：`.github/workflows/ci.yml`
- 环境变量模板：`.env.example`、`.env.production.example`
- 数据库初始化脚本：`scripts/setup-db.mjs`
- 备份脚本：`scripts/backup-database.ts`
- 数据库 schema 与种子：`database/vnerpdacahng_schema.sql`、`database/seeds/vnerp-seed-data.sql`
- 监控配置：`prometheus/prometheus.yml`、`alertmanager/alertmanager.yml`

## 环境基线

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Node.js | 18.x | 20.x LTS（Docker 基础镜像 `node:20-alpine`） |
| pnpm | 9.0.0 | 9.x（`package.json` 已锁定 `packageManager`） |
| MySQL | 8.0 | 8.0（`utf8mb4` 字符集） |
| Redis | 7.0 | 7-alpine（可选，用于 token 黑名单与共享缓存） |

## 阅读建议

1. 新加入的开发者：按 `本地开发 → 系统初始化配置 → 数据库迁移` 顺序阅读。
2. 部署人员：先读 `Docker部署指南`，正式上线前对照 `生产部署` 与 `Docker安全扫描报告` 完成安全清单。
3. DBA：直接阅读 `数据库迁移` 与 `备份恢复`，注意 SQL 迁移文件位于 `database/migrations/` 目录。
4. 运维/SRE：阅读 `监控告警接入方案` 配置 Prometheus + Grafana 监控栈。
