# 04-运维指南总览

本目录汇集 VNERP（印刷生产经营信息管理系统 Print MIS）系统的运维相关文档，覆盖本地开发、容器化部署、生产环境、数据库迁移、备份恢复与系统初始化全流程。所有内容基于当前代码现状编写，可直接对照源码执行。

## 文档索引

| 序号 | 文档 | 用途 | 适用读者 |
|------|------|------|----------|
| 1 | [本地开发.md](./本地开发.md) | 开发机环境搭建、依赖安装、数据库初始化、启动开发服务器 | 全体开发者 |
| 2 | [部署指南.md](./部署指南.md) | 基于 Docker / Docker Compose 的容器化部署流程 | DevOps、后端 |
| 3 | [生产部署.md](./生产部署.md) | 生产环境配置项、安全清单、Nginx 反向代理与更新流程 | 运维负责人 |
| 4 | [数据库迁移.md](./数据库迁移.md) | Drizzle ORM 迁移流程、`db:*` 脚本说明、回滚策略 | 后端、DBA |
| 5 | [备份恢复.md](./备份恢复.md) | 数据库备份策略、`backup-database.ts` 脚本、恢复演练 | DBA、运维 |
| 6 | [系统初始化配置.md](./系统初始化配置.md) | `pnpm setup:db` CLI 初始化流程、参数与容错机制 | 实施人员、运维 |

## 关键源码位置

- 部署编排：`Dockerfile`、`docker-compose.yml`、`docker-compose.prod.yml`
- 环境变量模板：`.env.example`、`.env.production.example`
- 数据库初始化脚本：`scripts/setup-db.mjs`
- 备份脚本：`scripts/backup-database.ts`
- Drizzle 配置：`drizzle.config.ts`
- 数据库 schema 与种子：`database/vnerpdacahng_schema.sql`、`database/seeds/vnerp-seed-data.sql`

## 环境基线

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Node.js | 18.x | 20.x LTS（Docker 基础镜像 `node:20-alpine`） |
| pnpm | 9.0.0 | 9.x（`package.json` 已锁定 `packageManager`） |
| MySQL | 8.0 | 8.0（`utf8mb4` 字符集） |
| Redis | 7.0 | 7-alpine（可选，用于 token 黑名单与共享缓存） |

## 阅读建议

1. 新加入的开发者：按 `本地开发 → 系统初始化配置 → 数据库迁移` 顺序阅读。
2. 部署人员：先读 `部署指南`，正式上线前对照 `生产部署` 完成安全清单。
3. DBA：直接阅读 `数据库迁移` 与 `备份恢复`，注意 Drizzle 迁移文件位于 `drizzle/` 目录。
