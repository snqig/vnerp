# 本地开发

本文档说明 vnerp（印刷生产经营信息管理系统 Print MIS）项目在开发机上的环境搭建流程，对应 `package.json` scripts、`.env.example` 与 `scripts/setup-db.mjs`。

## 1. 环境前置

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Node.js | 18+（推荐 20.x LTS） | Docker 基础镜像使用 `node:20-alpine` |
| pnpm | 9.0.0+ | `package.json` 中 `packageManager: pnpm@9.0.0`，`preinstall` 钩子经 `only-allow pnpm` 强制 |
| MySQL | 8.0+ | 字符集 `utf8mb4`，推荐 collation `utf8mb4_0900_ai_ci` |
| Redis | 7.0（可选） | 未配置时事件总线与 token 黑名单降级为进程内内存，仅单实例可用 |

通过 `corepack` 启用 pnpm：

```bash
corepack enable
corepack prepare pnpm@9.0.0 --activate
```

## 2. 安装依赖

```bash
pnpm install
```

`pnpm-lock.yaml` 必须提交到仓库，CI 使用 `--frozen-lockfile` 安装。`package.json` 中 `engines.pnpm: ">=9.0.0"`，使用 npm/yarn 会被 `preinstall` 钩子拦截并报错。

## 3. 配置环境变量

复制模板并填入本地值：

```bash
cp .env.example .env
```

关键字段（详见 `.env.example`）：

| 变量 | 默认 | 说明 |
|------|------|------|
| `DB_HOST` / `DB_PORT` / `DB_USER` / `DB_PASSWORD` / `DB_NAME` | 127.0.0.1 / 3306 / your_db_user / - / vnerp | MySQL 连接信息 |
| `JWT_SECRET` | CHANGE_ME | 生产必须替换为 64 字节随机十六进制串 |
| `JWT_ACCESS_TOKEN_TTL` | 24h | Access Token 过期时间（duration string） |
| `JWT_REFRESH_COOKIE_MAX_AGE` | 604800 | Refresh Token cookie 有效期（秒，默认 7 天） |
| `LOGIN_LOCKOUT_MINUTES` | 15 | 账号锁定时长（分钟） |
| `LOGIN_RATE_LIMIT_WINDOW_MS` / `LOGIN_RATE_LIMIT_MAX` | 900000 / 20 | 单 IP 限流窗口（毫秒）/ 最大次数 |
| `CSRF_COOKIE_MAX_AGE` | 604800 | CSRF cookie 有效期（秒） |
| `COOKIE_SECURE` | false | Cookie secure 标志（生产 HTTPS 设为 true） |
| `DEBUG_DB` | true | 开发环境打印 SQL 日志 |
| `DEV_ORIGINS` | localhost | `next.config.ts` 的 `allowedDevOrigins`，逗号分隔 |
| `CORS_ALLOW_ORIGIN` | * | 开发环境允许 `*`，生产必须设置具体域名 |
| `EVENT_BUS_TYPE` | db | 事件总线模式，`memory`（开发无 Redis）/ `db`（生产 + Redis） |
| `REDIS_URL` | redis://localhost:6379 | 未配置时降级进程内内存 |
| `ALLOW_SETUP_API` | false | 高危建表接口开关，生产必须为 false |
| `NEXT_PUBLIC_AUTH_COOKIE_MODE` | header | 认证模式（header / cookie），切换需 `next build` + 重启 |
| `APP_VERSION` | 1.0.0 | 应用版本号（健康检查展示） |

生成强随机 JWT 密钥：

```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

## 4. 初始化数据库

```bash
# 仅建表
pnpm setup:db

# 建表 + 导入种子数据
pnpm setup:db --seed
```

`scripts/setup-db.mjs` 执行流程（详见 [系统初始化配置.md](./系统初始化配置.md)）：

1. 从 `.env` 读取数据库配置（不依赖 dotenv，避免引入额外依赖）。
2. 连接 MySQL（不指定数据库），按 `DB_NAME` 创建数据库（`utf8mb4_0900_ai_ci`）。
3. 读取 `database/vnerpdacahng_schema.sql`，预处理为 `CREATE TABLE IF NOT EXISTS` 与 `INSERT IGNORE INTO`，按 `;` 拆分逐条执行，单条失败不中断。
4. `--seed` 时读取 `database/seeds/vnerp-seed-data.sql`，同样将 `INSERT INTO` 替换为 `INSERT IGNORE INTO`。
5. 标记 `database/migrations/` 下所有迁移文件为已执行（batch 0），避免重复执行增量迁移。
6. 输出当前表数量与失败明细（前 5 条）。

> 全新环境务必先 `pnpm setup:db --seed`；已有环境增量演进用 `pnpm migrate`，详见 [数据库迁移.md](./数据库迁移.md)。

## 5. 启动开发服务器

```bash
# 默认（Turbopack）
pnpm dev
```

`dev` 脚本为 `next dev -p 5000`，默认端口 **5000**。访问 `http://localhost:5000`。

### Windows 环境

Windows 上 Turbopack 处理 `globals.css` 时可能触发 `nul` 路径崩溃。改用 webpack 模式启动：

```bash
pnpm dev:webpack
# 等价于 next dev -p 5000 --webpack
```

## 7. 中间件与调试路由

中间件位于 `src/proxy.ts`（Next.js 16 自动探测，**`src/middleware.ts` 已删除**）。其行为：

- **i18n**：`createIntlMiddleware`，locales=`[zh-CN, zh-TW, en, vi]`，defaultLocale=`zh-CN`，`localePrefix: 'as-needed'`（默认 zh-CN 不带前缀）。
- **生产环境封堵调试路由**：`/qrcode`、`/api/init`、`/api/debug`、`/api/diagnose` → 404。
- **API 路由**（`/api/`）：公开 API（`/api/auth/login`、`/api/auth/register`、`/api/health`、`/api/migrations`）放行；其余须携带 `access_token` cookie（无则 401）；`requiresCsrfValidation` 的请求校验 CSRF token（失败 403）。
- **页面鉴权**：26 个受保护前缀（`dashboard` / `warehouse` / `sales` / `purchase` / `production` / `quality` / `hr` / `finance` / `equipment` / `engineering` / `outsource` / `orders` / `reports` / `settings` / `dcprint` / `sample` / `analysis` / `base-data` / `business` / `srm` / `prepress` / `plm` / `delivery` / `crm` / `tools` / `material-requisitions` / `modules` / `qrcode`）无 `access_token` → 重定向 `/login?next=原路径`；已登录访问 `/login` 等 → 重定向 `/dashboard`。
- **JWT 校验**：proxy 只校验 cookie 存在性（Edge runtime 不解析 JWT），JWT 实际有效性由 SSR `layout.tsx` 与 API 路由 `verifyToken` 校验。

开发环境正常访问所有路由；生产环境调试路由被封堵。

## 8. Git 钩子与提交规范

项目使用 husky + lint-staged + commitlint：

- `pre-commit`：lint-staged 对暂存的 `src/**/*.{ts,tsx,js,jsx}` 运行 `eslint --fix` + `prettier --write`。
- `commit-msg`：`@commitlint/config-conventional` 校验提交信息格式（如 `feat:`、`fix:`、`chore:`）。

```bash
# 安装钩子（pnpm install 时自动执行 prepare 脚本）
pnpm install

# 手动触发
pnpm prepare
```

## 9. 常见问题

### 9.1 `pnpm install` 失败：`Expected package manager: pnpm@9.0.0`

使用了 npm/yarn。改用 `pnpm install`，或通过 `corepack` 启用 pnpm。

### 9.2 端口 5000 被占用

```bash
# Windows
netstat -ano | findstr :5000

# Linux/Mac
lsof -i :5000
```

终止占用进程或修改 `package.json` 中 `dev` 脚本的端口（不建议改动，5000 是项目约定端口）。

### 9.3 `ER_ACCESS_DENIED_ERROR`

数据库账号或密码错误。检查 `.env` 中 `DB_USER` / `DB_PASSWORD`：

```bash
mysql -h 127.0.0.1 -u <DB_USER> -p<DB_PASSWORD> -e "SELECT 1"
```

### 9.4 Windows 上 Turbopack 崩溃

`globals.css` 触发 `nul` 路径错误。改用 `pnpm dev:webpack`。

### 9.5 事件丢失或处理器不执行

开发无 Redis 时若 `EVENT_BUS_TYPE=db`，OutboxPoller 仍会运行，但事件总线降级为 direct publish。建议开发无 Redis 时设 `EVENT_BUS_TYPE=memory`，生产环境用 `db` + Redis。详见 `.env.example` 注释。
