# 本地开发

本文档说明 VNERP 项目在开发机上的环境搭建流程，对应 `package.json` 脚本、`.env.example` 与 `scripts/setup-db.mjs`。

## 1. 环境前置

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Node.js | 18+（推荐 20.x LTS） | Docker 基础镜像使用 `node:20-alpine` |
| pnpm | 9.0+ | `package.json` 中 `packageManager: pnpm@9.0.0`，`preinstall` 钩子强制使用 pnpm |
| MySQL | 8.0+ | 字符集 `utf8mb4`，推荐 collation `utf8mb4_0900_ai_ci` |
| Redis | 7.0（可选） | 未配置时降级为进程内内存，仅单实例可用 |

通过 `corepack` 启用 pnpm：

```bash
corepack enable
corepack prepare pnpm@9.0.0 --activate
```

## 2. 安装依赖

```bash
pnpm install
```

`pnpm-lock.yaml` 必须提交到仓库，CI 使用 `--frozen-lockfile` 安装。`package.json` 中 `engines.pnpm: ">=9.0.0"`，使用 npm/yarn 会被 `preinstall` 钩子拦截。

## 3. 配置环境变量

复制模板并填入本地值：

```bash
cp .env.example .env
```

关键字段（详见 `.env.example`）：

| 变量 | 默认 | 说明 |
|------|------|------|
| `DB_HOST` / `DB_PORT` / `DB_USER` / `DB_PASSWORD` / `DB_NAME` | 127.0.0.1 / 3306 / - / - / vnerp | MySQL 连接信息 |
| `JWT_SECRET` | dev-only-secret-key | 生产必须替换为 64 字节随机串 |
| `DEBUG_DB` | true | 开发环境打印 SQL 日志 |
| `DEV_ORIGINS` | localhost | `next.config.ts` 的 `allowedDevOrigins`，逗号分隔 |
| `CORS_ALLOW_ORIGIN` | * | 开发环境允许 `*`，生产必须设置具体域名 |
| `EVENT_BUS_TYPE` | memory | 事件总线模式，可选 `memory` / `db` |
| `REDIS_URL` | redis://localhost:6379 | 未配置时降级进程内内存 |
| `ALLOW_SETUP_API` | false | 高危建表接口开关，生产必须为 false |

## 4. 初始化数据库

```bash
pnpm setup:db              # 仅建表
pnpm setup:db --seed       # 建表 + 导入种子数据
```

`scripts/setup-db.mjs` 执行流程：

1. 从 `.env` 读取数据库配置（不依赖 dotenv，避免引入额外依赖）。
2. 连接 MySQL，按 `DB_NAME` 创建数据库（`utf8mb4_0900_ai_ci`）。
3. 读取 `database/vnerpdacahng_schema.sql`，预处理为 `CREATE TABLE IF NOT EXISTS` 与 `INSERT IGNORE INTO`，逐语句执行，单条失败不中断。
4. `--seed` 时导入 `database/seeds/vnerp-seed-data.sql`。
5. 输出当前表数量与失败明细（前 5 条）。

## 5. 启动开发服务器

```bash
pnpm dev
```

`dev` 脚本为 `next dev -p 5000`，默认端口 5000。访问 `http://localhost:5000`。

## 6. 常用命令

| 命令 | 作用 |
|------|------|
| `pnpm lint` / `pnpm lint:fix` | ESLint 检查 / 自动修复 |
| `pnpm ts-check` | TypeScript 类型检查 |
| `pnpm test` | 运行 Playwright E2E |
| `pnpm test:unit` | 运行 Vitest 单元测试 |
| `pnpm db:studio` | 启动 Drizzle Studio 可视化数据库 |

## 7. 调试路由限制

`src/middleware.ts` 在生产环境屏蔽 `/debug`、`/test-api`、`/diagnostic`、`/test`、`/qrcode` 等调试路由（返回 404），开发环境正常访问。
