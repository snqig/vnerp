# 生产部署

本文档说明 印刷生产经营信息管理系统 Print MIS 生产环境部署的配置项、安全要求与更新流程，对应 `docker-compose.prod.yml` 与 `.env.production.example`。

## 1. 系统要求

| 组件 | 最低版本 | 推荐版本 | 备注 |
|------|---------|---------|------|
| Node.js | 18.x | 20.x LTS | Docker 镜像使用 `node:20-alpine` |
| pnpm | 9.0+ | 9.x | `corepack` 自动启用 |
| MySQL | 8.0+ | 8.0 | 字符集 `utf8mb4`，collation `utf8mb4_unicode_ci` |
| Redis | 7.0+ | 7-alpine | 用于 token 黑名单与缓存共享 |
| 服务器内存 | 2GB | 4GB+ | MySQL buffer pool 默认 512M |
| 磁盘 | 10GB | 50GB+ | 含数据卷与备份空间 |

## 2. 生产环境变量（.env.production）

复制 `.env.production.example` 为 `.env.production`，填入真实值（切勿提交到 git）：

| 变量 | 示例值 | 说明 |
|------|--------|------|
| `DB_HOST` | `mysql` | Docker 网络内的服务名 |
| `DB_PORT` | `3306` | MySQL 端口 |
| `DB_USER` | `vnerp_app` | 应用专用账号（不要使用 root） |
| `DB_PASSWORD` | 强密码 | 应用账号密码 |
| `DB_NAME` | `vnerp` | 数据库名 |
| `MYSQL_ROOT_PASSWORD` | 强密码 | 仅 docker-compose 初始化使用 |
| `JWT_SECRET` | 64 字节 hex 串 | `node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"` 生成 |
| `NODE_ENV` | `production` | 启用安全头与调试路由屏蔽 |
| `DEBUG_DB` | `false` | 关闭 SQL 日志 |
| `CORS_ALLOW_ORIGIN` | `https://erp.your-domain.com` | 生产禁止 `*` |
| `DEV_ORIGINS` | （空） | 生产无需配置 |
| `REDIS_URL` | `redis://redis:6379` | Docker 内部地址 |
| `PORT` | `5000` | 应用监听端口 |
| `HOSTNAME` | `0.0.0.0` | 容器内监听地址 |

## 3. 启动生产环境

```bash
# 构建
docker compose -f docker-compose.prod.yml --env-file .env.production build

# 启动（含 mysql / redis / app）
docker compose -f docker-compose.prod.yml --env-file .env.production up -d

# 查看应用日志
docker compose -f docker-compose.prod.yml logs -f app

# 查看服务状态
docker compose -f docker-compose.prod.yml ps
```

## 4. Nginx 反向代理（可选）

如需 HTTPS 或域名，建议在容器外或独立容器部署 Nginx：

```nginx
server {
    listen 443 ssl http2;
    server_name erp.your-domain.com;

    ssl_certificate     /etc/ssl/certs/your-cert.pem;
    ssl_certificate_key /etc/ssl/private/your-key.pem;

    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    location /_next/static/ {
        proxy_pass http://127.0.0.1:5000;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }
}
```

`next.config.ts` 在生产环境已自动添加 HSTS、X-Frame-Options 等安全响应头。

## 5. 更新部署

### 5.1 常规更新

```bash
git pull origin main
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build app
```

### 5.2 含数据库迁移

```bash
# 备份数据库
mysqldump -h 127.0.0.1 -u root -p vnerp > pre_migration_backup.sql

# 执行 Drizzle 迁移（详见 04-运维指南/数据库迁移.md）
pnpm db:migrate

# 重建并重启
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build app
```

## 6. 健康检查与监控

```bash
# 应用健康
curl -f http://localhost:5000/api/health

# 容器状态
docker compose -f docker-compose.prod.yml ps

# MySQL 连接
docker exec vnerp-mysql-prod mysqladmin -uvnerp_app -p ping

# Redis 连接
docker exec vnerp-redis-prod redis-cli ping
```

`/api/health` 返回 JSON：`{ status, database, memory, uptime, version }`，状态码 200 = 健康，503 = 降级。

## 7. 安全清单（上线前必查）

- [ ] `.env.production` 未提交到 git，密码为强随机串
- [ ] `JWT_SECRET` 为 64 字节随机 hex，非默认值
- [ ] `ALLOW_SETUP_API=false`（建表接口禁用）
- [ ] `CORS_ALLOW_ORIGIN` 配置为具体域名，非 `*`
- [ ] `EVENT_BUS_TYPE=db`（生产使用持久化事件总线）
- [ ] MySQL root 仅本地访问，应用使用 `vnerp_app` 等受限账号
- [ ] Redis 配置密码认证（如对外暴露）
- [ ] 启用 HTTPS（Nginx SSL）
- [ ] 防火墙仅开放 80/443，不直接暴露 3306/6379
- [ ] 已配置数据库定时备份（详见 `备份恢复.md`）
- [ ] 日志轮转已启用

## 8. 回滚

```bash
# 回滚到上一镜像版本
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --no-deps app

# 数据库回滚需先恢复备份
mysql -h 127.0.0.1 -u root -p vnerp < pre_migration_backup.sql
```
