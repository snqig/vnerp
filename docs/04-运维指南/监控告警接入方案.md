### 数据流

1. **Prometheus** 按配置间隔主动拉取（pull）各 Exporter 的指标
2. **Grafana** 查询 Prometheus 数据绘制仪表盘
3. **Alertmanager** 接收 Prometheus 触发的告警，路由到通知渠道
4. **Exportor** 将应用/数据库/缓存的内部状态转换为 Prometheus 格式

## 监控指标体系

### 四层监控模型

| 层级 | 数据源 | 关键指标 | 采集方式 |
|------|--------|----------|----------|
| **主机层** | node-exporter | CPU 使用率、内存使用率、磁盘空间、网络 IO | node-exporter :9100 |
| **数据库层** | mysqld-exporter | 连接数、QPS、慢查询、复制延迟、表锁 | mysqld-exporter :9104 |
| **缓存层** | redis-exporter | 内存使用、命中率、连接数、Key 数量 | redis-exporter :9121 |
| **应用层** | blackbox + json_exporter | TTFB、响应状态码、API 延迟、内存堆、DB 连接 | App :5000 |
| **反向代理** | nginx-exporter | 请求量、4xx/5xx 错误率、上游响应时间 | nginx stub_status |

### 核心告警指标

| 指标 | 阈值 | 持续时间 | 严重级别 |
|------|------|----------|----------|
| App 健康检查 | 非 200 | 1 分钟 | 🔴 P0-Critical |
| MySQL 连接 | 不可达 | 1 分钟 | 🔴 P0-Critical |
| Redis 连接 | 不可达 | 1 分钟 | 🔴 P0-Critical |
| CPU 使用率 | > 85% | 5 分钟 | 🟡 P1-Warning |
| 磁盘空间 | > 85% | 5 分钟 | 🟡 P1-Warning |
| 内存使用率 | > 90% | 3 分钟 | 🟡 P1-Warning |
| App 内存堆 | > 512MB | 3 分钟 | 🟡 P1-Warning |
| HTTP 5xx 错误率 | > 5% | 2 分钟 | 🟡 P1-Warning |
| MySQL 慢查询 | > 10/min | 5 分钟 | 🟢 P2-Info |
| Redis 命中率 | < 80% | 10 分钟 | 🟢 P2-Info |

## 部署监控栈

### 1. 创建监控编排文件

监控栈使用独立的 `docker-compose.monitoring.yml`，与生产服务共用 `vnerp-network` 网络。

```bash
# 启动监控栈（生产服务需已运行）
docker compose -f docker-compose.monitoring.yml --env-file .env.production up -d

# 查看监控服务状态
docker compose -f docker-compose.monitoring.yml ps
```

### 2. 访问地址

启动后可通过以下地址访问（建议通过 Nginx 反代，不直接暴露）：

| 服务 | 端口 | 默认凭据 | 说明 |
|------|------|----------|------|
| Grafana | http://localhost:3001 | admin / admin（首次登录必须修改） | 可视化仪表盘 |
| Prometheus | http://localhost:9090 | 无需认证 | 指标查询与告警评估 |
| Alertmanager | http://localhost:9093 | 无需认证 | 告警路由与管理 |

### 3. Nginx 反代配置（推荐）

在 `nginx/nginx.conf` 的 HTTPS server 块中添加：

```nginx
# Grafana（建议加 IP 白名单或 Basic Auth）
location /grafana/ {
    proxy_pass http://grafana:3000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

# Prometheus（仅内网访问）
location /prometheus/ {
    allow 10.0.0.0/8;
    allow 172.16.0.0/12;
    allow 192.168.0.0/16;
    deny all;
    proxy_pass http://prometheus:9090/;
    proxy_set_header Host $host;
}
```

## Prometheus 配置

### prometheus/prometheus.yml

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s
  external_labels:
    monitor: 'vnerp-prod'
    environment: 'production'

rule_files:
  - /etc/prometheus/rules/*.yml

alerting:
  alertmanagers:
    - static_configs:
        - targets: ['alertmanager:9093']

scrape_configs:
  # Prometheus 自身
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  # 主机指标
  - job_name: 'node'
    static_configs:
      - targets: ['node-exporter:9100']
        labels:
          service: 'host'

  # MySQL 指标
  - job_name: 'mysql'
    static_configs:
      - targets: ['mysqld-exporter:9104']
        labels:
          service: 'mysql'

  # Redis 指标
  - job_name: 'redis'
    static_configs:
      - targets: ['redis-exporter:9121']
        labels:
          service: 'redis'

  # 应用健康检查（HTTP 探针）
  - job_name: 'app-health'
    metrics_path: /probe
    params:
      module: [http_2xx]
    static_configs:
      - targets:
          - 'http://app:5000/api/health'
    relabel_configs:
      - source_labels: [__address__]
        target_label: __param_target
      - source_labels: [__param_target]
        target_label: instance
      - target_label: __address__
        replacement: blackbox-exporter:9115
```

## 告警规则

### prometheus/rules/alerts.yml

```yaml
groups:
  - name: app-alerts
    rules:
      # App 不可达
      - alert: AppDown
        expr: up{job="app-health"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "App 服务不可达"
          description: "App 健康检查连续 1 分钟失败 ({{ $labels.instance }})"

      # HTTP 5xx 错误率过高（需 nginx exporter 支持）
      - alert: HighErrorRate
        expr: |
          rate(nginx_requests_total{status=~"5.."}[2m]) /
          rate(nginx_requests_total[2m]) > 0.05
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "HTTP 5xx 错误率 > 5%"
          description: "过去 2 分钟 5xx 错误率: {{ $value | humanizePercentage }}"

  - name: infra-alerts
    rules:
      # MySQL 不可达
      - alert: MySQLDown
        expr: mysql_up == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "MySQL 不可达"

      # Redis 不可达
      - alert: RedisDown
        expr: redis_up == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Redis 不可达"

      # CPU 使用率过高
      - alert: HighCPU
        expr: |
          100 - (avg by(instance)(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "CPU 使用率 > 85%"
          description: "{{ $labels.instance }} CPU: {{ $value }}%"

      # 磁盘空间不足
      - alert: DiskSpaceLow
        expr: |
          100 - (node_filesystem_avail_bytes / node_filesystem_size_bytes * 100) > 85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "磁盘空间 > 85%"
          description: "{{ $labels.mountpoint }} 已用 {{ $value }}%"

      # 内存不足
      - alert: HighMemory
        expr: |
          (1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100 > 90
        for: 3m
        labels:
          severity: warning
        annotations:
          summary: "内存使用率 > 90%"

  - name: db-alerts
    rules:
      # MySQL 连接数过高
      - alert: MySQLHighConnections
        expr: |
          mysql_global_status_threads_connected / mysql_global_variables_max_connections * 100 > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "MySQL 连接数 > 80%"
          description: "当前连接: {{ $value }}%"

      # MySQL 慢查询
      - alert: MySQLSlowQueries
        expr: rate(mysql_global_status_slow_queries[5m]) > 0.17
        labels:
          severity: info
        annotations:
          summary: "MySQL 慢查询 > 10/min"

      # Redis 内存使用过高
      - alert: RedisHighMemory
        expr: |
          redis_memory_used_bytes / redis_memory_max_bytes * 100 > 90
        for: 3m
        labels:
          severity: warning
        annotations:
          summary: "Redis 内存使用 > 90%"

      # Redis 命中率低
      - alert: RedisLowHitRate
        expr: |
          rate(redis_keyspace_hits_total[10m]) /
          (rate(redis_keyspace_hits_total[10m]) + rate(redis_keyspace_misses_total[10m])) < 0.8
        for: 10m
        labels:
          severity: info
        annotations:
          summary: "Redis 命中率 < 80%"
```

## Grafana 仪表盘

### 推荐仪表盘 ID

在 Grafana 中 Import 以下社区仪表盘（Dashboard ID）：

| 仪表盘 | ID | 数据源 | 说明 |
|--------|----|----|------|
| Node Exporter Full | 1860 | Prometheus | 主机 CPU/内存/磁盘/网络 |
| MySQL Overview | 7362 | Prometheus | MySQL 性能概览 |
| Redis Dashboard | 11835 | Prometheus | Redis 性能概览 |
| Nginx Overview | 1270 | Prometheus | Nginx 请求/响应统计 |

### 导入步骤

1. 登录 Grafana → 左侧菜单 **Dashboards** → **Import**
2. 输入 Dashboard ID → **Load**
3. 选择 Prometheus 数据源 → **Import**

### 自定义 ERP 仪表盘

建议创建自定义仪表盘，关键 Panel：

| Panel | PromQL 查询 | 类型 |
|-------|-------------|------|
| App 状态 | `up{job="app-health"}` | Stat |
| API 响应时间 P95 | `histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))` | Graph |
| MySQL QPS | `rate(mysql_global_status_questions[1m])` | Graph |
| MySQL 连接数 | `mysql_global_status_threads_connected` | Gauge |
| Redis 内存 | `redis_memory_used_bytes` | Graph |
| Redis 命中率 | `rate(redis_keyspace_hits_total[5m]) / (rate(redis_keyspace_hits_total[5m]) + rate(redis_keyspace_misses_total[5m]))` | Gauge |
| 主机 CPU | `100 - avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100` | Gauge |
| 磁盘使用率 | `100 - (node_filesystem_avail_bytes / node_filesystem_size_bytes * 100)` | Bar Gauge |

## 告警通知渠道

### Alertmanager 配置

`alertmanager/alertmanager.yml`:

```yaml
global:
  resolve_timeout: 5m

route:
  group_by: ['alertname', 'severity']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  receiver: 'default'

receivers:
  - name: 'default'
    email_configs:
      - to: 'ops-team@your-domain.com'
        from: 'alertmanager@your-domain.com'
        smarthost: 'smtp.your-domain.com:587'
        auth_username: 'alertmanager@your-domain.com'
        auth_password: '${SMTP_PASSWORD}'
        require_tls: true

  # 钉钉通知（需部署 dingtalk-webhook 适配器）
  - name: 'dingtalk'
    webhook_configs:
      - url: 'http://dingtalk-webhook:8060/dingtalk/webhook1/send'
        send_resolved: true
```

### 通知渠道对照

| 渠道 | 适用告警 | 配置方式 |
|------|----------|----------|
| 邮件 | 全部告警 | Alertmanager email_configs |
| 钉钉/企微 | Critical + Warning | Webhook 适配器 |
| Slack | 全部告警 | Alertmanager slack_configs |

## 运维操作

### 查看告警状态

```bash
# 查看当前活跃告警
curl http://localhost:9090/api/v1/alerts | python3 -m json.tool

# 查看 Alertmanager 告警
curl http://localhost:9093/api/v2/alerts | python3 -m json.tool
```

### Prometheus 查询示例

```promql
# App 是否在线
up{job="app-health"}

# MySQL 连接数
mysql_global_status_threads_connected

# Redis 内存使用
redis_memory_used_bytes

# 主机 CPU 使用率
100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# 磁盘可用空间
node_filesystem_avail_bytes{mountpoint="/"}
```

### 查看监控服务日志

```bash
docker compose -f docker-compose.monitoring.yml logs -f prometheus
docker compose -f docker-compose.monitoring.yml logs -f alertmanager
docker compose -f docker-compose.monitoring.yml logs -f grafana
```

### 停止监控栈

```bash
docker compose -f docker-compose.monitoring.yml down
# 监控数据持久化在 named volume 中，不会丢失
```
