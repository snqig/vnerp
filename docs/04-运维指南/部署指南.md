# 部署指南

本文档描述 VNERP 基于 Docker / Docker Compose 的容器化部署流程，对应 `Dockerfile`、`docker-compose.yml` 与 `docker-compose.prod.yml`。

## 1. 镜像构建（Dockerfile）

`Dockerfile` 采用三阶段构建，依赖 `next.config.ts` 中 `output: 'standalone'`（已在 `next.config.ts:23` 启用）。

| 阶段 | 基础镜像 | 作用 |
|------|---------|------|
| `deps` | `node:20-alpine` | 安装依赖（`pnpm install --frozen-lockfile`） |
| `builder` | `node:20-alpine` | 复制源码并执行 `pnpm build` |
| `runner` | `node:20-alpine` | 仅复制 standalone 产物、静态资源、`public/`、`messages/` |

关键设计：

- 使用 `corepack` 启用 `pnpm@9.0.0`，与本地环境一致。
- 运行阶段使用非 root 用户 `nextjs`（UID/GID 1001）。
- 环境变量 `NODE_ENV=production`、`NEXT_TELEMETRY_DISABLED=1`、`PORT=5000`、`HOSTNAME=0.0.0.0`。
- 启动命令 `node server.js`，对外暴露 5000 端口。

手动构建：

```bash
docker build -t vnerp:latest .
```

## 2. 开发环境编排（docker-compose.yml）

`docker-compose.yml` 仅包含 MySQL 与 Redis 两个基础服务，供本地或开发环境使用。

| 服务 | 镜像 | 容器名 | 端口 | 说明 |
|------|------|--------|------|------|
| `mysql` | `mysql:8.0` | `vnerp-mysql` | 3306:3306 | 初始化时挂载 `database/vnerpdacahng_schema.sql` 与 `vnerp-seed-data.sql`，`innodb-buffer-pool-size=256M`，`max-connections=200` |
| `redis` | `redis:7-alpine` | `vnerp-redis` | 6379:6379 | `appendonly yes`，`maxmemory 128mb`，`allkeys-lru` |

两个服务均配置 healthcheck，应用可通过 `mysql` / `redis` 主机名访问。

启动：

```bash
docker compose up -d
docker compose logs -f mysql
```

## 3. 生产编排（docker-compose.prod.yml）

`docker-compose.prod.yml` 在开发编排基础上新增 `app` 服务，使用 `--env-file .env.production` 加载生产配置：

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production build
docker compose -f docker-compose.prod.yml --env-file .env.production up -d
```

`app` 服务关键配置：

- `depends_on` 等待 `mysql` / `redis` healthcheck 通过后再启动。
- `restart: always`，崩溃自动重启。
- 环境变量由 `.env.production` 注入，包含 `DB_USER`、`DB_PASSWORD`、`JWT_SECRET`、`CORS_ALLOW_ORIGIN`、`REDIS_URL=redis://redis:6379` 等。
- `innodb-buffer-pool-size` 升至 512M，Redis `maxmemory` 升至 256M。

容器列表：

| 容器 | 名称 | 端口 |
|------|------|------|
| Nginx | `vnerp-nginx-prod` | 80, 443（对外） |
| App | `vnerp-app-prod` | 5000（内部，由 Nginx 反代） |
| MySQL | `vnerp-mysql-prod` | 3306 |
| Redis | `vnerp-redis-prod` | 6379 |

> App 端口不对外暴露，所有外部请求经 Nginx 反向代理转发。Nginx 配置 SSL 终结、gzip 压缩、登录限流和安全头。

## 4. 部署流程清单

1. 准备生产服务器（推荐 4GB+ 内存、50GB+ 磁盘）。
2. 克隆仓库并 `cp .env.production.example .env.production`，填入真实密码与 `JWT_SECRET`。
3. 生成强随机 JWT 密钥：
   ```bash
   node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
   ```
4. 构建并启动：`docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build`。
5. 查看日志：`docker compose -f docker-compose.prod.yml logs -f app`。
6. 健康检查：`curl http://localhost:5000/api/health`。
7. 停止：`docker compose -f docker-compose.prod.yml down`（数据卷保留）。

## 5. 健康检查

应用提供 `GET /api/health` 接口，返回数据库连接状态、内存使用率、运行时间。状态码 200 表示健康，503 表示降级。可接入 Docker `HEALTHCHECK` 或负载均衡探针。

## 6. 调试路由屏蔽

`src/middleware.ts` 在 `NODE_ENV=production` 下对 `/debug`、`/test-api`、`/diagnostic`、`/test`、`/qrcode` 路径返回 404，避免调试代码泄露。
