## 2. 环境准备

### 2.1 安装 Node.js

```bash
# 使用 nvm 安装 Node.js
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc

nvm install 20
nvm use 20
nvm alias default 20

# 验证安装
node -v  # v20.x.x
npm -v   # 10.x.x
```

### 2.2 安装 PM2

```bash
# 全局安装 PM2
npm install -g pm2

# 验证安装
pm2 -v
```

### 2.3 安装 MySQL 8.0

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mysql-server

# CentOS/RHEL
sudo yum install mysql-server

# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation
```

### 2.4 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx

# 启动 Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

## 3. 项目部署

### 3.1 克隆代码

```bash
# 创建应用目录
sudo mkdir -p /var/www/vnerp
sudo chown -R $USER:$USER /var/www/vnerp

# 克隆代码
cd /var/www/vnerp
git clone https://github.com/your-org/vnerp.git .

# 切换到生产分支
git checkout main
```

### 3.2 安装依赖

```bash
# 安装 pnpm
npm install -g pnpm

# 安装依赖
pnpm install --prod
```

### 3.3 配置环境变量

```bash
# 创建环境变量文件
cp .env.example .env.production

# 编辑环境变量
nano .env.production
```

**.env.production 示例：**

```env
# 数据库配置
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=vnerp_user
DB_PASSWORD=your_secure_password
DB_NAME=vnerpdacahng

# 应用配置
NODE_ENV=production
PORT=5000
NEXT_PUBLIC_API_URL=https://vnerp.example.com

# JWT 配置
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=7d

# Redis 配置（可选）
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
```

### 3.4 构建项目

```bash
# 构建
pnpm build

# 验证构建
ls -la .next/
```

### 3.5 数据库初始化

```bash
# 创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS vnerpdacahng DEFAULT CHARACTER SET utf8mb4;"

# 创建用户
mysql -u root -p -e "CREATE USER 'vnerp_user'@'localhost' IDENTIFIED BY 'your_secure_password';"
mysql -u root -p -e "GRANT ALL PRIVILEGES ON vnerpdacahng.* TO 'vnerp_user'@'localhost';"
mysql -u root -p -e "FLUSH PRIVILEGES;"

# 导入数据库结构
mysql -u vnerp_user -p vnerpdacahng < database/vnerpdacahng_schema.sql

# 导入初始数据
mysql -u vnerp_user -p vnerpdacahng < database/seed-data.sql
```

## 4. PM2 配置

### 4.1 创建 PM2 配置文件

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [
    {
      name: 'vnerp',
      script: 'pnpm',
      args: 'start',
      cwd: '/var/www/vnerp',
      instances: 'max',
      exec_mode: 'cluster',
      autorestart: true,
      watch: false,
      max_memory_restart: '1G',
      env_production: {
        NODE_ENV: 'production',
        PORT: 5000,
      },
      error_file: '/var/log/vnerp/error.log',
      out_file: '/var/log/vnerp/out.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
    },
  ],
};
```

### 4.2 启动应用

```bash
# 创建日志目录
sudo mkdir -p /var/log/vnerp
sudo chown -R $USER:$USER /var/log/vnerp

# 启动应用
pm2 start ecosystem.config.js --env production

# 查看状态
pm2 status

# 查看日志
pm2 logs vnerp

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup
```

## 5. Nginx 配置

### 5.1 创建 Nginx 配置

创建 `/etc/nginx/sites-available/vnerp`：

```nginx
upstream vnerp_backend {
    server 127.0.0.1:5000;
    keepalive 64;
}

server {
    listen 80;
    server_name vnerp.example.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name vnerp.example.com;

    # SSL 配置
    ssl_certificate /etc/letsencrypt/live/vnerp.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/vnerp.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 日志
    access_log /var/log/nginx/vnerp-access.log;
    error_log /var/log/nginx/vnerp-error.log;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;

    # 静态文件
    location /_next/static/ {
        alias /var/www/vnerp/.next/static/;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    location /public/ {
        alias /var/www/vnerp/public/;
        expires 30d;
        add_header Cache-Control "public";
    }

    # API 代理
    location /api/ {
        proxy_pass http://vnerp_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    # 其他请求
    location / {
        proxy_pass http://vnerp_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 5.2 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/vnerp /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 5.3 配置 SSL 证书

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d vnerp.example.com

# 自动续期
sudo certbot renew --dry-run
```

## 7. 备份策略

### 7.1 数据库备份

创建备份脚本 `/usr/local/bin/backup-vnerp.sh`：

```bash
#!/bin/bash

# 配置
DB_NAME="vnerpdacahng"
DB_USER="vnerp_user"
DB_PASS="your_password"
BACKUP_DIR="/var/backups/vnerp"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u $DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 删除7天前的备份
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR/db_$DATE.sql.gz"
```

### 7.2 设置定时备份

```bash
# 添加执行权限
chmod +x /usr/local/bin/backup-vnerp.sh

# 添加定时任务
crontab -e

# 每天凌晨2点备份
0 2 * * * /usr/local/bin/backup-vnerp.sh >> /var/log/vnerp/backup.log 2>&1
```

## 8. 更新部署

### 8.1 更新流程

```bash
# 1. 备份当前版本
cp -r /var/www/vnerp /var/www/vnerp_backup_$(date +%Y%m%d)

# 2. 拉取最新代码
cd /var/www/vnerp
git pull origin main

# 3. 安装依赖
pnpm install --prod

# 4. 构建
pnpm build

# 5. 重启应用
pm2 reload vnerp

# 6. 验证
curl -I https://vnerp.example.com
```

### 8.2 回滚流程

```bash
# 1. 停止应用
pm2 stop vnerp

# 2. 恢复备份
rm -rf /var/www/vnerp
mv /var/www/vnerp_backup_YYYYMMDD /var/www/vnerp

# 3. 重启应用
pm2 start vnerp
```

## 9. 故障排查

### 9.1 常见问题

| 问题 | 排查步骤 |
|------|----------|
| 应用无法启动 | 检查日志 `pm2 logs vnerp`，检查端口占用 `lsof -i:5000` |
| 数据库连接失败 | 检查 MySQL 状态 `systemctl status mysql`，检查连接配置 |
| 页面加载慢 | 检查 Nginx 日志，检查后端响应时间 |
| 内存占用高 | 检查进程内存 `pm2 monit`，考虑增加 `max_memory_restart` |

### 9.2 日志查看

```bash
# 应用日志
pm2 logs vnerp

# Nginx 访问日志
tail -f /var/log/nginx/vnerp-access.log

# Nginx 错误日志
tail -f /var/log/nginx/vnerp-error.log

# MySQL 错误日志
tail -f /var/log/mysql/error.log
```
