## 测试体系

VNERP 项目采用 **单元测试（Vitest）+ E2E 测试（Playwright）** 双层体系，互不交叉：

| 层级 | 框架 | 配置文件 | 测试目录 | 覆盖目标 |
|------|------|----------|----------|----------|
| 单元 / 集成 | Vitest 4 + jsdom + @vitejs/plugin-react | `vitest.config.ts` | `src/**/*.test.ts(x)`、`tests/unit/**` | lib 核心算法、领域层、事件总线基础设施 |
| E2E | Playwright 1.58 | `playwright.config.ts` | `tests/**/*.spec.ts` | 关键业务流程（登录、入库、看板） |

### Vitest 关键配置

- 环境：`jsdom`，启用 `globals: true`
- 入口：`setupFiles: ['./src/tests/setup.ts']`
- 包含：`src/**/*.{test,spec}.{ts,tsx}` 与 `tests/**/*.{test,spec}.{ts,tsx}`
- 排除：`tests/**/*.spec.ts`（避免吞掉 Playwright 用例）
- 超时：测试与 hook 均 30s
- 别名：`@ → ./src`

### 覆盖率门槛（v8 provider，渐进式防退化）

| 指标 | 基线（2026-06） | 阈值 |
|------|------------------|------|
| Lines | 48.88% | 45% |
| Branches | 39.74% | 35% |
| Functions | 76.39% | 70% |
| Statements | 49.02% | 45% |

仅以下核心文件计入覆盖率：`src/lib/{bom-expansion,fifo-allocation,warehouse-state-machine,material-requisition,state-machine,inventory-sync,warehouse-core}.ts`、`src/domain/**`、`src/infrastructure/event-bus/{EventBus,OutboxPoller,MemoryDomainEventOutbox,DomainEventOutboxFactory}.ts`。

### Playwright 关键配置

- `testDir: ./tests`，匹配 `**/*.spec.ts`
- `globalSetup: ./tests/global-setup.ts`
- `baseURL: http://localhost:5000`，`webServer` 自动启动 `pnpm run dev`
- 浏览器：chromium / firefox / webkit / Mobile Chrome / Mobile Safari
- 视口：1920×1080，动作超时 30s，导航超时 60s
- 重试：CI 2 次，本地 0 次；CI 仅 1 worker
- 报告：HTML + list + JSON（`test-results.json`）

## 测试命令速查

```bash
pnpm test:unit          # 监听模式运行 Vitest
pnpm test:unit:run      # 单次运行 Vitest
pnpm test:coverage      # 生成覆盖率报告（coverage/）
pnpm test               # 运行 Playwright E2E
pnpm test:headed        # 有头模式
pnpm test:ui            # Playwright UI 模式
pnpm test:report        # 查看报告
pnpm test:install       # 安装浏览器二进制
```

## 现有测试覆盖范围

- **Vitest 单元/集成**：BOM 展开、FIFO 分配、库存状态机、API 响应、验证 schema、登录鉴权、角色权限继承、成本计算、性能、数据库事务、限流、领域值对象（Money）、入库并发等。
- **Playwright E2E**：`auth.spec.ts`、`dashboard.spec.ts`、`inbound.spec.ts`、`login.spec.ts`、`tests/unit/**`（BOM、FIFO、库存同步、仓库核心）。
- **并发场景**：`tests/concurrency/` 提供工具与文档。
