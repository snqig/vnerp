## 测试体系

vnerp 项目采用 **单元测试（Vitest）+ E2E 测试（Playwright）** 双层体系，互不交叉：

| 层级 | 框架 | 配置文件 | 测试目录 | 覆盖目标 |
|------|------|----------|----------|----------|
| 单元 / 集成 | Vitest 4 + @vitejs/plugin-react | `vitest.config.ts` | `tests/**/*.test.ts` | lib 核心算法、领域聚合根、应用服务、事件总线基础设施、领域事件处理器 |
| E2E | Playwright 1.58 | `playwright.config.ts` | `tests/**/*.spec.ts` | 关键业务流程（登录、入库、看板、销售单） |
| 并发 | Vitest + 模拟 | `tests/concurrency/` | `tests/concurrency/*.test.ts` | 乐观锁 / 并发冲突（出库、领料、盘点、采购入库） |

### Vitest 关键配置（以 `vitest.config.ts` 实际内容为准）

- `globals: true`
- `environment: 'node'`（领域/算法/仓储测试不依赖 DOM）
- `include: ['tests/**/*.test.ts']`
- `setupFiles: ['tests/setup-env.ts']`（加载 `.env`，集成测试依赖 `DB_PASSWORD` / `REDIS_URL`）
- 别名：`@ → ./src`
- 未配置覆盖率阈值（`coverage` 段未在 `vitest.config.ts` 中声明）

> 注：早期文档曾记录 `environment: 'jsdom'` 与覆盖率阈值 `lines 45 / functions 70 / branches 35 / statements 45`，那是计划态而非 `vitest.config.ts` 实际内容。如需启用覆盖率采集与阈值，需补充 `coverage` 配置段。

### 现有 Vitest 用例分布（`tests/unit/`）

| 子目录 | 覆盖内容 |
|--------|----------|
| `tests/unit/domain/` | warehouse / sales / purchase / finance / quality / shared 聚合根、实体、值对象、事件 |
| `tests/unit/application/services/` | ToolManagementService、SampleProcessCardService |
| `tests/unit/application/handlers/` | ReturnOrderInventoryHandler、DeliveryCancelledHandler、DeliveryReceivableHandler、ReconciliationWriteOffHandler、ToolUsageSyncHandler、ToolCostHandler |
| `tests/unit/application/api/` | alert-push |
| `tests/unit/infrastructure/event-bus/` | EventBus 发布、OutboxPoller 生命周期 / 死信 / 重启恢复、event-bus-type-switch、stream-flow-simulation |
| `tests/unit/`（顶层） | fifo-allocation、mrp-engine、bom-expansion、inventory-sync、material-requisition、over-issue-validation、warehouse-core、menu-tree、production-scheduling、production-scheduling-enhanced |

### Playwright 关键配置

- `testDir: ./tests`，`testMatch: '**/*.spec.ts'`
- `globalSetup: ./tests/global-setup.ts`
- `baseURL: http://localhost:5000`
- `webServer.command: 'pnpm run dev:webpack'`（webpack 模式规避 Windows Turbopack 处理 CSS 时读取保留设备名 `nul` 的崩溃），`reuseExistingServer: !CI`
- 浏览器矩阵：chromium / firefox / webkit / Mobile Chrome / Mobile Safari
- 视口 1920×1080，动作超时 30s，导航超时 60s，单测超时 120s（webpack 冷启动较慢）
- 重试：CI 2 次，本地 0 次；CI 仅 1 worker，`forbidOnly: true`
- 报告：HTML（`playwright-report/`）+ list + JSON（`test-results.json`）

## 测试命令速查

```bash
# 单元 / 集成（Vitest）
pnpm test:unit          # 监听模式
pnpm test:unit:run      # 单次运行（CI / 提交前）
pnpm test:coverage      # 生成覆盖率报告（coverage/）

# E2E（Playwright）
pnpm test               # 全量 E2E（首次会自动启动 dev server）
pnpm test:headed        # 有头模式
pnpm test:ui            # Playwright UI 模式
pnpm test:debug         # 调试模式
pnpm test:report        # 查看 HTML 报告
pnpm test:install       # 安装浏览器二进制

# 质量门禁
pnpm ts-check           # TypeScript 类型检查
pnpm lint               # ESLint
pnpm format:check       # Prettier 检查
```

> 完整命令定义见 `package.json` 的 `scripts` 段。

## 现有测试覆盖范围

- **Vitest 单元/集成**：BOM 展开、FIFO 分配、库存同步、仓库核心、物料领用、超领校验、MRP 引擎、生产调度（含 enhanced）、菜单树、领域聚合根（warehouse/sales/purchase/finance/quality）、事件总线（EventBus/OutboxPoller 死信/重启回收/stream 模拟/type 切换）、应用服务（Tool/SampleProcessCard）、关键 Handler（Reconciliation/ReturnOrder/Delivery/ToolUsage/ToolCost）、alert-push API。
- **Vitest 集成**：`tests/integration/` 提供 inbound-api、workorder-api、purchase-reconciliation-flow、material-issue-flow、stocktaking-flow、e2e-sales-to-receivable-flow、event-bus/xautoclaim-redelivery-simulation。
- **Playwright E2E**：`tests/auth.spec.ts`、`tests/dashboard.spec.ts`、`tests/inbound.spec.ts`、`tests/login.spec.ts`、`tests/sales-orders.spec.ts`、`tests/page-speed.spec.ts`、`tests/swagger-ui.spec.ts`、`tests/swagger-ui-diagnostic.spec.ts`。
- **并发场景**：`tests/concurrency/` 提供 `inventory-outbound`、`material-issue`、`purchase-inbound`、`stocktaking-approve` 四类并发测试与共享工具（`setup.ts`、`utils.ts`），报告输出到 `tests/concurrency/reports/`。
