# 全链路测试方案

> 编号：VNERP-TEST-MASTER | 版本：V2.0

## 一、测试总原则

| 原则 | 说明 |
|------|------|
| 全覆盖 | 覆盖每个框架、录入框、逻辑、计算、环节 |
| 自动化优先 | 可自动化的测试必须编写自动化用例 |
| 数据驱动 | 用正常、边界、异常多组数据验证同一功能 |
| 可重复 | 每次执行得到一致结果 |

## 二、框架级测试

| 测试项 | 验证方法 | 预期结果 |
|--------|----------|----------|
| 页面路由注册 | Playwright 访问每个页面 | 返回 200，无 404 |
| API 路由注册 | 自动化 GET 请求 | 所有 API 返回合法响应 |
| 路由守卫 | E2E 未登录访问 | 重定向登录页 |
| 数据库连接池 | 单元测试 | connectionLimit 正确，连接正常释放 |
| 事务管理 | 集成测试 | commit 后持久化，rollback 后不变 |
| 表单验证 | 手动 + E2E | 必填、类型、长度、格式、范围均拦截非法输入 |
| 权限框架 | E2E 多角色登录 | 菜单、按钮、API 按角色生效；Token 过期重登录 |
| 日志框架 | 手动 + 看日志 | 操作、错误、登录日志齐全 |

## 三、页面与录入框级测试

通用：页面加载无 JS 错误；按钮可点击；输入类型校验；最大长度；必填提示；下拉/日期/分页/搜索/响应式。

| 模块 | 关键测试点 |
|------|-----------|
| 生产 | 工单创建/编辑/状态流转；报工-印刷/模切；报工数量上限；标准卡创建与关联 |
| 仓库 | 采购入库、生产入库、领料出库、物料拆分、盘点创建/执行/审核、调拨、二维码追溯 |
| 采购 | 申请、审批、订单、确认、入库 |
| 品质 | IQC/IPQC/FQC 检验单与判定 |
| 系统 | 用户、角色、密码、禁用、菜单管理 |

## 七、异常与边界

| 编号 | 场景 | 预期 |
|------|------|------|
| ERR-001 | 并发创建同编号 | 一个成功，一个 409 |
| ERR-002 | 网络中断 | 错误提示，数据不丢失 |
| ERR-003 | 数据库连接失败 | 返回 500，不崩溃 |
| ERR-004 | 数量 999999999 | 正常处理或提示超范围 |
| ERR-005 | 数量 -1 | 拦截，必须为正数 |
| ERR-006 | 数量 0 | 拦截，必须大于 0 |
| ERR-007 | SQL 注入文本 | 参数化查询防护 |
| ERR-008 | `<script>` 文本 | HTML 转义，不执行 |
