## 5. 测试过程中发现的 Bug

### Bug #1: `convertToSalesOrder()` 未写入关联字段 (已修复)

- **严重级别**: P0
- **现象**: 调用 `convertToSalesOrder()` 后，`salesOrderId`、`convertedAt`、`convertedBy` 均为空，`feeDeducted` 未标记
- **根因**: 方法仅设置 `_status` 和推送事件，未写入业务字段；`convertedAt`/`convertedBy` 原为 `public readonly` 不可变
- **修复**: 改为 `private` 可变字段 + 公共 getter，`convertToSalesOrder()` 内设置 `_salesOrderId`、`_convertedAt`(ISO时间戳)、`_convertedBy`，并在 `feeCharged && feeDeductible` 时标记 `_feeDeducted = 1`
- **验证**: 单元测试 #23-#27 + E2E 测试 #5 组全部通过

### Bug #2: `cancel()` 未拦截 CONFIRMED 状态 (已修复)

- **严重级别**: P1
- **现象**: 已确认状态的打样单可被作废，与状态机配置不一致
- **根因**: `cancel()` 硬编码仅拦截 `CONVERTED` 和 `CANCELLED`，未使用 `canTransition()` 统一校验
- **修复**: 改为 `if (!canTransition(this._status, SampleOrderStatus.CANCELLED))` 与其他流转方法保持一致
- **验证**: 单元测试 #22 通过
