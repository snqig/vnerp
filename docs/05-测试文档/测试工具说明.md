## 二、Vitest 配置要点

文件：`vitest.config.ts`（仓库实际内容）

```ts
import { defineConfig } from 'vitest/config';
import { resolve } from 'path';

export default defineConfig({
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src'),
    },
  },
  test: {
    globals: true,
    environment: 'node',
    include: ['tests/**/*.test.ts'],
    setupFiles: ['tests/setup-env.ts'],
  },
});
```

要点（以实际文件为准）：

- `environment: 'node'`，不依赖 DOM；如需测试 React 组件，需将 environment 改为 `'jsdom'` 并补充 `@testing-library/jest-dom` setup。
- `include: ['tests/**/*.test.ts']`：仅匹配 `tests/` 下的 `.test.ts`，与 Playwright 的 `**/*.spec.ts` 自然隔离，无需 `exclude`。
- `setupFiles: ['tests/setup-env.ts']`：通过 `process.loadEnvFile()` 加载 `.env`，集成测试依赖 `DB_PASSWORD` / `REDIS_URL`，缺失会报 `Access denied (using password: NO)`。
- 别名 `@` 指向 `./src`，与生产代码保持一致。
- **未配置覆盖率阈值**：当前 `vitest.config.ts` 没有 `coverage` 段。如需启用覆盖率采集与渐进式阈值，需补充：

  ```ts
  coverage: {
    provider: 'v8',
    reporter: ['text', 'json', 'html', 'lcov'],
    include: ['src/lib/**/*.ts', 'src/domain/**/*.ts', 'src/infrastructure/event-bus/**/*.ts'],
    // thresholds: { lines: 45, functions: 70, branches: 35, statements: 45 },
  }
  ```

> 注：早期文档曾记录 `environment: 'jsdom'`、`exclude: ['tests/**/*.spec.ts']`、`testTimeout: 30000`、`thresholds: { lines: 45, ... }` 等内容，那是计划态而非 `vitest.config.ts` 实际内容。本节以实际文件为准。

## 三、Playwright 配置要点

文件：`playwright.config.ts`

- `testDir: ./tests`，`testMatch: '**/*.spec.ts'`
- `globalSetup: './tests/global-setup.ts'`
- `fullyParallel: true`，CI 下 `workers: 1`，`forbidOnly: true`，`retries: 2`
- 单测超时 120000ms（webpack 冷启动编译较慢）
- 浏览器矩阵：chromium / firefox / webkit / Mobile Chrome / Mobile Safari
- `webServer.command: 'pnpm run dev:webpack'`，`url: 'http://localhost:5000'`，`reuseExistingServer: !CI`，`timeout: 180000`
  - 使用 `--webpack` 模式启动开发服务器，规避 Windows 下 Turbopack 处理 CSS 时读取保留设备名 `nul` 导致的崩溃问题。
- 视口 1920×1080，`actionTimeout: 30000`，`navigationTimeout: 60000`
- `trace: 'on-first-retry'`，`screenshot: 'only-on-failure'`，`video: 'on-first-retry'`
- 报告：HTML（`playwright-report/`）+ list + JSON（`test-results.json`）

## 四、常用命令

```bash
# 单元 / 集成（Vitest）
pnpm test:unit              # 监听
pnpm test:unit:run          # 单次
pnpm test:coverage          # 覆盖率（需先在 vitest.config.ts 启用 coverage 段）

# E2E（Playwright）
pnpm test                   # 全量
pnpm test:headed            # 有头
pnpm test:ui                # UI 模式
pnpm test:debug             # 调试
pnpm test:report            # 报告
pnpm test:install           # 安装浏览器

# 质量
pnpm ts-check
pnpm lint
pnpm lint:fix
pnpm format
pnpm format:check
```

> 完整命令定义见 `package.json` 的 `scripts` 段。注意 `db:generate` / `db:push` / `db:migrate` 已废弃，打印警告；数据库初始化与迁移用 `pnpm setup:db`（全量）或 `pnpm migrate`（增量）。

## 六、测试数据

- 独立测试数据库（建议 `vnerp_test`，通过 `.env` 配置），禁止在生产数据库执行测试。
- 测试数据 SQL 位于 `docs/05-测试文档/测试数据/`：`base-data.sql`、`production-data.sql`、`warehouse-data.sql`。
- 集成测试在 `tests/setup-env.ts` 中加载 `.env`；具体夹具由各测试文件自行准备或通过 `tests/global-setup.ts`（Playwright）注入。
- 数据库初始化脚本：`scripts/setup-db.mjs`（建库 + 导入 schema + 可选 seed）；增量迁移 `scripts/migrate.ts`。

## 七、CI 集成

- `.github/workflows/ci.yml` 在 PR 与 push 时执行：`pnpm install` → `pnpm ts-check` → `pnpm lint` → `pnpm test:coverage` → `pnpm test`。
- CI 中 Playwright 单 worker、重试 2 次、`forbidOnly`。
- pre-push Husky 钩子执行单元测试（参见 commit `edb0906`）。
