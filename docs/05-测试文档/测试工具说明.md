> 描述 VNERP 项目测试工具链、配置文件、命令与产出物。

## 二、Vitest 配置要点

文件：`vitest.config.ts`

```ts
{
  plugins: [react()],
  test: {
    environment: 'jsdom',
    globals: true,
    setupFiles: ['./src/tests/setup.ts'],
    include: ['src/**/*.{test,spec}.{ts,tsx}', 'tests/**/*.{test,spec}.{ts,tsx}'],
    exclude: [...configDefaults.exclude, 'tests/**/*.spec.ts'],
    coverage: {
      provider: 'v8',
      reporter: ['text', 'json', 'html', 'lcov'],
      reportOnFailure: true,
      thresholds: { lines: 45, functions: 70, branches: 35, statements: 45 },
      include: [/* lib 核心算法、领域层、事件总线基础设施 */],
    },
    testTimeout: 30000,
    hookTimeout: 30000,
  },
  resolve: { alias: { '@': path.resolve(__dirname, './src') } },
}
```

要点：
- 显式排除 `tests/**/*.spec.ts`，避免 Vitest 把 Playwright 用例当成 Vitest 用例运行报错。
- 覆盖率仅统计核心文件，防止业务样板代码稀释指标。
- 别名 `@` 指向 `./src`，与生产代码保持一致。

## 三、Playwright 配置要点

文件：`playwright.config.ts`

- `testDir: ./tests`，`testMatch: '**/*.spec.ts'`
- `globalSetup: ./tests/global-setup.ts`
- `fullyParallel: true`，CI 下 `workers: 1`，`forbidOnly: true`，`retries: 2`
- 浏览器矩阵：chromium / firefox / webkit / Mobile Chrome / Mobile Safari
- `webServer.command: 'pnpm run dev'`，`url: 'http://localhost:5000'`，`reuseExistingServer: !CI`
- 视口 1920×1080，`trace: 'on-first-retry'`，`screenshot: 'only-on-failure'`，`video: 'on-first-retry'`
- 报告：HTML（`playwright-report/`）+ list + JSON（`test-results.json`）

## 四、常用命令

```bash
# 单元 / 集成
pnpm test:unit              # 监听
pnpm test:unit:run          # 单次
pnpm test:coverage          # 覆盖率

# E2E
pnpm test                   # 全量
pnpm test:headed            # 有头
pnpm test:ui                # UI 模式
pnpm test:debug             # 调试
pnpm test:report            # 报告
pnpm test:install           # 安装浏览器

# 质量
pnpm ts-check
pnpm lint
pnpm lint:fix
pnpm format
pnpm format:check
```

## 五、产出物

| 产出 | 路径 | 触发命令 |
|------|------|----------|
| 覆盖率 HTML 报告 | `coverage/index.html` | `test:coverage` |
| 覆盖率 LCOV | `coverage/lcov.info` | `test:coverage` |
| Playwright HTML 报告 | `playwright-report/index.html` | `test` |
| Playwright JSON 结果 | `test-results.json` | `test` |
| Playwright 跟踪 | `test-results/` | 失败重试时 |
| 类型检查输出 | 控制台 | `ts-check` |
| Lint 输出 | 控制台 | `lint` |

## 六、测试数据

- 独立测试数据库 `vnerp_test`（通过环境变量配置），禁止在生产数据库执行测试。
- 测试数据 SQL 位于 `docs/05-测试文档/测试数据/` 与 `database/seeds/`。
- 通过 `scripts/seed-*.js` 与 `database/seed.js` 重置。

## 七、CI 集成

- `.github/workflows/ci.yml` 在 PR 与 push 时执行：`pnpm install` → `pnpm ts-check` → `pnpm lint` → `pnpm test:coverage` → `pnpm test`。
- 覆盖率不达阈值则构建失败。
- CI 中 Playwright 单 worker、重试 2 次、`forbidOnly`。
