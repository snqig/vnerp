# 测试用例模板

> 提供单元测试、E2E 测试、验收用例三类标准模板，复制即用。

## 一、单元测试模板（Vitest）

```typescript
/**
 * 模块：BOM 展开
 * 文件：src/lib/__tests__/bom-expansion.test.ts
 */
import { describe, it, expect, beforeEach, vi } from 'vitest';
import { expandBom } from '@/lib/bom-expansion';

// 被测对象依赖时通过 vi.mock 替换
vi.mock('@/lib/db', () => ({
  query: vi.fn(),
  execute: vi.fn(),
  transaction: vi.fn((fn) => fn({})),
}));

describe('BOM 展开计算', () => {
  beforeEach(() => {
    vi.clearAllMocks();
  });

  it('应该正确展开单层 BOM 并加上损耗率', async () => {
    // Arrange
    const productId = 1;
    const quantity = 10;

    // Act
    const result = await expandBom(productId, quantity);

    // Assert
    expect(result.items).toHaveLength(2);
    expect(result.items[0].actualQuantity).toBeCloseTo(21); // 20 * (1 + 5/100)
  });

  it('应该检测循环引用并发出警告', async () => {
    const result = await expandBom(1, 10);
    expect(result.circularReferenceWarnings).toBeDefined();
  });
});
```

### 模板要素

| 字段 | 说明 |
|------|------|
| 模块 | 被测模块名 |
| 文件 | 测试文件相对路径 |
| Arrange | 准备数据与 Mock |
| Act | 调用被测函数 |
| Assert | 明确断言，避免 `toBeTruthy()` |

## 二、E2E 测试模板（Playwright）

```typescript
/**
 * 模块：登录流程
 * 文件：tests/login.spec.ts
 */
import { test, expect } from '@playwright/test';

test.describe('登录流程', () => {
  test('使用合法凭据登录后跳转看板', async ({ page }) => {
    await page.goto('/zh-CN/login');

    await page.fill('input[name="username"]', 'admin');
    await page.fill('input[name="password"]', 'admin123');
    await page.click('button[type="submit"]');

    await expect(page).toHaveURL(/\/dashboard/);
    await expect(page.locator('header')).toContainText('首页');
  });

  test('密码错误时显示错误提示', async ({ page }) => {
    await page.goto('/zh-CN/login');
    await page.fill('input[name="username"]', 'admin');
    await page.fill('input[name="password"]', 'wrong');
    await page.click('button[type="submit"]');

    await expect(page.locator('[role="alert"]')).toBeVisible();
  });
});
```

### E2E 模板要素

| 字段 | 说明 |
|------|------|
| `test.describe` | 用模块名分组 |
| `baseURL` | 已在 `playwright.config.ts` 配置，`goto` 用相对路径 |
| 断言 | 优先使用 `toHaveURL`、`toContainText`、`toBeVisible` |
| 清理 | 测试间状态由 `global-setup.ts` 统一重置 |

## 四、示例：FIFO 批次推荐

| 项目 | 内容 |
|------|------|
| **用例编号** | TC-WH-005 |
| **用例名称** | 系统按入库时间升序推荐批次 |
| **测试目的** | 验证 FIFO 推荐算法 |
| **优先级** | P0 |
| **前置条件** | 存在 3 批同种物料：批次 A（5/1，50）、批次 B（5/5，30）、批次 C（5/10，20）；FIFO 已开启 |
| **测试步骤** | 1. 创建领料单 MR002，需求 60<br>2. 系统自动推荐批次<br>3. 检查推荐顺序与数量 |
| **预期结果** | 1. 推荐顺序：A → B → C<br>2. 分配：A(50) + B(10) = 60<br>3. 不允许手动选其他批次（除非例外审批） |
| **测试状态** | □ 通过 □ 失败 □ 阻塞 |
