## 二、全链路测试总原则

| 原则 | 说明 |
|------|------|
| 全覆盖 | 覆盖每一个框架、录入框、逻辑、计算、环节 |
| 自动化优先 | 可自动化的测试必须编写自动化用例 |
| 数据驱动 | 用正常、边界、异常多组数据验证同一功能 |
| 可重复 | 每次执行得到一致结果 |

## 三、测试分层要点

1. **框架级**：路由、ORM、表单验证、权限、日志框架均需被验证可用。
2. **页面级**：所有页面无 JS 错误；按钮、录入框、下拉、日期选择器、分页、搜索、响应式。
3. **逻辑计算级**：物料需求、工时、完工率、库存金额（FIFO）、应收应付、毛利。
4. **全流程级**：销售 → 生产 → 仓库 → 发货 → 财务 完整闭环；二维码全链路追溯。
5. **异常与边界**：并发同编号、网络中断、超大数量、负数、零值、SQL 注入、XSS。

## 四、关键业务规则速记

- **FIFO**：优先推荐余料 → 按入库时间升序 → 最早入库先出；非 FIFO 批次需例外审批。
- **拆分**：整料按标准单位拆分，余料 `split_flag=2`，标准小料 `split_flag=1`，整料 `split_flag=0`。
- **整料锁定**：未拆分整料禁止直接领用，必须先拆分。
- **超领**：超过计划量需走审批流程，可配置上限。
- **无单发料**：默认禁止，配置 `allow_no_order_issue` 可放开。
- **跳工序**：默认禁止，配置 `allow_skip_process` 可放开。
- **FQC 必检**：成品入库前必须 FQC，配置 `final_qc_required` 控制。
- **寿命管理**：刀模 90 天 / 5000 次；网版 60 天 / 3000 次；油墨开盖 30 天；混合油墨 24 小时。

## 五、三账一致性

每个物料在以下三处的数量必须一致：

- `inv_inventory`（总账）
- `inv_inventory_batch`（批次账）
- `warehouse_location_card`（货位卡）

差异为 0 或仅在四舍五入误差内，差异需有 reconciliation 报告说明原因。
