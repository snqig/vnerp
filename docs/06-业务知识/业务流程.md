# 业务流程

本文档描述 印刷生产经营信息管理系统 Print MIS 印刷 ERP 的核心业务流程，覆盖采购入库、生产领用、生产报工、发货出库、盘点调拨、财务核算全链路。

## 1. 采购入库流程

```
供应商报价 → 创建采购订单(pur_order)
   ↓ 到货验收
创建入库单(inv_inbound_order, status=draft)
   ↓ submit (draft→pending)
   ↓ audit (pending→completed)
   ↓ 事务内：
   ↓   - 写入 inv_inventory_batch（新批次，含 expire_date/version）
   ↓   - 累加 inv_inventory.quantity
   ↓   - 生成 qrcode_record（物料二维码 MA-前缀）
   ↓   - 写入 inv_inventory_transaction（type=inbound）
   ↓   - 生成 fin_payable（应付单，status=pending）
   ↓   - 记录 material_batch_costs（批次成本）
```

## 2. 生产领用流程

```
工单创建(prd_work_order, 关联标准卡)
   ↓ autoGenerateRequisition（按 BOM 生成正常领用单）
领用单(material_requisition, type=normal, status=draft)
   ↓ submit (draft→pending)
   ↓ approve (pending→approved)
   ↓ issueMaterial 发料
   ↓   - 调用 allocateFIFO 推荐批次
   ↓   - 仓管员确认或覆盖（覆盖需 logFIFOOverride）
   ↓   - 扣减 inv_inventory_batch.available_qty
   ↓   - 累加 inv_inventory_batch.reserved_qty
   ↓   - 写入 inv_inventory_transaction（type=outbound）
   ↓   - status=issued
   ↓   - 归集 work_order_costs（cost_type=material）
```

补充/超额领用：
- 补充领用：生产中发现不足，`submitSupplementaryRequisition`
- 超额领用：超出标准 BOM，`submitOverRequisition`，需说明超耗原因

## 3. 生产报工流程

```
工序开始 → 扫码(qrcode)确认物料
   ↓ 完成工序
报工(prd_production_report)
   ↓ 记录工时、产量、质量结果
   ↓ 归集 work_order_costs（cost_type=labor）
   ↓ 工时 × 时薪 = 人工成本
全部工序完成 → 工单状态=completed
```

## 4. 发货出库流程

```
销售订单(sal_order) → 创建发货单(shipments, status=draft)
   ↓ submit (draft→pending_review)
   ↓ audit (pending_review→approved)
   ↓ start_picking (approved→picking)
   ↓ 扫码拣货（扫描成品二维码）
   ↓   - 校验 qrcode_record.status=1
   ↓   - 校验 product_id 匹配
   ↓   - 写入 shipment_items（scanned=1）
   ↓ complete_picking (picking→picked)
   ↓ ship (picked→shipped)
   ↓   - 扣减 wh_inventory.quantity
   ↓   - 更新 qrcode_record.status=2
   ↓   - 写入 inv_inventory_log（type=outbound）
   ↓   - 更新 sal_order 发货状态
   ↓   - 生成 fin_receivable（应收单）
   ↓ start_transit (shipped→in_transit)
   ↓ deliver (in_transit→delivered)
```

部分发货：`picked → partial_ship`，剩余明细保持 picked 可继续发货。

## 5. 盘点流程

```
创建盘点单(inv_stocktaking, status=1)
   ↓ 锁定仓库（inv_inventory.stocktaking_flag=1）
   ↓ 自动生成明细（system_qty 来自 inv_inventory）
扫码盘点（POST /scan）
   ↓ 录入 actual_qty，计算 diff_qty
submit (status=1→2)
   ↓ 校验所有明细已盘点
approve (status=2→3)
   ↓ 事务内：
   ↓   - 校验负库存（quantity + diff_qty >= 0）
   ↓   - 更新 inv_inventory.quantity
   ↓   - 写入 inv_inventory_log（change_type=STOCKTAKING）
   ↓   - 解锁仓库（stocktaking_flag=0）
```

## 6. 调拨流程

```
创建调拨单(status=draft)
   ↓ submit (draft→pending)
   ↓ confirm (pending→completed)
   ↓ 事务内：
   ↓   - 源仓库 inv_inventory.quantity -= 数量
   ↓   - 源仓库批次 available_qty -= 数量
   ↓   - 目标仓库 inv_inventory.quantity += 数量
   ↓   - 目标仓库新建/更新批次
   ↓   - 写入两条流水（transfer_out / transfer_in）
```

## 7. 财务核算流程

### 7.1 应收

```
发货完成 → 自动生成 fin_receivable（status=pending）
   ↓ 客户付款
   ↓ recordReceipt → fin_receipt_record
   ↓ 更新 received_amount
   ↓ status: pending → partial → received
```

### 7.2 应付

```
采购入库完成 → 自动生成 fin_payable（status=pending）
   ↓ 向供应商付款
   ↓ recordPayment → fin_payment_record
   ↓ 更新 paid_amount
   ↓ status: pending → partial → paid
```

### 7.3 工单成本

```
物料领用 → work_order_costs (material)
生产报工 → work_order_costs (labor)
外协确认 → work_order_costs (outsource)
制造费用分摊 → work_order_costs (overhead)
   ↓
calculateWorkOrderCost(workOrderId)
   ↓ 汇总四类成本
   ↓ 工单总成本 = material + labor + overhead + outsource
```

## 8. 退货流程

```
发货退货（in_transit → returned）
   ↓ 仓管员验收
   ↓   - wh_inventory.quantity 回退
   ↓   - qrcode_record.status 恢复为 1
   ↓   - fin_receivable.status → disputed 或冲销
```

## 9. 二维码追溯链路

```
采购入库 → 生成物料二维码(MA-)
   ↓ 生产领用扫码 → qrcode_scan_log
   ↓ 小料拆分 → 子二维码(MA-001-S1)，parent_qr_code 关联
   ↓ 工序扫码 → 关联 prd_work_order
   ↓ 成品入库 → 生成成品二维码(PR-)
   ↓ 发货扫码 → qrcode_record.status=2
   ↓ 全链路追溯查询 /api/qrcode/trace
```

## 10. 业务驱动财务原则

| 业务事件 | 财务单据 | 状态 |
|---|---|---|
| 采购入库完成 | `fin_payable` | pending |
| 发货完成 | `fin_receivable` | pending |
| 物料领用发料 | `work_order_costs` | - |
| 生产报工 | `work_order_costs` | - |
| 客户收款 | `fin_receipt_record` | - |
| 供应商付款 | `fin_payment_record` | - |

所有财务单据由业务事件自动生成，禁止手工录入应收/应付。
