# 技术决策记录：二维码实现方案

> 编号：VNERP-DEC-002 | 日期：2026-05-10 | 状态：已决定

## 背景

丝网印刷 ERP 需要实现从原材料入库到成品出库的全流程二维码追溯。需要决定二维码的实现方案。

## 选项

### 选项 A：统一 qrcode_record 表

- 优势：数据统一、查询简单、维护方便
- 劣势：表结构需支持多种类型

### 选项 B：分表存储（qr_codes + qrcode_record）

- 优势：各表职责单一
- 劣势：数据不互通、查询复杂、维护困难

### 选项 C：使用现有开源追溯框架

- 优势：功能完善
- 劣势：学习成本高、定制困难

## 决策

选择 **选项 A：统一 qrcode_record 表**。

## 理由

1. 统一表结构简化查询逻辑
2. 通过 `qr_type` 字段区分类型
3. 通过 `split_flag` 和 `parent_qr_id` 支持拆分追溯
4. 避免了之前双表方案的数据不互通问题（ARCH-002 已修复）
5. 单表方案更易维护和扩展

## 影响

- qrcode_record 表需要包含所有类型共有的字段
- 拆分功能通过 split_flag + parent_qr_id 实现
- 追溯查询通过 parent_qr_id 递归实现
