# 数据库表关系

> 基于 `src/lib/db/schema.ts` 中 Drizzle ORM 定义整理。

## 一、概述

印刷生产经营信息管理系统 Print MIS 数据库 `vnerpdacahng`（MySQL 8.0），核心业务表通过 Drizzle ORM 维护类型化外键，应用层保证一致性。`schema.ts` 共定义 41 张表，覆盖主数据、仓库、订单、生产、品质、采购、委外、派送、打样、设备保养与 dcprint 迁移功能。

## 三、关键字段约束

- 所有主键为 `serial`（BIGINT UNSIGNED AUTO_INCREMENT）。
- 业务编码字段（如 `code`、`order_no`、`batch_no`、`qr_code`）均带 `unique` 约束。
- 外键关系通过 `.references()` 显式声明，但生产数据库本身不强制外键约束（应用层保证一致性，便于分库与迁移）。
- 状态字段统一使用字符串（如 `'draft'`、`'pending'`、`'completed'`、`'cancelled'`）。
- 软删除通过 `status` 或 `deleted` 字段实现（如 `invInboundOrders.deleted`）。

## 四、二维码追溯链

```
materialLabels (parentLabelId)
   │
   ├── cuttingRecords ──< cuttingDetails >── materialLabels (newLabelId)
   │
   ├── processCardMaterials >── processCards
   │
   └── traceRecords >── processCards
```

- `materialLabels.parentLabelId` 指向被分切的源标签，形成自引用父子链。
- `processCardMaterials` 关联流程卡与物料标签（`main` / `auxiliary`）。
- `traceRecords` 记录正向 / 反向追溯（`trace_type: forward/backward`）。
