# 数据库表关系

> 基于 `src/lib/db/schema.ts` 中 Drizzle ORM 定义整理。关系基数与外键语义通过索引名（`fk_*` / `idx_*`）推断，生产数据库不强制外键约束，一致性由应用层保证。

## 一、概述

印刷生产经营信息管理系统 Print MIS 数据库 `vnerpdacahng`（MySQL 8.0），`src/lib/db/schema.ts` 共定义 42 张 Drizzle 消费表，覆盖仓库、销售、采购、财务、生产、打样工艺模板、印前七大业务域。各表通过业务 ID 字段（如 `order_id`、`material_id`、`work_order_id`）建立逻辑关联，Drizzle schema 中以索引名前缀 `fk_` 标注显式外键语义，`idx_` 标注普通查询索引。

遗留表（`sys_user`、`material`、`customer`、`supplier`、`bom`、`equipment` 等）经原始 SQL 管理，与 Drizzle 消费表存在跨层引用（如 `inv_inventory.material_id` → `material.id`），本文档仅描述 Drizzle 消费表内部及与关键遗留表的关系。

## 二、关系标注约定

- `A 1 ──< B`：A 的一条记录对应 B 的多条记录（一对多）。
- `A >── 1 B`：B 的一条记录对应 A 的多条记录（多对一，A 引用 B）。
- 索引名 `fk_<表>_<引用表>` 表示该字段为外键语义引用；`uk_<字段>` 表示唯一约束；`idx_<字段>` 表示普通索引。

## 十一、关键字段约束

- 所有主键为 `serial`（`BIGINT UNSIGNED AUTO_INCREMENT`）或 `bigint unsigned autoincrement`。
- 业务编码字段（如 `order_no`、`po_no`、`delivery_no`、`work_order_no`、`tool_code`、`color_code`、`template_no`）均带 `unique` 约束（`uk_*`）。
- 外键关系通过索引名（`fk_*`）声明语义，但生产数据库本身不强制外键约束（应用层保证一致性，便于分库与迁移）。
- 复合唯一约束：`inv_inventory`（`material_id` + `warehouse_id`）、`pur_purchase_order_line`（`po_id` + `line_no`）、`dcprint_ink_formula_version`（`color_id` + `version_no`）。
- 状态字段：部分表使用 `tinyint`（如 `sal_order.status`、`prd_work_order.status`），部分使用 `varchar`（如 `inv_inbound_order.status`、`prod_work_order.status`），迁移 060 正在统一。
- 软删除通过 `deleted` 字段实现（`boolean` 或 `tinyint`）。
