# 数据库架构

> 基于 `drizzle.config.ts` 与 `src/lib/db/schema.ts` 现状整理。

## 一、技术栈

| 维度 | 选型 |
|------|------|
| 数据库 | MySQL 8.0（`utf8mb4`） |
| ORM | Drizzle ORM 0.45（`drizzle-orm/mysql2`） |
| 迁移工具 | Drizzle Kit 0.31（`pnpm db:generate` / `db:push` / `db:migrate` / `db:studio`） |
| 驱动 | mysql2 3.20（Promise API + 连接池） |
| 默认数据库 | `vnerpdacahng` |
| 配置入口 | `drizzle.config.ts`、`src/lib/db/index.ts` |

## 二、设计原则

1. 表命名使用模块前缀（`prd_`、`inv_`、`fin_`、`sys_` 等），dcprint 迁移表保留原名。
2. 字段命名使用 `snake_case`，TypeScript 导出使用 `camelCase`。
3. 状态使用字符串编码（如 `'draft'`、`'pending'`、`'completed'`、`'cancelled'`）。
4. 软删除优先（`status` 或 `deleted` 字段），不物理删除。
5. 不强制外键约束（应用层保证一致性），但 Drizzle schema 通过 `.references()` 声明类型化关系，便于类型推导。
6. 不使用存储过程和触发器。

## 三、连接与查询

`src/lib/db/index.ts` 提供：

- `getPool()`：全局单例连接池（`connectionLimit: 20`、`maxIdle: 10`、`idleTimeout: 30000`、`queueLimit: 100`、`connectTimeout: 8000`）。
- `query<T>()` / `execute()` / `queryOne<T>()`：基于 `mysql2` 参数化预编译语句，防 SQL 注入。
- `transaction()`：手动事务；`transactionWithRetry()`：乐观锁冲突自动重试（指数退避，默认 3 次）。
- `queryPaginated<T>()`：分页查询辅助。
- `drizzleDb`：Drizzle ORM 实例，加载 `schema` 提供类型安全查询构建器；新代码优先使用，旧代码继续用 `query/execute`。

配置通过 `DB_HOST` / `DB_PORT` / `DB_USER` / `DB_PASSWORD` / `DB_NAME` 环境变量注入；`DEBUG_DB=true` 或 `NODE_ENV=development` 时输出 SQL 调试日志。

## 四、Drizzle Kit 配置

文件：`drizzle.config.ts`

```ts
defineConfig({
  schema: './src/lib/db/schema.ts',
  out: './drizzle',
  dialect: 'mysql',
  dbCredentials: {
    host: process.env.DB_HOST || '127.0.0.1',
    port: Number(process.env.DB_PORT || 3306),
    user: process.env.DB_USER || 'root',
    password: process.env.DB_PASSWORD || '',
    database: process.env.DB_NAME || 'vnerpdacahng',
  },
  verbose: true,
  strict: true,
});
```

迁移产物位于 `drizzle/`：

- `0000_chilly_retro_girl.sql`：初始迁移
- `0001_ordinary_valkyrie.sql`：增量迁移（仓库入库 5 个核心查询示例）
- `meta/`：迁移日志与快照（`_journal.json`、`0000_snapshot.json`、`0001_snapshot.json`）

## 五、核心表清单

### 主数据

| 表名 | 说明 |
|------|------|
| `customers` | 客户档案（含信用额度） |
| `products` | 产品档案（含 BOM 版本、所属客户） |
| `materials` | 物料（原料/辅料，含安全库存） |
| `suppliers` | 供应商 |
| `equipments` | 设备档案（含保养周期） |
| `employees` | 员工档案 |

### 仓库

| 表名 | 说明 |
|------|------|
| `warehouses` | 仓库（四仓分离：原料/成品/板房/油墨） |
| `locations` | 库位 |
| `inventory_batches` | 批次库存（含 QR、预警级别、检验/隔离状态） |
| `inventory_transactions` | 库存事务记录 |
| `inv_inbound_order` / `inv_inbound_item` | 入库单主从表（实际表，验证 Drizzle 迁移） |

### 订单与生产

| 表名 | 说明 |
|------|------|
| `sales_orders` / `sales_order_items` | 销售订单 |
| `boms` / `bom_items` | BOM 配方 |
| `work_orders` | 生产工单 |
| `processes` | 工序路线 |
| `work_order_processes` | 工单工序 |
| `production_reports` | 生产报工 |

### 品质 / 采购 / 委外

| 表名 | 说明 |
|------|------|
| `inspection_standards` / `inspection_records` / `defect_records` | 检验标准、记录、不良品 |
| `purchase_requests` / `purchase_request_items` | 采购申请 |
| `purchase_orders` / `purchase_order_items` | 采购订单 |
| `outsource_orders` | 委外工单 |

### 派送 / 打样 / 保养

| 表名 | 说明 |
|------|------|
| `vehicles` / `delivery_orders` / `delivery_items` / `delivery_receipts` | 车辆与派送 |
| `sample_requests` | 打样申请 |
| `maintenance_plans` | 保养计划 |

### dcprint 迁移模块

| 表名 | 说明 |
|------|------|
| `material_labels` | 物料标签（含分切父子关系、二维码、宽幅、长度） |
| `cutting_records` / `cutting_details` | 分切记录与明细 |
| `process_cards` / `process_card_materials` | 生产流程卡与物料关联 |
| `trace_records` / `scan_logs` | 物料追溯记录与扫码日志 |

## 六、命令速查

```bash
pnpm db:generate    # 基于 schema 变更生成迁移
pnpm db:push        # 直接推送 schema 到数据库（开发用）
pnpm db:migrate     # 执行 drizzle/ 下迁移文件
pnpm db:studio      # Drizzle Studio 可视化
pnpm setup:db       # 一键初始化（scripts/setup-db.mjs）
```
