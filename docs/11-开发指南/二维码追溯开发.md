# 二维码追溯开发

本文档说明 VNERP 二维码追溯模块的开发指南，对应 `src/lib/qrcode-service.ts`、`src/app/api/qrcode/`、`src/app/[locale]/qrcode/page.tsx` 与 `database/qrcode_and_print_template.sql`。

## 1. 模块概览

二维码追溯模块用于物料、批次、工单的全流程追踪，覆盖生成、打印、扫描、追溯、失效等场景。

| 类型 | qr_type | 用途 |
|------|---------|------|
| 物料二维码 | `material` | 单个物料标识 |
| 批次二维码 | `batch` | 批次级追溯 |
| 工单二维码 | `workorder` | 工单级追溯 |
| 产品二维码 | `product` | 成品追溯 |
| 入库二维码 | `inbound` | 入库单追溯 |
| 出库二维码 | `outbound` | 出库单追溯 |

## 2. 关键源码位置

| 文件 | 作用 |
|------|------|
| `src/lib/qrcode-service.ts` | 二维码生成、解析、追溯核心服务（`QRCodeService` 类） |
| `src/app/api/qrcode/route.ts` | 二维码 CRUD 与操作 API |

| `database/qrcode_and_print_template.sql` | 二维码与打印模板表 DDL |

## 3. 二维码内容格式

`QRCodeService.buildQRContent` 生成 JSON 字符串作为二维码内容：

```json
{
  "v": "1",
  "t": "material",
  "id": 123,
  "c": "MAT1231700000000000",
  "mc": "MAT001",
  "b": "BATCH2026-001",
  "w": "WO-2026-001",
  "pd": "2026-06-30",
  "ed": "2027-06-30"
}
```

| 字段 | 含义 |
|------|------|
| `v` | 版本号，当前为 `1` |
| `t` | 类型（material / batch / workorder 等） |
| `id` | 关联业务对象 ID |
| `c` | 二维码编号 |
| `mc` | 物料编码 |
| `b` | 批次号 |
| `w` | 工单号 |
| `pd` | 生产日期 |
| `ed` | 过期日期 |

`parseQRContent(content)` 反向解析，返回 `QRCodeData | null`。

## 4. 生成二维码

### 4.1 物料二维码

```ts
import { QRCodeService } from '@/lib/qrcode-service';

const { qrCode, qrImageUrl, qrId } = await QRCodeService.generateMaterialQRCode(
  materialId,
  materialCode,
  materialName,
  {
    warehouseId: 1,
    productionDate: '2026-06-30',
    expireDate: '2027-06-30',
  }
);
```

内部流程：

1. 在 `inv_qr_code` 表插入记录（含 `trace_url`）。
2. 调用 `buildQRContent` 生成 JSON 内容。
3. 调用 `generateQRCodeImage` 生成 base64 图片（`qrcode` 库，宽 200px，纠错级别 M）。
4. 更新 `qr_image_url` 字段。
5. 全程在 `transaction` 中，失败回滚。

### 4.2 批次二维码

```ts
const result = await QRCodeService.generateBatchQRCode(
  batchNo, materialId, materialCode, materialName,
  { warehouseId: 1, quantity: 1000 }
);
```

### 4.3 工单二维码

```ts
const result = await QRCodeService.generateWorkOrderQRCode(
  workOrderId, workOrderNo, productName,
  { plannedQty: 5000, plannedEndDate: '2026-07-31' }
);
```

## 5. 二维码图像生成

`QRCodeService.generateQRCodeImage(content: string)` 返回 base64 Data URL：

```ts
const dataUrl = await QRCode.toDataURL(content, {
  width: 200,
  margin: 2,
  color: { dark: '#000000', light: '#ffffff' },
  errorCorrectionLevel: 'M',
});
```

底层使用 `qrcode` npm 包（`package.json` 中已声明）。前端展示可使用 `qrcode.react` 组件直接渲染。

## 6. 追溯查询

### 6.1 单码追溯

```ts
const result = await QRCodeService.traceQRCode(qrCode);
// 返回 TraceResult：含物料信息、当前仓库、追溯历史
```

### 6.2 追溯历史

`QRCodeService.getTraceHistory(qrType, sourceId, batchNo?)` 查询 `inv_inventory_transaction` 表，按时间升序返回：

| 事件 | 说明 |
|------|------|
| 入库 / inbound | 物料入库 |
| 出库 / outbound | 物料出库 |
| 调拨 / transfer | 仓库间调拨 |
| 调整 / adjust | 库存调整 |
| 退货 / return | 退货记录 |

每条记录含时间戳、操作人、仓库、数量、备注。

## 7. API 路由

### 7.1 `GET /api/qrcode`

分页查询二维码记录，支持 `qr_type`、`status`、`keyword` 过滤。

### 7.2 `POST /api/qrcode`

创建二维码记录，关联批次时校验 `inv_inventory_batch` 是否存在。

### 7.3 `PUT /api/qrcode`

支持多种 `action`：

| action | 说明 |
|--------|------|
| `batch-consumed` | 批次消耗，将关联二维码状态置为 4 |
| `print` | 打印计数 +1，更新 `last_print_time` |
| `scan` | 扫描记录，写入 `qrcode_scan_log` |
| `invalidate` | 失效（status=3） |
| `void` | 作废（status=9） |
| 默认 | 更新 status / remark 字段 |

### 7.4 `GET /api/qrcode/trace?code=xxx`

公开追溯查询接口，返回完整追溯链路。

## 8. 数据表

| 表 | 作用 |
|----|------|
| `inv_qr_code` | 二维码主记录（旧表，`QRCodeService` 使用） |
| `qrcode_record` | 二维码记录（新表，`/api/qrcode` 使用） |
| `qrcode_scan_log` | 扫描日志 |
| `inv_inventory_transaction` | 库存交易流水（追溯数据源） |
| `inv_inventory_batch` | 批次信息 |

> 注意：项目中存在 `inv_qr_code` 与 `qrcode_record` 两套表，前者由 `QRCodeService` 使用，后者由 `/api/qrcode` 路由使用。新功能建议统一到 `qrcode_record`，逐步迁移。

## 9. 状态码

| status | 含义 |
|--------|------|
| 1 | 正常 / 激活 |
| 2 | 已扫描 |
| 3 | 已失效 |
| 4 | 已消耗（批次） |
| 9 | 已作废 |

## 10. 开发约定

- 二维码生成必须通过 `QRCodeService`，禁止直接操作数据库表。
- 二维码内容必须通过 `buildQRContent` 生成，保证格式统一。
- 失效 / 作废操作必须记录原因（`remark` 字段）。
- 追溯查询走 `trace_url` 公开接口，无需认证，注意脱敏。
- `src/middleware.ts` 在生产环境屏蔽 `/qrcode` 页面（仅 API 可访问），调试需在生产临时改为开发环境。

## 11. 路由访问限制

`/qrcode` 页面在 `NODE_ENV=production` 下被中间件屏蔽（返回 404）。生产环境仅可通过 API 调用二维码功能，前端页面应集成到正式业务模块（如仓储、生产模块）中。
