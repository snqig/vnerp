# 模块创建

本文档说明在 印刷生产经营信息管理系统 Print MIS 中新增一个业务模块的标准步骤，遵循项目既有的 DDD 分层结构与目录约定。

## 1. DDD 分层结构

项目代码按 DDD 分层组织，新增模块应遵循以下分层：

| 层 | 目录 | 职责 |
|----|------|------|
| 表现层 | `src/app/[locale]/<module>/page.tsx` + `src/app/api/<module>/route.ts` | 页面渲染与 HTTP 路由 |
| 应用层 | `src/application/` | 应用初始化、事件注册（跨模块协同时使用） |
| 领域层 | `src/domain/` | 领域类型定义（`DomainTypes.ts`）、业务规则常量 |
| 基础设施层 | `src/infrastructure/` | 事件总线、Outbox、外部服务适配器 |
| 工具层 | `src/lib/<module>-service.ts` | 业务服务、数据库访问、领域逻辑实现 |
| 数据层 | `src/lib/db/schema.ts` + `database/vnerpdacahng_schema.sql` | Drizzle schema 与原始 SQL schema |

## 2. 新增模块步骤

以 `sample-order`（样品订单）模块为例：

### 2.1 数据库 Schema

1. 编辑 `database/vnerpdacahng_schema.sql`，新增表 DDL：

```sql
CREATE TABLE IF NOT EXISTS sample_order (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  order_no VARCHAR(50) NOT NULL,
  customer_id INT NOT NULL,
  status VARCHAR(20) DEFAULT 'draft',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  deleted TINYINT DEFAULT 0
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

2. 同步到 Drizzle schema：编辑 `src/lib/db/schema.ts`：

```ts
import { mysqlTable, serial, varchar, datetime, int } from 'drizzle-orm/mysql-core';

export const sampleOrder = mysqlTable('sample_order', {
  id: serial('id').primaryKey(),
  orderNo: varchar('order_no', { length: 50 }).notNull(),
  customerId: int('customer_id').notNull(),
  status: varchar('status', { length: 20 }).default('draft'),
  createTime: datetime('create_time').default(sql`CURRENT_TIMESTAMP`),
});
```

3. 生成迁移：`pnpm db:generate`，检查 `drizzle/` 下新文件。

### 2.2 业务服务（工具层）

在 `src/lib/` 创建 `sample-order-service.ts`，封装核心业务逻辑：

```ts
import { query, execute, transaction } from '@/lib/db';
import { secureLog } from '@/lib/logger';

export class SampleOrderService {
  static async create(data: { customerId: number; orderNo: string }) {
    return await transaction(async (conn) => {
      const [result]: any = await conn.execute(
        'INSERT INTO sample_order (order_no, customer_id) VALUES (?, ?)',
        [data.orderNo, data.customerId]
      );
      secureLog('info', 'Sample order created', { id: result.insertId });
      return result.insertId;
    });
  }

  static async list(page: number, pageSize: number) {
    return await query(
      'SELECT * FROM sample_order WHERE deleted = 0 ORDER BY id DESC LIMIT ? OFFSET ?',
      [pageSize, (page - 1) * pageSize]
    );
  }
}
```

### 2.3 API 路由

在 `src/app/api/sample-order/route.ts` 创建 REST 端点：

```ts
import { NextRequest } from 'next/server';
import { successResponse, withErrorHandler } from '@/lib/api-response';
import { withAuthAndErrorHandler, UserInfo } from '@/lib/api-auth';
import { SampleOrderService } from '@/lib/sample-order-service';

export const GET = withAuthAndErrorHandler(async (req: NextRequest, userInfo: UserInfo) => {
  const page = Number(new URL(req.url).searchParams.get('page') || 1);
  const list = await SampleOrderService.list(page, 20);
  return successResponse(list);
});

export const POST = withAuthAndErrorHandler(async (req: NextRequest, userInfo: UserInfo) => {
  const body = await req.json();
  const id = await SampleOrderService.create(body);
  return successResponse({ id }, '创建成功');
});
```

### 2.4 页面

在 `src/app/[locale]/sample-order/page.tsx` 创建 Next.js 页面，使用 `useTranslations` 实现多语言：

```tsx
'use client';
import { useTranslations } from 'next-intl';

export default function SampleOrderPage() {
  const t = useTranslations('SampleOrder');
  return <h1>{t('title')}</h1>;
}
```

### 2.5 i18n 文案

在 `messages/zh-CN.json`、`en.json`、`zh-TW.json`、`vi.json` 中新增 `SampleOrder` 命名空间：

```json
"SampleOrder": {
  "title": "样品订单",
  "create": "新增",
  "list": "列表"
}
```

### 2.6 菜单与权限

1. 在 `database/seeds/vnerp-seed-data.sql` 或 `dcprint_menus.sql` 中插入菜单记录。
2. 在 `sys_role_menu` 关联角色权限。
3. 使用 `PermissionGuard` 组件或 `usePermission` Hook 控制按钮可见性。

## 3. 文件清单

新增一个模块通常涉及以下文件：

```
database/vnerpdacahng_schema.sql          # 修改：新增表 DDL
src/lib/db/schema.ts                      # 修改：新增 Drizzle 表定义
drizzle/000X_xxx.sql                      # 自动生成：迁移文件
src/lib/sample-order-service.ts           # 新增：业务服务
src/app/api/sample-order/route.ts         # 新增：API 路由
src/app/[locale]/sample-order/page.tsx    # 新增：页面
messages/{zh-CN,zh-TW,en,vi}.json         # 修改：新增命名空间
```

## 4. 检查清单

- [ ] schema 已同步到 `vnerpdacahng_schema.sql` 与 `src/lib/db/schema.ts`
- [ ] 已生成 Drizzle 迁移并测试 `pnpm db:migrate`
- [ ] API 路由使用 `withAuthAndErrorHandler` 包装
- [ ] 数据库访问使用参数化查询，禁止字符串拼接
- [ ] 所有用户可见文案已加入 4 个语言包
- [ ] ESLint 通过（`pnpm lint`），无硬编码中文告警
- [ ] 单元测试覆盖核心服务逻辑
- [ ] 菜单与权限点已配置

## 5. 反模式（避免）

- ❌ 在 API 路由中直接写业务逻辑，应抽到 `src/lib/*-service.ts`。
- ❌ 字符串拼接 SQL，必须使用参数化查询。
- ❌ 在组件中硬编码中文文案，应使用 `useTranslations`。
- ❌ 在 `src/app/api/` 中创建非 RESTful 路由（如 `/api/sampleOrderList`），应使用 RESTful 命名（`/api/sample-orders`）。
- ❌ 在生产环境暴露调试路由（`/api/diagnose/*`、`/api/test/*`），中间件已屏蔽。
