# 日常操作

本文档说明 VNERP 日常运维的高频操作，供运维人员参考。

## 1. 服务管理

### 1.1 Docker 部署（生产）

```bash
# 查看服务状态
docker compose -f docker-compose.prod.yml ps

# 启动
docker compose -f docker-compose.prod.yml --env-file .env.production up -d

# 停止
docker compose -f docker-compose.prod.yml down

# 重启单个服务
docker compose -f docker-compose.prod.yml restart app
docker compose -f docker-compose.prod.yml restart mysql
docker compose -f docker-compose.prod.yml restart redis

# 查看日志（实时跟踪）
docker compose -f docker-compose.prod.yml logs -f app
docker compose -f docker-compose.prod.yml logs -f --tail=200 app
```

### 1.2 PM2 部署（非 Docker）

```bash
pm2 status                    # 状态
pm2 logs vnerp --lines 100    # 日志
pm2 restart vnerp             # 重启
pm2 reload vnerp              # 零停机重启
pm2 monit                     # 监控面板
```

## 2. 健康检查

```bash
# 应用健康（返回 200 = 健康，503 = 降级）
curl -f http://localhost:5000/api/health

# 数据库连接
docker exec vnerp-mysql-prod mysqladmin -uvnerp_app -p ping

# Redis 连接
docker exec vnerp-redis-prod redis-cli ping
```

`/api/health` 返回 JSON 含 `database`、`memory`、`uptime` 字段，可接入监控告警。

## 3. 日志查看

### 3.1 应用日志

- Docker：`docker compose -f docker-compose.prod.yml logs -f app`
- PM2：`pm2 logs vnerp`
- 直接部署：`/var/log/vnerp/*.log`（按 logrotate 轮转）

日志格式由 `src/lib/logger.ts` 定义，结构化字段：`[时间戳] [级别] [模块:操作] [用户ID] [追踪ID] 消息`。

### 3.2 数据库日志

```sql
-- 操作日志
SELECT * FROM sys_operation_log ORDER BY id DESC LIMIT 50;

-- 登录日志
SELECT * FROM sys_login_log ORDER BY id DESC LIMIT 50;

-- 审计日志
SELECT * FROM audit_logs ORDER BY create_time DESC LIMIT 50;
```

通过 `/api/audit/logs` 与 `/api/audit/report` 接口查询审计日志。

## 4. 数据库日常操作

### 4.1 进入 MySQL 客户端

```bash
# Docker
docker exec -it vnerp-mysql-prod mysql -uvnerp_app -p vnerp

# 本地
mysql -h 127.0.0.1 -u root -p vnerp
```

### 4.2 常用查询

```sql
-- 表大小排序
SELECT table_name, ROUND(data_length/1024/1024, 2) AS size_mb
FROM information_schema.tables
WHERE table_schema = 'vnerp'
ORDER BY data_length DESC LIMIT 20;

-- 当前连接数
SHOW STATUS LIKE 'Threads_connected';

-- 慢查询开关
SHOW VARIABLES LIKE 'slow_query_log%';
```

### 4.3 Drizzle Studio

```bash
pnpm db:studio
# 访问 http://localhost:3000 可视化查看表数据
```

## 5. Redis 日常操作

```bash
# 进入 Redis CLI
docker exec -it vnerp-redis-prod redis-cli

# 查看 token 黑名单
KEYS token:blacklist:*

# 查看缓存
KEYS cache:*

# 清空所有缓存（谨慎）
FLUSHDB

# 内存使用
INFO memory
```

## 6. 备份

### 6.1 手动备份

```bash
# 完整备份
npx tsx scripts/backup-database.ts

# 或 mysqldump
docker exec vnerp-mysql-prod sh -c \
  'exec mysqldump -uvnerp_app -p"$DB_PASSWORD" --single-transaction vnerp' \
  > backup_$(date +%Y%m%d).sql
```

详见 `04-运维指南/备份恢复.md`。

### 6.2 验证备份


## 7. 配置变更

### 7.1 修改环境变量

1. 编辑 `.env.production`。
2. 重启 app 容器：`docker compose -f docker-compose.prod.yml --env-file .env.production up -d --no-deps app`。

⚠️ 修改 `JWT_SECRET` 会使所有现有 token 失效，需通知所有用户重新登录。

### 7.2 修改系统配置

通过 `/api/system/config` 接口或直接 SQL 修改 `sys_config` 表：

```sql
-- 修改密码过期天数
UPDATE sys_config SET config_value = '90'
WHERE config_key = 'system.password_expire_days';

-- 强制首次登录改密码
UPDATE sys_config SET config_value = 'true'
WHERE config_key = 'system.force_change_password';
```

## 8. 应用更新

```bash
# 拉取最新代码
git pull origin main

# 重新构建并启动
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build app

# 验证
curl http://localhost:5000/api/health
```

## 9. 监控指标

| 指标 | 命令 | 告警阈值 |
|------|------|----------|
| 应用响应 | `curl -o /dev/null -w '%{time_total}' http://localhost:5000/api/health` | > 2s |
| 内存 | `docker stats vnerp-app-prod` | > 80% |
| 磁盘 | `df -h` | > 85% |
| MySQL 连接 | `SHOW STATUS LIKE 'Threads_connected'` | > 150（max 200） |
| MySQL 慢查询 | `SHOW STATUS LIKE 'Slow_queries'` | 增长异常 |
| Redis 内存 | `INFO memory` | `used_memory` > maxmemory 80% |
