# 用户管理

本文档说明 VNERP 用户与权限管理操作，对应 `src/app/api/system/user/`、`src/app/api/system/roles/`、`src/app/api/system/profile/password/` 与 `scripts/reset-password.ts`。

## 1. 用户与权限模型

| 表 | 作用 |
|----|------|
| `sys_user` | 用户主表，含账号、密码哈希、状态、部门 |
| `sys_role` | 角色表，含 `role_code`、`role_name`、`data_scope`、`permissions` |
| `sys_user_role` | 用户-角色关联（多对多） |
| `sys_menu` | 菜单表，含 `permission` 权限点 |
| `sys_role_menu` | 角色-菜单关联 |
| `sys_department` | 部门表 |
| `sys_employee` | 员工表（与 `sys_user.username` 关联） |
| `sys_login_log` | 登录日志 |
| `sys_operation_log` | 操作日志 |

权限点格式：`<模块>:<资源>:<操作>`，如 `system:user:delete`、`warehouse:inbound:audit`。

## 2. 用户管理 API

### 2.1 列表查询

`GET /api/system/user`，需认证。参数：

| 参数 | 说明 |
|------|------|
| `page` / `pageSize` | 分页（默认 1 / 20） |
| `username` | 用户名模糊匹配 |
| `realName` | 真实姓名模糊匹配（关联 `sys_employee.name`） |
| `status` | 状态过滤（1 启用 / 0 停用） |

返回用户列表与角色信息。

### 2.2 创建用户

`POST /api/system/user`：

```json
{
  "username": "zhangsan",
  "password": "初始密码",
  "real_name": "张三",
  "email": "zhangsan@example.com",
  "phone": "13800000000",
  "department_id": 1,
  "role_ids": [2, 3],
  "status": 1
}
```

- 密码使用 bcrypt 加盐哈希（10 轮）。
- `first_login` 字段置为 1，配合 `system.force_change_password` 强制首次登录改密。
- 用户名重复返回 409。

### 2.3 启用 / 停用

通过 PUT 接口更新 `status` 字段：

- `status=1`：启用
- `status=0`：停用（用户无法登录，已签发 token 仍有效，必要时撤销）

### 2.4 删除用户

采用软删除（`deleted=1`），不物理删除，保留历史数据完整性。

## 3. 角色管理

### 3.1 查询角色

`GET /api/system/roles`，支持 `withPermissions=true` 返回完整权限继承链。

`sys_role` 关键字段：

| 字段 | 说明 |
|------|------|
| `role_code` | 角色编码（如 `admin`、`warehouse_manager`） |
| `role_name` | 角色名称 |
| `parent_id` | 父角色 ID（支持权限继承） |
| `inherit_mode` | 继承模式 |
| `data_scope` | 数据范围（控制可见数据） |
| `permissions` | 权限 JSON 数组 |

### 3.2 数据范围（data_scope）

| 值 | 含义 |
|----|------|
| `all` | 全部数据 |
| `dept` | 本部门数据 |
| `dept_and_below` | 本部门及下属部门 |
| `self` | 仅本人数据 |

由 `src/lib/api-permissions.ts` 在查询时过滤。

### 3.3 权限继承

角色可通过 `parent_id` 形成继承链，`resolveEffectivePermissions` 递归解析完整权限集，防止循环继承。

## 4. 密码管理

### 4.1 修改密码（用户自助）

`PUT /api/system/profile/password`：

```json
{
  "oldPassword": "旧密码",
  "newPassword": "新密码"
}
```

校验规则：

- 必须包含字母与数字。
- 长度至少 6 位。
- 旧密码必须正确。

修改成功后：

- 更新 `password` 与 `pwd_update_time`，`first_login` 置为 0。
- 调用 `revokeAllUserTokens` 撤销该用户在 **当前 token 签发时刻 +1ms 之前**的所有 token，强制其他设备重新登录。当前请求的 token 保留。

### 4.2 重置密码（管理员）

通过 CLI 脚本 `scripts/reset-password.ts`：

```bash
npx tsx scripts/reset-password.ts <用户名> <新密码>
```

脚本流程：

1. 查询用户是否存在。
2. bcrypt 哈希新密码。
3. `UPDATE sys_user SET password = ? WHERE id = ?`。

⚠️ 该脚本不撤销已签发 token，重置后建议通知用户在其他设备重新登录，或手动调用 `revokeAllUserTokens`。

### 4.3 密码过期策略

`sys_config` 表配置：

| key | 作用 |
|-----|------|
| `system.password_expire_days` | 密码过期天数（如 90） |
| `system.force_change_password` | 是否强制首次登录改密 |

登录时（`src/app/api/auth/login/route.ts`）检查 `pwd_update_time`，超期则返回 `passwordExpired: true`，前端引导改密。

## 5. 登录安全

### 5.1 限流

`src/app/api/auth/login/route.ts` 对单 IP 限流：15 分钟内 20 次请求，超限返回 429。

### 5.2 失败锁定

| 配置 | 默认 |
|------|------|
| 最大尝试次数 | 5 次 |
| 锁定时长 | 15 分钟 |

连续失败 5 次后写入 `lock_time`，锁定期间登录返回 429 + 剩余分钟数。锁定过期后自动重置 `login_fail_count`。

### 5.3 异地登录检测

登录时对比 `last_login_ip` 与当前 IP，不一致时插入 `sys_notification` 告警通知用户。

### 5.4 登录日志

每次登录（成功 / 失败）写入 `sys_login_log`，含 IP、UA、浏览器、OS、状态、消息。

## 6. 日常管理操作

### 6.1 查询用户最近登录

```sql
SELECT username, last_login_ip, last_login_time, login_fail_count, lock_time, status
FROM sys_user WHERE deleted = 0 ORDER BY last_login_time DESC LIMIT 20;
```

### 6.2 解锁账号

```sql
UPDATE sys_user SET login_fail_count = 0, lock_time = NULL WHERE username = 'xxx';
```

### 6.3 查询用户权限

```sql
SELECT m.permission
FROM sys_user_role ur
JOIN sys_role_menu rm ON ur.role_id = rm.role_id
JOIN sys_menu m ON rm.menu_id = m.id
WHERE ur.user_id = <用户ID>
  AND m.permission IS NOT NULL AND m.permission != '';
```

### 6.4 批量停用账号

```sql
UPDATE sys_user SET status = 0 WHERE department_id = <部门ID> AND deleted = 0;
```

### 6.5 撤销用户所有 token

通过 Redis 或 `src/lib/token-blacklist.ts` 的 `revokeAllUserTokens(userId, beforeTs?)` 撤销：

```ts
import { revokeAllUserTokens } from '@/lib/token-blacklist';
await revokeAllUserTokens(userId, Date.now());  // 撤销所有
```

未配置 Redis 时降级为进程内 Map，仅当前实例生效。

## 7. 安全建议

- 默认账号密码（如 `admin / 521223`）仅开发环境可用，生产环境必须修改。
- 生产环境关闭 `NODE_ENV !== 'production' && password === '521223'` 的开发后门（已通过环境判断保护）。
- 离职员工账号应立即停用（`status=0`），不要删除。
- 定期审计 `sys_user_role`，清理多余角色。
- 关键操作（删除、审核）启用二次确认或操作日志审计。
