# 采购管理业务规则

> 文档编号：VNERP-BIZ-003 | 版本：V1.0 | 更新日期：2026-05-10

## 1. 采购申请

### 1.1 申请来源

| 来源 | 说明 | 自动生成条件 |
|------|------|-------------|
| MRP运算 | 物料需求计划自动生成 | 工单下达时库存不足 |
| 手工申请 | 人工发起 | - |
| 安全库存 | 低于安全库存自动触发 | 库存 < 安全库存 |

### 1.2 采购申请状态

```
1(草稿) → 2(待审批) → 3(已审批) → 4(已转订单)
              ↓
           5(已驳回)
```

### 1.3 申请规则

- 申请必须指定物料、数量、期望到货日期
- 审批通过后可转为采购订单
- 一个申请可拆分为多个订单

## 2. 采购订单

### 2.1 订单编号规则

```
PO{年月日}{4位序号}
示例：PO202605100001
```

### 2.2 订单状态

```
1(草稿) → 2(已确认) → 3(部分入库) → 4(已完成)
              ↓
           5(已取消)
```

### 2.3 订单规则

- 订单必须关联供应商
- 订单必须包含至少一条明细
- 确认后自动生成应付账款记录
- 入库数量不得超过订单数量
- 部分入库后状态自动变为"部分入库"

## 3. 采购入库

### 3.1 入库验收规则

- 入库必须经过来料检验（IQC）
- 检验合格后方可入库
- 不合格品退回供应商或进入不合格品处理流程
- 入库自动创建二维码记录

### 3.2 应付账款生成

```
入库时自动生成应付账款：
应付金额 = 入库数量 × 采购单价
```

使用 `fin_payable` 表记录，状态：
- 1(未付款) → 2(部分付款) → 3(已结清)
