# 仓库管理业务规则

> 文档编号：VNERP-BIZ-002 | 版本：V1.0 | 更新日期：2026-05-10

## 1. 入库管理

### 1.1 入库类型

| 类型 | 说明 | 触发来源 |
|------|------|---------|
| 采购入库 | 采购物料到货入库 | 采购订单 |
| 生产入库 | 生产成品完工入库 | 生产工单 |
| 退货入库 | 销售退货入库 | 销售退货单 |
| 调拨入库 | 仓库间调拨入库 | 调拨单 |

### 1.2 入库规则

- 入库必须生成入库单号
- 入库必须指定仓库和库位
- 入库必须记录批次号
- 入库自动创建二维码记录（qrcode_record）
- 采购入库需关联采购订单
- 生产入库需关联生产工单

### 1.3 物料自动拆分

大包装物料入库时自动拆分规则：

```
拆分数量 = 入库数量 / 标准拆分单位数量
余料数量 = 入库数量 % 标准拆分单位数量
```

- 拆分后每条 qrcode_record 记录 `split_flag = 1`（小料）
- 余料记录 `split_flag = 2`（余料）
- 拆分记录的 `parent_qr_id` 指向原始记录

## 2. 出库管理

### 2.1 出库类型

| 类型 | 说明 | 触发来源 |
|------|------|---------|
| 领料出库 | 生产领料出库 | 物料领用单 |
| 销售出库 | 销售发货出库 | 销售发货单 |
| 调拨出库 | 仓库间调拨出库 | 调拨单 |

### 2.2 FIFO 先进先出规则

出库时严格执行 FIFO 原则：

1. 按入库时间排序，优先出库最早入库的物料
2. 同批次物料优先出库
3. 拆分小料优先于整料出库
4. FIFO 分配算法在 `fifo-allocation.ts` 中实现

```
分配逻辑：
1. 查询仓库中该物料的可用库存（status=1, split_flag=0或1）
2. 按入库时间升序排列
3. 依次扣减，直到满足需求数量
4. 更新 qrcode_record 的 status 和数量
```

## 3. 盘点管理

### 3.1 盘点类型

| 编码 | 类型 | 说明 |
|------|------|------|
| 1 | 定期盘点 | 按计划周期执行 |
| 2 | 不定期盘点 | 临时发起 |
| 3 | 循环盘点 | 按物料分类轮转 |
| 4 | 抽盘 | 随机抽查 |

### 3.2 盘点状态流转

```
1(待盘点) → 2(盘点中) → 3(待审核) → 4(已完成)
                ↓
              5(已取消)
```

### 3.3 盘点差异处理

```
盘盈数量 = 实际数量 - 账面数量（> 0）
盘亏数量 = 账面数量 - 实际数量（> 0）
```

- 盘盈：自动生成盘盈入库单
- 盘亏：自动生成盘亏出库单
- 差异需审核确认后调整库存

### 3.4 盘点数据表

使用 `inv_stocktaking` 表，核心字段：
- `taking_no` - 盘点单号
- `taking_type` - 盘点类型（1-4）
- `warehouse_id` - 盘点仓库
- `status` - 状态（1-5）
- `operator_name` - 盘点人

## 4. 调拨管理

### 4.1 调拨类型

| 编码 | 类型 | 说明 |
|------|------|------|
| 1 | 库位调拨 | 同仓库不同库位间调拨 |
| 2 | 仓库调拨 | 不同仓库间调拨 |
| 3 | 其他 | 其他调拨场景 |

### 4.2 调拨状态流转

```
1(待审核) → 2(已出库) → 3(已入库)
   ↓
4(已取消)
```

### 4.3 调拨规则

- 调拨必须指定源仓库和目标仓库
- 调拨出库时扣减源仓库库存
- 调拨入库时增加目标仓库库存
- 调拨过程中物料二维码记录更新仓库归属

### 4.4 调拨数据表

使用 `inv_transfer_order` 表，核心字段：
- `transfer_no` - 调拨单号
- `from_warehouse_id` / `to_warehouse_id` - 源/目标仓库
- `transfer_type` - 调拨类型（1-3）
- `status` - 状态（1-4）
- `operator_name` - 操作人

## 5. 二维码追溯

### 5.1 二维码类型

| qr_type | 说明 | 生成时机 |
|---------|------|---------|
| material | 原料码 | 物料入库时 |
| product | 成品码 | 成品入库时 |
| semi | 半成品码 | 半成品入库时 |

### 5.2 二维码记录表

统一使用 `qrcode_record` 表，核心字段：
- `qr_code` - 二维码值
- `qr_type` - 类型
- `ref_id` - 关联单据ID
- `material_id` - 物料ID
- `quantity` - 数量
- `warehouse_id` - 所在仓库
- `split_flag` - 拆分标识（0-整料 1-小料 2-余料）
- `parent_qr_id` - 父级二维码ID
- `status` - 状态（1-有效 2-已使用 9-作废）

### 5.3 追溯链路

```
原材料入库 → 生成原料码 → 领料出库(关联工单) → 生产报工 → 成品入库(生成成品码) → 销售出库
```

通过 `parent_qr_id` 可追溯拆分关系，通过 `ref_id` 可追溯业务关联。

## 6. 库存预警

### 6.1 预警类型

| 类型 | 规则 | 通知方式 |
|------|------|---------|
| 低库存预警 | 当前库存 < 安全库存 | 系统通知 + 邮件 |
| 过期预警 | 物料有效期 < 30天 | 系统通知 |
| 呆滞预警 | 超过 90 天无出入库记录 | 系统通知 |
