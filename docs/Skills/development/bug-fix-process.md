# Bug 修复流程 SOP

> SOP 编号：VNERP-SKILL-008 | 版本：V1.0 | 更新日期：2026-05-10

## 前置条件

- Bug 已被记录（含复现步骤、预期结果、实际结果）
- 本地开发环境已搭建

## 操作步骤

### 步骤 1：复现 Bug

1. 切换到最新代码：`git pull origin main`
2. 启动开发服务器：`pnpm dev`
3. 按照 Bug 描述复现问题
4. 确认 Bug 存在

### 步骤 2：定位问题

1. **前端问题**：
   - 打开浏览器开发者工具
   - 查看 Console 错误信息
   - 查看 Network 请求和响应
   - 在相关组件中添加 `console.log` 调试

2. **后端问题**：
   - 查看服务器日志输出
   - 检查 API 请求和响应
   - 检查数据库查询结果
   - 使用 `console.error` 记录错误堆栈

3. **数据库问题**：
   - 检查 SQL 语句是否正确
   - 检查数据是否存在
   - 检查字段类型是否匹配
   - 检查事务是否正确提交

### 步骤 3：编写修复代码

1. 创建修复分支：`git checkout -b fix/bug-description`
2. 编写修复代码
3. 确保修复不影响其他功能
4. 添加必要的错误处理

### 步骤 4：验证修复

1. 重现原始 Bug，确认已修复
2. 测试相关功能，确认无回归
3. 运行类型检查：`pnpm ts-check`
4. 运行代码检查：`pnpm lint`
5. 运行单元测试：`pnpm test:unit`

### 步骤 5：提交修复

```bash
git add .
git commit -m "fix: 修复XXX问题

- 问题描述
- 修复方式
- 影响范围"
```

### 步骤 6：记录 Bug 修复

在 `docs/Changes/bug-fixes/` 下创建修复记录。

## 常见 Bug 分类

| 类型 | 常见原因 | 排查方向 |
|------|---------|---------|
| 页面白屏 | JS 运行时错误 | Console 错误、组件渲染异常 |
| API 500 | 服务端异常 | 服务器日志、SQL 错误 |
| 数据不显示 | 接口字段不匹配 | Network 响应、前端字段映射 |
| 操作无响应 | 事件未绑定 | 事件处理函数、状态更新 |
| 数据不一致 | 事务未正确处理 | 数据库操作、事务提交/回滚 |
