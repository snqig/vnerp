# 用户与权限管理 SOP

> SOP 编号：VNERP-SKILL-009 | 版本：V1.0 | 更新日期：2026-05-10

## 前置条件

- 已以管理员身份登录系统
- 了解 RBAC 权限模型

## 操作步骤

### 新增用户

1. 进入 系统管理 → 用户管理
2. 点击"新增用户"
3. 填写用户信息：
   - 用户名（唯一，不可修改）
   - 姓名
   - 邮箱
   - 手机号
   - 初始密码（最少 8 位，含大小写字母和数字）
   - 所属部门
   - 角色（可多选）
4. 点击"保存"
5. 通知用户初始密码，要求首次登录修改

### 修改用户权限

1. 进入 系统管理 → 用户管理
2. 找到目标用户，点击"编辑"
3. 修改角色分配
4. 点击"保存"
5. 用户下次登录时权限生效

### 重置密码

```bash
# 使用脚本重置
cd /opt/vnerp
pnpm tsx scripts/reset-password.ts <username> <new_password>
```

### 禁用/启用用户

1. 进入 系统管理 → 用户管理
2. 找到目标用户
3. 点击"禁用"或"启用"
4. 禁用后用户无法登录

### 角色管理

1. 进入 系统管理 → 角色管理
2. 创建角色并分配权限
3. 权限粒度：菜单权限 + 按钮权限 + API 权限

### 预设角色

| 角色 | 权限范围 |
|------|---------|
| 系统管理员 | 全部权限 |
| 生产经理 | 生产模块全部权限 |
| 仓库管理员 | 仓库模块全部权限 |
| 采购员 | 采购模块读写权限 |
| 品质检验员 | 品质模块读写权限 |
| 财务人员 | 财务模块读写权限 |
| 普通员工 | 基础查看权限 |

## 异常处理

| 问题 | 解决方案 |
|------|---------|
| 用户无法登录 | 检查账户是否被禁用、密码是否正确、是否被锁定 |
| 权限不生效 | 检查角色权限配置，确认用户已分配角色 |
| 忘记密码 | 管理员使用脚本重置密码 |
