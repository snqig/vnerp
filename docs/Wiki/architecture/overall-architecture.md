## 架构特点

### 1. 全栈一体化

- 前端和后端统一在 Next.js 框架中
- 前端页面和 API 路由在同一项目中
- 共享 TypeScript 类型定义
- 统一部署，无需前后端分离

### 2. 原生 SQL 方案

- 不使用 ORM（如 Prisma/TypeORM）
- 直接使用 mysql2 驱动执行参数化 SQL
- 优势：SQL 性能可控、复杂查询灵活
- 劣势：需要手动管理类型安全

### 3. 模块化设计

- 每个业务模块独立目录
- API 路由按模块组织
- 数据库表按模块前缀区分
- 模块间通过 API 接口交互

## 核心数据流

### 生产流程数据流

```
销售订单 → MRP运算 → 采购申请 → 采购订单 → 采购入库
    ↓                                          ↓
生产工单 ← 物料需求 ← BOM展开              来料检验(IQC)
    ↓                                          ↓
标准卡关联 → 派工 → 报工 → 成品入库 → 成品检验(FQC) → 发货
    ↓              ↓        ↓
  成本核算      二维码追溯  库存更新
```

### 二维码追溯数据流

```
入库 → qrcode_record(status=1) → 出库 → qrcode_record(status=2)
  ↓                                    ↓
拆分 → 子记录(split_flag=1/2)      FIFO分配 → 按入库时间排序
  ↓                                    ↓
parent_qr_id 关联                    ref_id 关联工单/订单
```
