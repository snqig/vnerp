# 技术决策记录：数据库选型

> 编号：VNERP-DEC-001 | 日期：2026-05-10 | 状态：已决定

## 背景

项目启动时需要选择关系型数据库，主要考虑以下因素：
- 团队技术栈熟悉度
- 部署运维成本
- 性能需求
- 事务支持

## 选项

### 选项 A：MySQL 8.0

- 优势：团队熟悉、社区成熟、运维工具丰富、性能优秀
- 劣势：JSON 支持不如 PostgreSQL

### 选项 B：PostgreSQL 15

- 优势：JSON 支持优秀、扩展性强、复杂查询性能好
- 劣势：团队不熟悉、运维经验少

### 选项 C：MariaDB 10.x

- 优势：与 MySQL 兼容、性能优秀
- 劣势：生态不如 MySQL、部分特性差异

## 决策

选择 **MySQL 8.0**。

## 理由

1. 团队对 MySQL 最熟悉，开发效率最高
2. MySQL 8.0 的 InnoDB 引擎完全满足事务需求
3. 运维工具和监控方案成熟
4. utf8mb4 字符集完全支持中文
5. 社区活跃，问题容易找到解决方案

## 影响

- 所有 SQL 需兼容 MySQL 语法
- 使用 mysql2 驱动
- 不使用 ORM，直接编写 SQL
