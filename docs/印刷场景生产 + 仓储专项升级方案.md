完全贴合丝网印刷行业刚需，拒绝通用 MES/ERP 的冗余功能，所有能力围绕「降损耗、提效率、可追溯」三个核心目标设计，严格复用现有工单、库存、工装、油墨模块底座，通过事件驱动解耦，不破坏现有 DDD 分层架构。

---

## 一、整体设计原则

1. **行业优先** ：只做印刷场景特有功能，通用生产 / 仓储能力复用现有底座，不重复造轮子
2. **轻量落地** ：每个功能第一版只做 80% 核心场景，比如排产只做机台甘特图拖拽，不上复杂 APS 自动排程
3. **链路闭环** ：所有新能力和现有工单 - 领料 - 完工 - 成本链路打通，不做孤立功能点
4. **事件解耦** ：跨模块联动全部通过领域事件，生产不直接操作库存，库存不直接修改生产数据

# 第一部分：生产管理 印刷场景适配升级

## 模块一：工序报工轻量化落地

### 1.1 核心业务规则（印刷专属）

针对印刷核心工序（晒版、印刷、模切、覆膜、分切、品检）做极简报工，不搞复杂工序流转，只记录核心生产数据。

表格

| 报工核心字段 | 印刷场景说明                                                                                |
| ------------ | ------------------------------------------------------------------------------------------- |
| 机台         | 绑定具体生产设备（如 1 号印刷机、2 号模切机），后续用于排产、OEE 统计                       |
| 操作人员     | 支持多人报工，工时按人数折算                                                                |
| 班次         | 早班 / 中班 / 晚班，适配印刷行业两班倒 / 三班倒特性                                         |
| 产量         | 合格数量 + 不良数量，不良品预置印刷常见原因：套印不准、糊版、刀丝、色差、模切偏位、覆膜起泡 |
| 实际工时     | 开机时长 + 有效生产时长，自动核算机台稼动率                                                 |
| 关联工装     | 自动带出工单绑定的网版 / 刀模，报工后自动累计工装使用次数、触发寿命预警                     |
| 油墨消耗     | 关联工单油墨配方，报工后自动扣减对应油墨库存                                                |

 **强制规则** ：

1. 报工必须关联对应工序的生产工单，无工单不允许报工
2. 累计合格产量达到工单计划量时，工单自动标记为「生产完成」
3. 不良品数量超过工单允许损耗率时，弹窗预警，需备注原因才可提交
4. 报工审核后不可修改，作废需走反向流程冲抵

## 模块二：分切业务专项优化

印刷行业核心特色场景：大卷原材料（纸张、PVC、不干胶）分切成多个小卷投入生产，直接关系物料损耗管控。

### 2.1 核心业务规则

1. **分切单流程** ：创建分切单→选择原材料大卷批次→录入分切后小卷规格与数量→审核执行→库存自动变更
2. **库存自动转换规则**
   * 审核通过后自动扣减大卷原材料库存（对应批次清零或部分扣减）
   * 自动生成 N 个小卷新批次，批次号规则：`原批次号-F001/F002`
   * 自动计算分切损耗率：`损耗率 = (大卷总重量 - 小卷总重量) / 大卷总重量 × 100%`
   * 损耗率超过行业阈值（默认 3%）自动预警，需备注原因
3. **余料自动入库** ：分切剩余的边角料 / 小余卷自动标记为「余料批次」，单独入库，优先供后续小批量工单领用
4. **成本自动分摊** ：大卷原材料成本按重量比例分摊到各小卷批次，余料按实际重量分摊对应成本，保证成本连续

2. **余料管理** ：标记余料批次，纳入余料专属库存池
3. **成本模块** ：同步分切损耗到工单成本（关联工单时）

## 模块三：简易机台排产适配

### 3.1 核心定位

不做复杂 APS 自动排程，只做 **机台维度的可视化甘特图排产** ，满足中小印刷企业「人工排程、拖拽调整」的刚需。

### 3.2 核心功能

1. **机台维度甘特图** ：纵轴为生产机台（印刷机、模切机等），横轴为时间，每个工单对应一个可拖拽的任务条
2. **基础操作**
   * 待排产工单池：左侧展示已审核未排产的工单，可拖拽到对应机台时间轴
   * 拖拽调整：支持拖动工单调整开始时间、拉长缩短调整工期
   * 冲突校验：同机台时间重叠时红色高亮预警，不强制拦截，人工判断
3. **状态同步**
   * 排产确认后工单状态变为「待生产」，同步更新计划开始 / 结束时间
   * 报工后自动更新甘特图任务条的进度（完成百分比）
   * 工单完工后任务条变灰，从排产视图归档
4. **视图切换** ：按日 / 周 / 月三种视图切换，支持导出排产表

# 第二部分：仓储管理 印刷物料适配升级

## 模块一：批次全链路追溯深化

### 1.1 追溯目标

实现 **双向全链路追溯** ，满足印刷行业质量追责、客户验厂需求：

* 正向追溯：原材料批次 → 领料单 → 生产工单 → 工序报工 → 成品入库 → 销售出库 → 客户
* 反向追溯：成品批次 → 对应生产工单 → 所用原材料批次 → 供应商 → 操作人员 / 机台

### 1.2 实现逻辑

无需新建冗余追溯表，基于现有单据的批次关联关系，通过聚合查询实现：

1. **关联规则**
   * 原材料出库时记录关联工单 ID，批次号与工单绑定
   * 成品入库时记录所用原材料批次列表，写入成品批次属性
   * 工序报工记录操作人员、机台、不良信息，与工单绑定
2. **追溯查询服务**
   * 输入任意批次号（原料 / 成品），自动串联全链路节点
   * 输出链路图谱 + 明细列表，每个节点可跳转对应单据详情
3. **数据一致性** ：所有关联关系通过事件自动写入，无需人工维护，避免漏填错填

### 1.3 前端交互

* 批次详情页新增「全链路追溯」Tab，可视化展示追溯链路
* 支持导出追溯报告，用于客户验厂、质量分析

## 模块二：物料有效期与批次管控

针对油墨、纸张、胶水等有保质期的印刷物料，做全生命周期有效期管控。

### 2.1 核心规则

1. **基础配置** ：物料档案新增「保质期天数」字段，分类设置（如油墨 180 天、纸张 365 天）
2. **入库自动算效期** ：批次入库时，根据生产日期 + 保质期自动计算到期日，无生产日期则按入库日期算
3. **临期预警**
   * 提前 30 天进入临期状态，库存列表黄色标记
   * 首页预警看板展示临期物料清单，推送通知给仓管
   * 临期物料领用时弹窗提醒，优先消耗
4. **过期锁定** ：到期批次自动锁定，禁止领料、出库，需走报废流程
5. **强制先进先出（FIFO）**
   * 有有效期的物料出库时，系统自动优先分配最早到期的批次
   * 不允许手动跳批次领新货，强制临期先消耗，减少过期浪费

### 2.2 数据模型扩展

ts

```
// inv_material 物料表扩展

needExpiryManage: tinyint('need_expiry_manage').default(0), // 是否需要效期管理

expireDate: date('expire_date'), // 到期日

```

### 2.3 联动逻辑

* 入库时自动计算到期日，标记效期状态
* 每日定时任务扫描批次，自动更新临期 / 过期状态，触发预警通知
* 出库分配时，效期管理物料强制按到期日升序分配，替代原纯 FIFO 规则

## 模块三：余料专属管理

针对分切余料、生产退料余料，做单独标识与优先领用机制，直接降低物料损耗。

### 3.1 核心规则

1. **余料标识** ：所有余料批次统一标记 `is_surplus=1`，批次号带特定后缀便于识别

* 分切余料：原批次号 - FY001
* 生产退料余料：原批次号 - TY001

1. **优先领用规则**
   * 创建领料单时，系统自动优先匹配同规格的余料库存
   * 余料不足部分再分配整卷 / 整包常规批次
   * 仓管可手动调整，但需备注不领用余料的原因
2. **余料预警** ：余料库存超过 30 天未领用自动提醒，优先安排小批量工单消耗
3. **损耗统计** ：按月统计余料产生量、领用量、报废量，核算物料损耗率，支撑工艺优化

### 3.2 数据模型扩展

批次表新增余料标记：

ts

```
// inv_inventory_batch 扩展

surplusSource: tinyint('surplus_source'), // 1-分切产生 2-生产退料
surplusCreateTime: datetime('surplus_create_time'), // 余料生成时间
```

### 3.3 联动逻辑

* 分切审核后自动生成余料批次
* 生产退料审核后，不足最小包装的退料自动标记为余料
* 领料单生成时，自动按「余料优先→常规批次 FIFO」的顺序分配库存
* 库存报表新增「余料库存统计表」，展示余料库龄、金额、消耗情况

### 4.2 报工模块实际表结构与设计的差异

实际 `prd_work_report` 表与本文档 1.2 节设计存在以下差异，对齐时需注意：

| 设计字段 | 实际字段 | 说明 |
| -------- | -------- | ---- |
| `machine_id` / `machine_name` | `equipment_id` | 实际关联 `eqp_equipment` 设备表（机台 = 设备） |
| `operator_ids`（JSON 数组） | `operator_id` / `operator_name` | 实际单操作人，未支持多人报工 |
| `shift`（班次） | 未实现 | 班次字段缺失，OEE 按班次统计待补 |
| `qualified_qty` / `bad_qty` | `completed_qty` / `qualified_qty` / `defective_qty` / `scrap_qty` | 实际拆分更细：完成数/合格/不良/报废 |
| `tool_ids`（关联工装） | `die_template_id`（刀模模板） | 实际仅关联刀模模板，网版关联未实现 |
| 油墨消耗扣减 | 未实现 | 报工未联动油墨库存扣减 |
| — | `is_first_piece` / `first_piece_status` / `first_piece_inspector` | 实际增补首件检验字段（印刷行业刚需，本文档未覆盖） |

### 4.3 报工跨模块联动现状

本文档 1.3 节设计的 `WorkReportedEvent` 事件驱动联动（工装/库存/成本/工单）**尚未统一实现**：

- 工装寿命累计：报工 API 内直接处理 `die_template_id` 累计 `cumulative_impressions`，未走事件 → 工装模块订阅的标准链路
- 油墨库存扣减：未实现
- 工单产量累计：未通过事件联动
- 成本归集：未通过事件联动

> 建议后续将报工审核后的联动统一收敛为 `WorkReportedEvent` 事件驱动，对齐本文档事件解耦设计。
