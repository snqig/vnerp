# 打样工艺卡录入页统一完善方案

## 一、背景与目标

当前存在两套打样工艺卡录入页面：
- **/sample/standard-card/input（经典版）**：分步引导式表单，适合新人操作和非标复杂工艺。
- **/sample/standard-card/input-v2（高效版）**：单页高密度快捷录入，支持批量粘贴、实时成本预览，适合熟手和批量录单。

两套页面在数据字段、业务逻辑、校验规则上存在显著差异，导致同一份工艺卡在不同页面录入后数据不一致，无法联动报价、生产等后续环节，严重制约业务效率。

**本方案的核心目标**：
1. 经典版以 v2 高效版为基准，补齐全部数据字段和业务能力，保证两套页面产出完全一致的工艺卡数据。
2. 将打样工艺卡打造为全链路（打样→报价→生产）的统一数据源头，实现一次录入、全流程复用。
3. 双录入页底层共用一套数据模型、业务逻辑和接口，仅前端交互形态差异化，降低长期维护成本。

## 二、核心对齐原则（双录入页同源同库）

- **数据层 100% 共用**：同一套数据库表（`dcprint_sample_process_card` 及明细表），不区分 v1/v2 字段。
- **业务逻辑 100% 共用**：同一套校验规则（Zod Schema）、成本计算、版本管理、状态流转。
- **后端接口 100% 共用**：增删改查、提交、生成工单/报价等接口入参完全一致。
- **仅前端渲染形态不同**：经典版分步引导，高效版单页快捷，但底层表单状态管理、提交逻辑、数据转换完全复用公共 Hook。
- **数据互通**：同一份工艺卡可从任意入口编辑，数据实时同步，版本历史完全共用。

## 五、打样工艺卡全流程融入方案

### 5.1 核心定位与全链路流转逻辑

打样工艺卡是印刷行业“客户需求 → 成本测算 → 报价 → 大货生产”全链路的核心数据源头。其承载三类关键信息：
- **产品定义**：基材、规格、印刷要求、质量标准。
- **工艺路线**：工序顺序、参数、工装要求、油墨配方。
- **成本基准**：物料用量、工时定额、工装分摊、损耗率。

**全链路流转**：
```

客户打样需求 → 录入工艺卡 → 生成打样工单
↓
打样验证 → 工艺卡版本确认 → 一键生成报价单
↓
客户确认报价 → 一键生成正式生产工单 → 自动带出 BOM/工艺/工装
↓
生产执行 → 完工入库 → 实际成本 vs 打样预估成本对比分析

**text**

```

**核心规则**：同一款产品的报价、生产工艺，必须以“已确认的打样工艺卡版本”为基准，保证打样-报价-大货工艺一致性。

### 5.3 融入细节

#### 5.3.1 打样自身闭环（基础能力强化）

* **实时成本预览** ：录入物料、工时、工装同时，底部实时计算总成本（物料成本 = Σ(单耗×库存成本)×(1+损耗率)，工时成本 = 总工时×工时单价，工装成本全额或分摊）。
* **关联印前主数据** ：油墨色号、工装、工序均下拉选择，自动带出关联数据。
* **版本管理** ：一键“生成新版本”，版本号自动+1，仅“已确认”版本可用于报价和正式生产。
* **一键生成打样工单** ：提交后自动生成打样工单，带出 BOM、工艺、工装。

#### 5.3.2 融入报价流程

* **一键生成报价单** ：在已确认的工艺卡详情页点击“生成报价单”，自动带入产品信息、物料成本、人工成本、工装成本、制造费用。报价仅需调整利润率、含税点、优惠金额，效率提升 80%。
* **版本绑定** ：报价单绑定工艺卡版本，工艺卡变更后报价可一键同步更新。
* **报价溯源** ：报价详情可直接跳转查看对应工艺卡。

#### 5.3.3 融入正式生产流程

* **一键生成正式工单** ：从确认的报价单/工艺卡直接生成，自动带出 BOM、工艺路线、工装、油墨配方。
* **成本核算基准对齐** ：工单预估成本 = 工艺卡基准成本 × 生产数量；完工后自动对比实际成本，分析差异。
* **标准工艺沉淀** ：量产验证稳定的工艺卡可标记为“标准工艺模板”，后续同类型产品直接复用。

#### 5.3.4 融入印前模块内闭环

* **油墨配方联动** ：选择色号自动带出配方明细，写入物料 BOM，领料自动带出配比。
* **工装联动** ：选择刀模/网版自动带入成本，工单自动关联并累计寿命。
* **反向迭代** ：打样中调整的参数可同步更新到油墨/工装模块的新版本。

---

### 5.4 架构落地方式

* **领域归属** ：打样工艺卡属于印前领域（dcprint）下的打样子域，聚合根为 `SampleProcessCard`，封装版本流转、校验、生成报价/工单规则。
* **跨模块事件驱动** ：通过领域事件联动（如 `SampleCardConfirmedEvent` → 报价模块，`SampleWorkOrderCreatedEvent` → 生产模块），零耦合。
* **复用现有能力** ：版本管理复用油墨配方范式，成本计算复用库存成本逻辑，工单流程复用现有生产链路。

### 5.5 前端交互升级（input-v2 改造）

* **顶部区** ：打样编号、版本号、状态标签、操作按钮（保存、提交、生成报价、转正式生产、版本对比）。
* **左侧表单区** ：基础信息、印前参数（色号/工装联动）、物料明细（支持增删行、自动算成本）、工序路线（拖拽排序）。

* **快捷入口** ：工艺卡列表页直接操作“生成报价”“转正式工单”；报价/工单创建页支持“从工艺卡导入”。

## 八、最终验收标准

1. **数据一致性** ：同一组数据分别在两个页面提交，生成的工艺卡所有字段、关联明细、版本号、状态完全一致。

4. **工程健康** ：公共逻辑抽离完成，无重复代码，通过 lint 和类型检查。
