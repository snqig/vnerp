二、各目录核心内容与规则
1. Rules (规则体系)
核心原则：所有规则必须可执行、可验证、可自动化检查
编码规范：统一代码风格、命名规则、注释规范、文件结构
数据库规则：表命名、字段命名、索引设计、外键约束、事务使用规范
API 设计规则：RESTful 规范、状态码使用、请求 / 响应格式、错误处理
业务规则：每个业务流程的核心逻辑规则、计算公式、异常处理规则
安全规则：用户认证、权限控制、数据加密、SQL 注入防护、XSS 防护
2. Skills (技能 SOP)
核心原则：步骤化、可复制、无歧义，任何人按照 SOP 都能完成操作
每个 SOP 必须包含：前置条件、操作步骤、预期结果、异常处理
关键步骤必须配有截图或命令示例
复杂操作拆分为多个子 SOP，每个子 SOP 不超过 10 个步骤
3. harness (测试框架)
核心原则：自动化优先、覆盖全面、可重复执行
单元测试：覆盖所有模型方法、服务层方法、工具函数
集成测试：覆盖所有 API 接口、数据库操作、模块间交互
E2E 测试：覆盖所有核心业务流程的完整链路
测试数据：使用独立的测试数据库，每次测试前自动重置数据
测试报告：自动生成详细的测试报告，包含覆盖率、失败用例详情
4. Wiki (知识库)
核心原则：结构化、可搜索、持续更新
架构文档：必须包含清晰的架构图，说明各组件之间的关系
业务知识库：记录行业知识和业务逻辑，帮助新成员快速上手
技术知识库：记录技术难点、解决方案、最佳实践
技术决策记录：记录每个重要技术决策的背景、选项、理由和结果
5. Changes (变更记录)
核心原则：可追溯、可回滚、透明化
版本变更记录：每个版本包含新增功能、修复 Bug、破坏性变更
数据库迁移记录：每个数据库变更都有对应的迁移脚本和回滚脚本
Bug 修复记录：每个 Bug 包含问题描述、影响范围、解决方案、验证方法
三、全链路精确测试验证方案
测试总原则
全覆盖原则：测试必须覆盖每一个框架、每一个录入框、每一个逻辑、每一个计算、每一个环节
自动化优先原则：所有可自动化的测试都必须编写自动化测试用例
数据驱动原则：使用多组测试数据验证同一个功能，包括正常数据、边界数据和异常数据
可重复原则：测试过程必须可重复，每次测试都能得到相同的结果
1. 框架级测试
表格
测试项	测试内容	验证方法	预期结果
路由框架	所有路由是否正确注册	自动化测试 + 手动验证	所有路由都能正常访问，不存在 404 错误
ORM 框架	所有模型是否正确映射数据库表	自动化测试	模型 CRUD 操作正常，数据一致性
表单验证框架	所有表单验证规则是否生效	自动化测试 + 手动输入	非法输入被正确拦截，显示正确的错误信息
权限框架	所有权限控制是否生效	自动化测试 + 多角色登录验证	无权限用户无法访问受保护资源
日志框架	所有关键操作是否正确记录日志	手动操作 + 查看日志文件	日志包含时间、用户、操作、结果等信息
2. 页面与录入框级测试
通用测试用例：
所有页面是否能正常加载，无 JS 错误
所有按钮、链接是否能正常点击
所有录入框是否支持正确的输入类型
所有录入框的最大长度限制是否生效
所有必填项是否有明确的标识，未填写时是否有正确的提示
所有下拉框、单选框、复选框是否能正常选择
所有日期选择器是否能正常使用
所有文件上传功能是否能正常工作，支持正确的文件类型和大小限制
分模块测试：
生产管理模块：工单创建、工单编辑、工单派工、工单报工、工单完工
仓库管理模块：入库单、出库单、库存盘点、库存调拨、二维码追溯
采购管理模块：采购申请、采购订单、采购入库、采购付款
品质管理模块：来料检验、过程检验、成品检验、不合格品处理
系统管理模块：用户管理、角色管理、权限管理、系统设置
3. 业务逻辑与计算级测试
核心计算公式验证：
生产工单：物料需求计算、工时计算、成本计算
库存管理：库存数量计算、库存金额计算、库存预警计算
采购管理：采购金额计算、应付账款计算
财务管理：应收账款计算、利润计算
测试方法：
编写自动化测试用例，使用多组测试数据验证计算结果
手动计算预期结果，与系统计算结果进行对比
边界值测试：使用最小值、最大值、零值、空值等边界数据进行测试
异常场景测试：测试数据异常、网络异常、系统异常等情况下的处理逻辑
4. 全流程端到端测试
核心业务流程测试：
订单到生产流程：
创建销售订单 → 生成生产工单 → 物料需求计算 → 采购原材料 → 原材料入库 → 生产派工 → 生产报工 → 成品入库 → 销售出库
二维码追溯流程：
生成原材料二维码 → 原材料入库扫码 → 生产领料扫码 → 生产过程扫码 → 成品二维码生成 → 成品入库扫码 → 成品出库扫码 → 扫码追溯全流程
测试要求：
每个流程至少测试 3 次，使用不同的测试数据
测试过程中模拟各种异常情况，如物料不足、生产延期、质量不合格等
验证流程中所有数据的一致性，如库存数量、工单状态、财务数据等
5. 性能与安全测试
性能测试：测试系统在高并发情况下的响应时间和吞吐量
负载测试：测试系统在最大负载情况下的稳定性
安全测试：测试 SQL 注入、XSS 攻击、CSRF 攻击等常见安全漏洞
数据备份与恢复测试：验证数据备份和恢复功能是否正常工作
四、测试执行与报告
测试计划：制定详细的测试计划，明确测试范围、测试时间、测试人员和测试资源
测试用例管理：使用测试用例管理工具管理所有测试用例，记录每个用例的执行结果
缺陷管理：使用缺陷管理工具记录所有发现的缺陷，跟踪缺陷的修复过程
测试报告：每次测试完成后生成详细的测试报告，包含测试覆盖率、缺陷统计、风险评估等内容
回归测试：每个缺陷修复后都必须进行回归测试，确保缺陷已被正确修复且没有引入新的缺陷
