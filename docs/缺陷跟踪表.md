#### BUG-LOGIN-001-20260326
- **缺陷标题**: 登录按钮点击事件未触发
- **所属模块**: 登录模块
- **严重程度**: 严重 (High)
- **优先级**: P1
- **发现日期**: 2026-03-26
- **发现人**: 测试工程师
- **状态**: 已修复
- **指派给**: 开发团队

**缺陷描述**:
在登录页面输入用户名和密码后，点击登录按钮没有触发提交事件，无法完成登录操作。

**重现步骤**:
1. 访问 http://localhost:5000/login
2. 输入用户名: admin
3. 输入密码: admin
4. 点击"登录"按钮

**预期结果**:
表单提交，调用登录API，成功后跳转到首页

**实际结果**:
按钮点击无响应，表单未提交

**环境信息**:
- 浏览器: Chrome 120+
- 操作系统: Windows 11
- 分辨率: 1920x1080

**修复方案**:
检查Button组件的type="submit"属性是否正确传递

**验证结果**:
✅ 已验证通过 - 2026-03-26

#### BUG-AUTH-001-20260326
- **缺陷标题**: admin用户显示为"普通用户"角色
- **所属模块**: 权限管理模块
- **严重程度**: 中等 (Medium)
- **优先级**: P2
- **发现日期**: 2026-03-26
- **发现人**: 测试工程师
- **状态**: 已修复
- **指派给**: 开发团队

**缺陷描述**:
admin用户登录后，侧边栏显示角色为"普通用户"，但实际应该是"超级管理员"。导致无法访问入库管理等需要管理员权限的页面。

**重现步骤**:
1. 使用admin/admin登录系统
2. 查看侧边栏底部用户信息显示的角色
3. 尝试访问 /warehouse/inbound 页面

**预期结果**:
- 显示角色: 超级管理员
- 可以访问所有管理页面

**实际结果**:
- 显示角色: 普通用户
- 提示"暂无菜单权限"

**根因分析**:
usePermission.ts中检查超级管理员权限时，错误地使用了`user.role_code`而不是`user.roles`数组

**修复方案**:
将权限检查逻辑从 `user.role_code === 'super_admin'` 修改为 `user.roles?.some((r: any) => r.role_code === 'super_admin')`

**验证结果**:
✅ 已验证通过 - 2026-03-26

#### BUG-UI-001-20260326
- **缺陷标题**: 登录页面标签文字与背景对比度不足
- **所属模块**: UI/登录模块
- **严重程度**: 轻微 (Low)
- **优先级**: P3
- **发现日期**: 2026-03-26
- **发现人**: 测试工程师
- **状态**: 已修复
- **指派给**: 开发团队

**缺陷描述**:
登录页面表单标签（用户名、密码、记住我）使用灰色文字，在透明背景下对比度不足，难以阅读。

**重现步骤**:
1. 访问 http://localhost:5000/login
2. 观察表单标签文字颜色

**预期结果**:
文字颜色与背景有足够对比度，易于阅读

**实际结果**:
灰色文字在深色透明背景下显示不清

**修复方案**:
将标签文字颜色从 `text-gray-700` 改为 `text-white`

**验证结果**:
✅ 已验证通过 - 2026-03-26

## 严重程度定义

| 级别 | 定义 | 示例 |
|------|------|------|
| 致命 | 系统崩溃、数据丢失、安全漏洞 | 系统无法启动、数据库连接失败 |
| 严重 | 核心功能不可用、主要流程阻断 | 无法登录、无法提交订单 |
| 中等 | 次要功能异常、有替代方案 | 页面加载慢、部分数据显示错误 |
| 轻微 | 界面问题、不影响功能使用 | 文字错别字、布局轻微错位 |
