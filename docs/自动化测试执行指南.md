# 达昌ERP系统自动化测试执行指南

## 测试框架概述

本项目使用 **Playwright** 作为端到端（E2E）自动化测试框架，支持多浏览器测试和可视化测试报告。

## 环境要求

- Node.js 18+
- pnpm 包管理器
- Chrome/Firefox/Safari 浏览器

## 快速开始

### 1. 安装浏览器驱动

首次运行测试前，需要安装 Playwright 浏览器：

```bash
cd erp-project
pnpm run test:install
```

或手动安装：

```bash
npx playwright install
```

### 2. 启动开发服务器

确保应用已启动：

```bash
pnpm run dev
```

应用将在 http://localhost:5000 运行

### 3. 执行测试

#### 运行所有测试

```bash
pnpm run test
```

#### 运行特定测试文件

```bash
npx playwright test tests/login.spec.ts
```

#### 以有头模式运行（可见浏览器窗口）

```bash
pnpm run test:headed
```

#### 使用UI模式（调试友好）

```bash
pnpm run test:ui
```

#### 调试模式

```bash
pnpm run test:debug
```

### 4. 查看测试报告

测试完成后，生成并查看HTML报告：

```bash
pnpm run test:report
```

报告将自动在浏览器中打开，包含：
- 测试通过率统计
- 失败的测试截图
- 测试执行视频回放
- 详细的错误日志

## 持续集成配置

### GitHub Actions 示例

```yaml
name: Playwright Tests

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - uses: actions/setup-node@v3
      with:
        node-version: 18
    - name: Install dependencies
      run: pnpm install
    - name: Install Playwright
      run: npx playwright install --with-deps
    - name: Run tests
      run: pnpm run test
    - name: Upload report
      uses: actions/upload-artifact@v3
      if: always()
      with:
        name: playwright-report
        path: playwright-report/
```

## 常见问题

### Q1: 测试运行时浏览器闪退

**解决方案**:
```bash
# 重新安装浏览器
npx playwright install --force
```

### Q2: 测试超时

**解决方案**: 在 `playwright.config.ts` 中增加超时时间：
```typescript
use: {
  actionTimeout: 30000,
  navigationTimeout: 60000,
}
```

### Q3: 测试在CI中失败但在本地通过

**可能原因**:
- 屏幕分辨率差异
- 网络延迟
- 浏览器版本差异

**解决方案**:
```bash
# 使用相同的视口大小
npx playwright test --viewport-size="1920,1080"
```

## 编写新测试用例

### 基本结构

```typescript
import { test, expect } from '@playwright/test';

test.describe('功能模块名称', () => {
  
  test.beforeEach(async ({ page }) => {
    // 前置条件
    await page.goto('/path');
  });

  test('测试用例标题', async ({ page }) => {
    // 测试步骤
    await page.click('selector');
    
    // 断言验证
    await expect(page.locator('selector')).toHaveText('expected');
  });
});
```

### 最佳实践

1. **使用语义化选择器**: 优先使用 `data-testid` 或文本内容
2. **保持测试独立**: 每个测试用例应该独立运行
3. **添加适当的等待**: 使用 `waitForSelector` 而非固定延时
4. **截图记录**: 关键步骤添加截图便于问题定位
